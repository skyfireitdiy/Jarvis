# 全局文件内容搜索功能规范

## 功能概述

为文件编辑器添加全局搜索功能，允许用户在当前 Agent 工作目录下搜索文件内容，快速定位代码和文本。

**使用场景**：
- 作为开发者，我需要在整个工作目录中搜索特定的函数名、变量名或代码片段
- 作为开发者，我需要找到包含特定错误信息或注释的文件
- 作为开发者，我需要在多个文件中查找相同的关键词

**功能边界**：
- 范围：当前选中的 Agent 工作目录
- 搜索对象：文本文件内容（排除二进制文件）
- 展示方式：在编辑器面板中展示搜索结果列表

## 接口定义

### 1. 后端 API：全局文件内容搜索

**端点**：
```
POST /api/global-search/{agent_id}
```

**请求参数**：
```json
{
  "query": "string",       // 搜索关键词
  "case_sensitive": false, // 是否区分大小写（默认 false）
  "max_results": 100,      // 最大结果数量（默认 100）
  "file_glob": "*.py,!tests/**" // 文件过滤 glob，逗号分隔，支持 ! 排除
}
```

**响应格式**：
```json
{
  "success": true,
  "data": {
    "query": "search_string",
    "total_files": 10,
    "total_matches": 45,
    "results": [
      {
        "file_path": "src/module/file.py",
        "matches": [
          {
            "line_number": 42,
            "line_content": "    def search_string(self, param):",
            "match_start": 12,
            "match_end": 25
          }
        ]
      }
    ]
  }
}
```

**错误响应**：
```json
{
  "success": false,
  "error": {
    "code": "AGENT_NOT_FOUND",
    "message": "Agent not found"
  }
}
```

### 2. 前端界面组件

**编辑器侧边栏布局**：
- 位置：编辑器面板内部，采用活动栏 + 侧边栏布局
- 活动栏视图：目录树、全局搜索
- 默认视图：目录树
- 关闭行为：侧边栏支持手动关闭，不遮挡 Monaco 编辑区域

**目录树视图**：
- 数据来源：复用左侧已有 Agent 文件树数据与展开状态
- 展示范围：当前活动 Agent 的完整工作目录树
- 目录行为：支持展开/折叠
- 文件行为：点击文件节点后直接在编辑器中打开对应文件

**搜索视图**：
- 位置：编辑器内部侧边栏中的独立视图

**搜索输入框**：
- 类型：文本输入
- 占位符："全局搜索文件内容..."
- 快捷键：Ctrl/Cmd + Shift + F

**搜索结果面板**：
- 显示位置：编辑器内部侧边栏
- 结果列表格式：
  - 文件路径（可点击打开文件）
  - 匹配行号
  - 匹配行内容（高亮关键词）

**文件过滤输入框**：
- 类型：文本输入
- 用途：限制搜索参与匹配的文件路径
- 语法：glob 通配符，多个规则以逗号分隔，支持 `!` 前缀排除
- 示例：`*.py`、`src/**/*.ts`、`*.py,!tests/**`

## 输入输出说明

### 输入参数

**query（搜索关键词）**：
- 类型：string
- 约束：非空字符串，最大长度 200 字符
- 示例："function_name", "TODO", "import os"

**case_sensitive（区分大小写）**：
- 类型：boolean
- 默认值：false
- 约束：可选

**max_results（最大结果数）**：
- 类型：integer
- 默认值：100
- 约束：1-500 之间的整数

**file_glob（文件过滤规则）**：
- 类型：string
- 默认值：空字符串（不过滤）
- 约束：可选，最大长度 500 字符
- 语法：glob 通配符，多个规则使用逗号分隔
- 包含规则示例：`*.py`、`src/**/*.vue`
- 排除规则示例：`!tests/**`、`!node_modules/**`
- 组合示例：`*.py,!tests/**`

### 输出说明

**成功响应**：
- success: true
- data: 搜索结果对象
  - query: 使用的搜索关键词
  - total_files: 命中的文件数量
  - total_matches: 命中的总匹配数
  - results: 匹配结果列表
    - file_path: 文件相对路径
    - matches: 该文件的匹配列表
      - line_number: 行号（从 1 开始）
      - line_content: 行完整内容
      - match_start: 匹配起始位置
      - match_end: 匹配结束位置

**错误响应**：
- success: false
- error: 错误对象
  - code: 错误代码（AGENT_NOT_FOUND, INVALID_QUERY, SEARCH_FAILED）
  - message: 错误描述

## 功能行为

### 正常情况处理

1. **用户发起搜索**：
   - 用户在搜索框输入关键词，并可选输入文件过滤 glob
   - 用户按回车或点击搜索按钮
   - 前端发送 POST 请求到 `/api/global-search/{agent_id}`
   - 后端在 Agent 工作目录下结合 `file_glob` 过滤文件范围
   - 对过滤后的文本文件内容进行关键词匹配
   - 返回匹配结果列表
   - 前端展示搜索结果

2. **查看目录树**：
   - 编辑器侧边栏默认展示当前活动 Agent 的完整工作目录树
   - 点击目录节点，可展开或折叠子节点
   - 点击文件节点，直接在编辑器中打开对应文件

3. **查看搜索结果**：
   - 切换到搜索视图后，点击文件路径，打开对应文件
   - 点击匹配行，跳转到对应行号并高亮

4. **搜索性能优化**：
   - 搜索大文件时，限制每个文件的最大匹配数（如 50 个）
   - 支持搜索超时机制（如 30 秒超时）
   - 优先搜索源代码文件（.py, .js, .ts, .vue, .java 等）

### 边界条件处理

1. **空搜索词**：
   - 不发起搜索请求，提示用户输入搜索词

2. **搜索词过长**：
   - 限制最大长度 200 字符
   - 超过长度提示用户

3. **没有匹配结果**：
   - 显示"未找到匹配结果"提示
   - 保持搜索框可继续输入

4. **glob 规则为空**：
   - 不做额外文件过滤
   - 搜索行为与当前实现一致

5. **glob 规则存在但未命中任何文件**：
   - 返回空结果列表
   - total_files: 0, total_matches: 0

6. **工作目录为空**：
   - 返回空结果列表
   - total_files: 0, total_matches: 0

### 异常情况处理

1. **Agent 不存在**：
   - 返回错误代码 AGENT_NOT_FOUND
   - 前端显示错误提示

2. **工作目录不存在**：
   - 返回错误代码 WORKING_DIR_NOT_FOUND
   - 前端显示错误提示

3. **权限不足**：
   - 跳过无权限访问的文件
   - 在响应中记录被跳过的文件数（可选）

4. **搜索超时**：
   - 终止搜索并返回已找到的结果
   - 在响应中标注"部分结果"

5. **二进制文件**：
   - 自动检测并跳过二进制文件
   - 不搜索 .png, .jpg, .pdf, .exe 等文件

6. **glob 语法输入异常**：
   - 对空片段进行忽略
   - 对非空 glob 片段按原样传递给搜索命令
   - 若搜索命令执行失败，返回 SEARCH_FAILED

## 性能要求

- 搜索响应时间：小于 5 秒（搜索 100 个文件）
- 支持最大文件数：1000 个文件
- 单个文件最大大小：1MB（超过则跳过）
- 最大结果数：默认 100 个匹配项（可配置）

## 实现建议

### 后端实现

1. 使用 `ripgrep (rg)` 或 `grep` 命令进行高效搜索
2. 优先搜索源代码文件（通过文件扩展名过滤）
3. 实现搜索超时机制
4. 使用文件大小限制避免处理过大的文件

### 前端实现

1. 在编辑器工具栏添加搜索输入框
2. 使用折叠面板展示搜索结果
3. 支持点击结果跳转到对应行（使用 Monaco 编辑器的 `revealLineInCenter`）
4. 高亮显示匹配的关键词

### 文件过滤建议

**默认搜索的文件扩展名**：
- Python: .py, .pyx
- JavaScript: .js, .ts, .jsx, .tsx
- Web: .html, .css, .scss, .vue, .svelte
- 其他: .java, .c, .cpp, .h, .go, .rs, .rb, .php
- 配置文件: .json, .yaml, .yml, .toml, .ini, .conf
- 文档: .md, .txt, .rst

**默认排除的文件和目录**：
- 版本控制: .git/, .svn/
- 依赖目录: node_modules/, __pycache__/, .venv/, venv/
- 构建产物: dist/, build/, .next/
- 二进制文件: .png, .jpg, .jpeg, .gif, .pdf, .zip, .tar.gz, .exe
