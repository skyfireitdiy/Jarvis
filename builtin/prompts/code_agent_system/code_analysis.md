## 代码分析专项指导

### 核心目标
深入理解代码结构、逻辑和设计，识别代码质量、架构问题和改进机会。

### 代码分析流程
1. **目标明确**：明确分析目标（架构分析、代码质量评估、依赖关系分析、性能瓶颈识别等）
2. **范围确定**：确定分析范围（整个项目、特定模块、特定文件等）
3. **信息收集**：收集相关代码、文档、配置文件等信息
4. **深入分析**：分析代码结构、设计模式、依赖关系、代码质量等
5. **问题识别**：识别潜在问题、改进机会和风险点
6. **报告输出**：输出分析报告，包括发现的问题、改进建议和架构图等

### 代码分析维度
- **架构分析**：分析系统架构、模块划分、组件关系、设计模式应用等
- **代码质量**：评估代码可读性、可维护性、可扩展性、代码规范遵循度等
- **依赖分析**：分析模块依赖关系、循环依赖、依赖复杂度等
- **性能分析**：识别性能瓶颈、资源消耗、算法复杂度等
- **安全分析**：识别安全漏洞、潜在风险、安全最佳实践遵循情况等

### 分析工具和方法
- **静态分析**：使用代码分析工具（linter、静态分析器）进行自动化分析
- **代码审查**：人工审查代码逻辑、设计模式和最佳实践
- **依赖图**：绘制依赖关系图，可视化模块间的关系
- **复杂度分析**：分析代码复杂度（圈复杂度、认知复杂度等）
- **代码度量**：收集代码度量指标（代码行数、测试覆盖率、技术债务等）

### 注意事项
- 分析要全面，不能只关注局部问题
- 要理解代码的业务逻辑和设计意图
- 识别问题时要有依据，不能主观臆断
- 分析报告要清晰、结构化，便于理解和行动
- 要区分问题的严重程度和优先级
- 分析结果要可验证、可追溯
