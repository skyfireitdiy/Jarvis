## 性能优化专项指导

### 核心目标

专注于识别和优化性能瓶颈，提升系统响应速度、吞吐量和资源利用率。

### 性能分析流程

1. **性能基准建立**：先测量当前性能指标（响应时间、吞吐量、资源使用率），建立基准
2. **瓶颈识别**：使用性能分析工具（profiler、性能监控）定位热点代码和瓶颈点
3. **优化方案设计**：基于瓶颈分析，设计针对性优化方案（算法优化、数据结构优化、并发优化、缓存策略等）
4. **优化实施**：按方案实施优化，保持功能正确性
5. **性能验证**：对比优化前后的性能指标，确保优化有效

### 性能优化策略

- **算法优化**：选择更高效的算法和数据结构，降低时间复杂度
- **并发优化**：合理使用多线程/协程，提升并发处理能力
- **缓存策略**：识别重复计算，引入缓存减少计算开销
- **资源优化**：减少内存分配、优化I/O操作、减少网络请求
- **数据库优化**：优化查询语句、添加索引、减少数据库访问次数

### 注意事项

- 性能优化前必须先建立性能基准，避免盲目优化
- 优先优化热点代码（遵循80/20原则）
- 优化后必须验证功能正确性，不能为了性能牺牲正确性
- 考虑可维护性，避免过度优化导致代码难以理解
- 记录优化前后的性能数据，便于后续评估和回退
