# 项目初始化综述规则

## 规则简介

本规则指导如何分析项目并生成结构化的项目综述文件，帮助开发者快速了解项目全貌。生成的综述文件将写入 `.jarvis/rule` 文件，作为后续开发的上下文参考。

## 你必须遵守的原则

### 1. 分析完整性原则

**要求说明：**

- **必须**：分析项目的目录结构，理解源码组织方式
- **必须**：识别项目的技术栈（编程语言、框架、依赖库）
- **必须**：理解项目的架构设计和核心模块
- **必须**：确认项目的构建、运行和测试方式
- **必须**：识别关键配置文件和环境变量
- **禁止**：跳过任何核心模块的分析
- **禁止**：臆测项目信息，必须基于实际文件内容

### 2. 综述结构化原则

**要求说明：**

- **必须**：使用 Markdown 格式组织综述内容
- **必须**：按章节组织（项目概述、技术栈、目录结构、核心模块、构建与运行、测试、关键配置）
- **必须**：提供具体的命令示例（构建、运行、测试）
- **必须**：说明模块间的依赖关系
- **禁止**：使用模糊的描述（如"合理的架构"、"常见的框架"）

## 你必须执行的操作

### 操作1：浏览项目结构

**执行步骤：**

1. 使用 `fd` 或 `ls` 浏览项目根目录结构
2. 识别主要目录的用途（src、tests、docs、config等）
3. 查看项目配置文件（pyproject.toml、package.json、Makefile等）

### 操作2：分析技术栈

**执行步骤：**

1. 读取依赖配置文件，识别核心依赖
2. 确定编程语言及版本要求
3. 识别使用的框架和工具链

### 操作3：理解核心模块

**执行步骤：**

1. 浏览源码目录，识别核心模块
2. 读取关键文件的接口定义和文档字符串
3. 分析模块间的导入和依赖关系

### 操作4：生成综述文件

**执行步骤：**

1. 按照结构化模板组织分析结果
2. 使用 `edit_file` 工具将综述写入 `.jarvis/rule` 文件
3. 确保内容准确、完整、可操作

### 综述文件模板

```markdown
# 项目综述

## 项目概述

[项目目标、定位、使用场景]

## 技术栈

- 编程语言：[语言及版本]
- 核心框架：[框架名称及版本]
- 关键依赖：[依赖列表]

## 目录结构

[目录树及说明]

## 核心模块

### [模块名]

- 职责：[模块职责]
- 关键文件：[文件列表]
- 依赖：[依赖的其他模块]

## 构建与运行

- 构建：`[构建命令]`
- 运行：`[运行命令]`
- 环境要求：[环境要求]

## 测试

- 测试框架：[框架名称]
- 运行测试：`[测试命令]`
- 测试目录：[测试文件位置]

## 关键配置

- 配置文件：[配置文件路径]
- 环境变量：[关键环境变量]
```
