# 项目初始化综述规则

## 规则简介

本规则指导如何分析项目并生成结构化的项目综述文件，帮助开发者快速了解项目全貌。生成的综述文件将写入 `.jarvis/rule.md` 文件，作为后续开发的上下文参考。

**核心原则**：综述必须**完备、准确、可操作**，覆盖项目全貌，不留盲区。如果综述文件已存在，必须先读取现有内容再进行增量更新，禁止直接覆盖。

## 你必须遵守的原则

### 1. 分析完备性原则

**要求说明：**

- **必须**：分析项目的目录结构，理解源码组织方式
- **必须**：识别项目的技术栈（编程语言、框架、依赖库及版本）
- **必须**：理解项目的架构设计和核心模块
- **必须**：确认项目的构建、运行和测试方式
- **必须**：识别关键配置文件和环境变量
- **必须**：统计代码规模（语言分布、文件数、代码行数）
- **必须**：识别项目的架构特点（设计模式、扩展机制、本地优先等）
- **必须**：梳理API与接口设计（公共API端点、协议、数据流）
- **必须**：识别项目的部署方式和运行环境要求
- **必须**：分析模块间的依赖关系和数据流向
- **禁止**：跳过任何核心模块的分析
- **禁止**：臆测项目信息，必须基于实际文件内容
- **禁止**：遗漏重要的配置文件或环境变量
- **禁止**：忽略测试和部署相关信息的收集

### 2. 综述结构化原则

**要求说明：**

- **必须**：使用 Markdown 格式组织综述内容
- **必须**：按章节组织（项目概述、技术栈、目录结构、核心模块、构建与运行、测试、关键配置、代码统计、架构特点、API与接口）
- **必须**：提供具体的命令示例（构建、运行、测试）
- **必须**：说明模块间的依赖关系
- **必须**：使用具体的版本号和数值，避免模糊描述
- **禁止**：使用模糊的描述（如"合理的架构"、"常见的框架"、"部分依赖"）

### 3. 增量更新原则

**要求说明：**

- **必须**：在写入综述文件前，先检查 `.jarvis/rule.md` 是否已存在
- **必须**：如果文件已存在，先使用 `read_code` 读取现有内容
- **必须**：基于现有内容进行增量更新，保留仍然有效的信息
- **必须**：标注更新时间和变更摘要（在文件顶部注明最后更新日期）
- **必须**：对比新旧信息，更新已过时的内容（如版本号、依赖变更）
- **禁止**：在已有综述文件时直接覆盖写入，丢弃已有分析成果
- **禁止**：忽略现有综述中的有效信息

## 你必须执行的操作

### 操作0：检查现有综述

**执行步骤：**

1. 检查 `{{ git_root_dir }}/.jarvis/rule.md` 文件是否存在
2. 如果存在，使用 `read_code` 读取现有综述的完整内容
3. 分析现有综述中哪些信息仍然有效、哪些需要更新
4. 记录需要新增的章节和需要更新的内容

### 操作1：浏览项目结构

**执行步骤：**

1. 使用 `fd` 或 `ls` 浏览项目根目录结构
2. 识别主要目录的用途（src、tests、docs、config等）
3. 查看项目配置文件（pyproject.toml、package.json、Makefile等）
4. 统计各目录的文件数量和代码行数（使用 `cloc` 或 `wc` 等工具）

### 操作2：分析技术栈

**执行步骤：**

1. 读取依赖配置文件，识别核心依赖及版本号
2. 确定编程语言及版本要求
3. 识别使用的框架和工具链
4. 区分核心依赖、开发依赖和可选依赖

### 操作3：理解核心模块

**执行步骤：**

1. 浏览源码目录，识别核心模块
2. 读取关键文件的接口定义和文档字符串
3. 分析模块间的导入和依赖关系
4. 识别每个模块的职责边界和对外接口

### 操作4：分析架构与接口

**执行步骤：**

1. 识别项目的整体架构模式（单体/微服务/插件化等）
2. 梳理核心数据流和组件间通信方式
3. 识别公共API端点和协议（HTTP/WebSocket/RPC等）
4. 分析扩展机制（插件系统、钩子、配置驱动等）

### 操作5：生成/更新综述文件

**执行步骤：**

1. 如果已有综述（操作0中读取），基于现有内容进行增量更新：
   - 保留仍然有效的章节和信息
   - 更新已过时的版本号、依赖列表等
   - 新增之前缺失的章节
   - 在文件顶部更新最后更新日期
2. 如果没有已有综述，按照结构化模板从零生成
3. 使用 `edit_file` 工具将综述写入 `{{ git_root_dir }}/.jarvis/rule.md` 文件
4. 确保内容准确、完整、可操作
