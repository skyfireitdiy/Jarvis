# 投资分析框架（价值投资通用版）

## 简介

整体分为**6大模块**，层层递进：
宏观行业 → 公司基本面 → 财务数据 → 估值定价 → 风险排查 → 操作策略

适合个股深度分析、选股复盘、研报撰写。

## 数据获取方式

分析所需数据统一通过 **search_web** 工具搜索互联网获取：

- 行业研报、政策动态、竞争格局等定性信息
- 公司财务数据、估值指标、历史分位
- 管理层背景、股权结构、公司公告
- 可比公司对标、行业均值
- 风险事件、监管政策、新闻舆情

**搜索建议**：

- 财务数据搜索：`"公司名" 年报 营收 ROE 毛利率`
- 估值数据搜索：`"公司名" PE PB 估值分位`
- 行业分析搜索：`"行业名" 市场规模 增速 竞争格局`
- 风险信息搜索：`"公司名" 风险 商誉减值 诉讼`

## 你必须遵守的原则

### 1. 分析顺序原则

- **必须**：按6大模块顺序逐层分析，不得跳过
- **必须**：每个模块得出结论后再进入下一模块
- **禁止**：先看估值再倒推逻辑（避免确认偏差）

### 2. 数据驱动原则

- **必须**：每个判断必须有数据支撑
- **必须**：区分事实与推测，推测需标注
- **禁止**：无数据支撑的主观臆断

### 3. 风险优先原则

- **必须**：风险排查模块不得省略
- **必须**：列出至少3个风险因素
- **禁止**：只讲利好不提风险

## 你必须执行的操作

### 模块一：行业与赛道分析（决定天花板）

#### 1. 行业景气度

- 行业生命周期：萌芽期/成长期/成熟期/衰退期
- 增长空间：未来3-5年复合增速、市场规模、渗透率
- 政策导向：扶持/中性/强监管

#### 2. 行业竞争格局

- 格局类型：完全竞争/寡头垄断/独家垄断
- 壁垒类型：技术专利、牌照、规模成本、渠道、品牌、网络效应
- 上下游议价能力：上游卡脖子风险、下游客户集中度

#### 3. 供需与周期性

- 周期属性：强周期/弱周期/成长赛道
- 产能状况：是否过剩、扩张周期、价格波动逻辑

**数据获取**：search_web 搜索行业研报、政策文件

### 模块二：公司核心竞争力（判断能不能长期赚钱）

#### 1. 商业模式本质

一句话说清：靠什么产品/服务、卖给谁、怎么赚钱、盈利是否可持续。

#### 2. 核心护城河（最重要）

- 品牌壁垒：消费者溢价意愿
- 技术壁垒：专利、独家工艺、研发迭代速度
- 成本壁垒：规模效应、产业链一体化
- 渠道壁垒：线下网点、经销商体系、大客户绑定
- 行政壁垒：特许经营牌照、准入门槛
- 网络效应：用户越多价值越高

#### 3. 管理层与治理

- 实控人、股权结构：质押率、减持记录
- 管理层履历：从业经验、是否深耕主业
- 分红政策：稳定性、回购情况
- 企业文化：激进扩张还是稳健保守

#### 4. 产品与市场地位

- 主营产品毛利率、市占率、行业排名
- 新品迭代能力、管线储备
- 客户结构：单一客户依赖、海外收入占比

**数据获取**：search_web 搜索公司公告、管理层信息、财务指标

### 模块三：财务报表量化分析（用数据验证）

#### （一）盈利能力

1. 营收：连续3-5年增速稳定性，是否靠并购堆砌
2. 归母净利润：关注扣非净利润
3. 关键指标：
   - 毛利率：反映议价权，逐年下滑需警惕
   - 净利率：真实赚钱能力
   - ROE：长期＞15%为优秀，拆解杜邦三因子

#### （二）盈利含金量

- 经营现金流净额：长期应大于净利润
- 应收账款周转率：回款速度
- 存货周转率：存货积压减值风险

#### （三）偿债与财务稳健性

- 资产负债率：轻资产＜60%，重资产可偏高
- 流动比率、速动比率：短期偿债能力
- 有息负债规模与利息覆盖倍数

#### （四）运营与扩张效率

- 固定资产周转率、资本开支合理性
- 费用管控：销售/管理/研发费用率

#### （五）排雷预警信号

大额商誉、其他应收款异常、大股东占用资金、
频繁商誉减值、净利润大增但现金流为负、毛利率持续下滑。

**数据获取**：search_web 搜索公司年报、财务指标、排雷信号

### 模块四：估值分析（决定买入贵不贵）

#### 1. 常用估值指标

- PE市盈率：适合盈利稳定的成熟企业，对比历史分位和行业均值
- PB市净率：适合强周期、金融、地产
- PEG：成长股使用，PEG＜1合理
- DCF现金流折现：内在价值测算
- 股息率：高股息价值股

#### 2. 横向+纵向对比

- 纵向：近10年估值百分位
- 横向：同行业可比公司对标

**数据获取**：search_web 搜索PE、PB、历史估值分位、可比公司数据

### 模块五：风险因素分析（决定底线）

#### 1. 经营风险

行业需求下滑、竞品冲击、技术替代、管理层变动、
产能过剩、原材料涨价

#### 2. 政策与外部风险

监管收紧、集采、反垄断、关税、地缘政治、海外制裁

#### 3. 财务风险

高负债偿债压力、商誉减值、坏账计提、
业绩变脸、现金流恶化

#### 4. 市场与估值风险

行业估值下杀、系统性熊市、流动性收紧

#### 5. 其他黑天鹅

诉讼纠纷、安全事故、产品质量问题、实控人风险

**数据获取**：search_web 搜索公司公告、行业新闻、监管政策

### 模块六：最终投资结论与操作计划

1. **核心逻辑**：看好的3个核心理由
2. **主要担忧**：最大2-3个风险
3. **合理估值区间**：目标价、安全边际买入价
4. **仓位配置**：止损条件、持有周期

## 报告输出规范

### 1. 必须写入文件

- **必须**：分析完成后，将完整投资分析报告写入本地 Markdown 文件
- **必须**：文件命名格式为 `投资分析_{公司名}_{日期}.md`，例如 `投资分析_贵州茅台_20250101.md`
- **必须**：文件保存在当前工作目录下

### 2. 分段写入（重要）

投资分析报告内容较长，**必须分段写入文件**，避免单次写入内容过长导致截断或失败：

1. **第一段**：写入报告头部（标题、分析日期、公司概况）和模块一（行业与赛道分析）
2. **第二段**：追加写入模块二（公司核心竞争力）和模块三（财务报表量化分析）
3. **第三段**：追加写入模块四（估值分析）和模块五（风险因素分析）
4. **第四段**：追加写入模块六（最终投资结论与操作计划）和检查清单

**操作方式**：

- 第一段使用 `edit_file` 创建文件（search 为空字符串）
- 后续段使用 `edit_file` 追加内容（在文件末尾的 `<!-- APPEND_MARKER -->` 处插入）
- 每段写入后确认成功，再写下一段

### 3. 报告格式要求

- 使用 Markdown 格式，层次清晰
- 每个模块使用二级标题，子项使用三级/四级标题
- 数据表格使用 Markdown 表格语法
- 关键结论加粗标注
- 风险因素使用 ⚠️ 标记
