# 系统环境画像规则

## 规则简介

本规则用于分析当前系统环境的硬件、软件、资源等信息，生成本机环境画像报告。报告将帮助用户识别当前系统适合从事何种类型的工作（如 AI 训练、Web 开发、编译构建等）。

## 你必须遵守的原则

### 1. 报告路径规范

**要求说明：**

- **必须**：报告文件保存在 `{{ jarvis_data_dir }}/reports/` 目录下
- **必须**：报告文件名固定为 `system_profile.md`
- **必须**：每次运行时覆盖旧报告，确保报告内容为最新
- **禁止**：将报告保存在其他目录
- **禁止**：使用动态文件名（如包含时间戳）

### 2. 信息收集原则

**要求说明：**

- **必须**：收集真实、准确的系统信息
- **必须**：使用标准系统命令获取信息（如 `uname`, `free`, `df`, `lscpu` 等）
- **必须**：对收集的信息进行结构化整理
- **禁止**：虚构或猜测系统信息
- **禁止**：收集敏感信息（如密码、密钥等）

### 3. 报告格式规范

**要求说明：**

- **必须**：使用 Markdown 格式生成报告
- **必须**：包含报告生成时间
- **必须**：使用清晰的标题和分节
- **必须**：对关键指标进行总结和评估

## 你必须执行的操作

### 操作1：收集系统信息

**执行步骤：**

1. 获取操作系统信息：`uname -a`, `cat /etc/os-release`
2. 获取 CPU 信息：`lscpu` 或 `cat /proc/cpuinfo`
3. 获取内存信息：`free -h`
4. 获取磁盘信息：`df -h`
5. 获取 GPU 信息（如有）：`nvidia-smi` 或 `lspci | grep -i vga`
6. 获取网络信息：`ip addr` 或 `ifconfig`
7. 获取已安装的开发工具：`python3 --version`, `node --version`, `docker --version` 等

### 操作2：生成画像报告

**执行步骤：**

1. 确保 `{{ jarvis_data_dir }}/reports/` 目录存在，不存在则创建
2. 将收集的信息整理为 Markdown 格式

### 操作3：工作类型评估

**评估维度：**

1. **AI/ML 训练**：评估 GPU 型号、显存大小、CUDA 支持
2. **Web 开发**：评估 Node.js、Python、Docker 等环境
3. **编译构建**：评估 CPU 核心数、内存大小、磁盘速度
4. **数据处理**：评估内存大小、磁盘空间、Python/R 环境
5. **容器化部署**：评估 Docker、Kubernetes 等环境
