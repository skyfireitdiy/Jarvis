# 节点描述生成规则

## 规则简介

本规则用于将用户提供的节点描述信息写入 `~/.jarvis/gateway/node_description.md` 文件。该文件会被系统信息模块自动读取，并在 Child-Master 心跳中上报到 Master 节点，供 `gateway_manager` 工具的 `list_nodes` 操作返回。

## 你必须遵守的原则

### 1. 描述内容来源

- **必须**：描述内容由用户提供，不得自行编造
- **必须**：用户可以提供任意格式的描述文本，规则负责写入文件
- **禁止**：在用户未提供内容时生成默认描述

### 2. 文件路径

- **必须**：写入路径为 `~/.jarvis/gateway/node_description.md`
- **必须**：如果目录不存在，先创建目录再写入文件
- **禁止**：写入其他路径

### 3. 大小限制

- **必须**：描述文件大小不得超过 10KB（10240 字节）
- **必须**：如果用户提供的描述超过 10KB，截断内容并告知用户
- **建议**：描述内容控制在 1-2KB 以内，保持简洁实用

### 4. 文件格式

- **必须**：使用 Markdown 格式
- **必须**：使用 UTF-8 编码
- **建议**：包含节点名称、用途、硬件配置、负责人等关键信息

## 你必须执行的操作

### 操作1：收集描述内容

1. 询问用户提供节点描述内容
2. 确认用户提供的描述内容完整

### 操作2：写入描述文件

1. 检查目录 `~/.jarvis/gateway/` 是否存在，不存在则创建
2. 检查描述内容大小，超过 10KB 则截断并告知用户
3. 将描述内容写入 `~/.jarvis/gateway/node_description.md`
4. 确认写入成功

### 操作3：验证

1. 读取写入的文件内容，确认与预期一致
2. 告知用户描述将在下次心跳时自动上报到 Master 节点

## 描述内容建议模板

```markdown
# 节点名称

## 基本信息

- **节点ID**: xxx
- **用途**: xxx
- **负责人**: xxx

## 硬件配置

- **CPU**: xxx
- **内存**: xxx GB
- **磁盘**: xxx GB

## 备注

xxx
```

## 检查清单

在完成任务后，你必须确认：

- [ ] 描述文件已写入 `~/.jarvis/gateway/node_description.md`
- [ ] 文件大小未超过 10KB
- [ ] 文件编码为 UTF-8
- [ ] 文件内容与用户提供的描述一致
- [ ] 已告知用户描述将在心跳中自动上报
