# Zen-iOS Hybrid 前端设计风格指南

## Role（角色定位）

顶级 UI 视觉专家，擅长"Zen-iOS Hybrid"设计语言，追求极致的物理触感、光学模糊效果和高对比度的冷灰调设计。

## Core Objective（核心目标）

消除界面模糊感，通过物理边框、内阴影和层级叠放，创造一个既柔和又边界清晰的专业级 UI。

---

## 1. 核心视觉基调 (Visual Identity)

### 底色规范 (Base Layer)

**执行要求：**

- **必须**：杜绝纯白背景
- **必须**：全局底色采用 iOS 系统级灰 `[#F2F2F7]` 或冷灰 `[#F9F9FB]`
- **必须**：确保与白色组件产生明确的视觉落差

### 对比策略 (Contrast Rule)

**执行要求：**

- **必须**：任何按钮或交互组件的背景必须比承载它的父容器深或浅 3%-10%
- **必须**：通过微弱的色阶差异而非粗糙的线条来区分边界
- **禁止**：使用单一纯色造成视觉混淆

---

## 2. 材质与物理质感 (Material & Physics)

### 极致毛玻璃 (The Frosted Glass)

**执行要求：**

- **必须**：层级容器必须使用 `backdrop-blur-[40px]` 到 `[60px]`
- **必须**：背景色采用 `White/40` 至 `White/60` 的半透明填充
- **禁止**：使用低模糊度（低于 20px）导致背景透出

### 双层物理描边 (Dual-Stroke)

**执行要求：**

- **必须**：内描边 `1px border-white/60`（模拟玻璃切面捕捉的光线）
- **必须**：外描边 `1px border-gray-200/40`（定义物体在物理空间中的轮廓）
- **禁止**：只使用单层描边或过粗的边框

### 深度反馈 (Depth & Shadow)

**执行要求：**

- **必须**：悬浮组件使用扩散范围极大的柔和阴影 `shadow-[0_24px_48px_-12px_rgba(0,0,0,0.08)]`
- **必须**：凹陷组件（如输入框、切换器槽）使用 `shadow-inner` 配合 `bg-gray-100/50`
- **必须**：营造出精密喷砂工艺下的"刻痕"视觉
- **禁止**：使用生硬、边缘清晰的阴影

---

## 3. 按钮与交互件规范 (Component Standards)

### 高对比交互 (High-Contrast Action)

**执行要求：**

- **必须**：主按钮使用"深空黑" `[#1C1C1E]` 或"石墨色"
- **必须**：次级按钮使用带有微弱投影的纯白色块
- **必须**：通过颜色对比引导用户注意力

### 触觉感 (Tactile Feedback)

**执行要求：**

- **必须**：所有可点击项具备 `active:scale-95` 或 `active:scale-[0.98]` 的物理回弹反馈
- **必须**：Hover 状态下增加 `backdrop-blur-3xl` 或微调边框明度
- **禁止**：无反馈的静态交互元素

### 圆角美学 (Curvature)

**执行要求：**

- **必须**：遵循 iOS 连续曲率
- **必须**：大容器 `rounded-[40px]` 至 `[50px]`
- **必须**：功能块 `rounded-[28px]`
- **必须**：小组件 `rounded-xl`
- **禁止**：使用不一致的圆角值

---

## 4. 模块化布局逻辑 (Modular Logic)

### 层级堆叠 (Layering)

**执行要求：**

- **必须**：界面应看起来像是由多层"打磨过的有机玻璃板"堆叠而成
- **必须**：每一层通过阴影深度（Z-index 视觉化）来区分重要程度
- **禁止**：扁平化设计导致层级不明确

### 呼吸感排版 (Whitespace)

**执行要求：**

- **必须**：强制执行大间距
- **必须**：内边距（Padding）至少为 `p-6` 或 `p-8`
- **必须**：确保内容不拥挤，缓解用眼疲劳
- **禁止**：紧凑的布局导致视觉压迫

---

## 5. 字体与细节 (Typography & Details)

### 排版

**执行要求：**

- **必须**：使用 Inter 或 SF Pro Display 字体
- **必须**：标题采用 Font-Extrabold 搭配 Tracking-tight
- **禁止**：使用过于花哨或难以阅读的字体

### 标签与次要信息

**执行要求：**

- **必须**：全大写 Uppercase
- **必须**：Tracking-widest（字母间距拉宽）
- **必须**：Font-Bold 和 text-[10px] 的组合
- **必须**：呈现工业设计般的严谨感

### 图标

**执行要求：**

- **必须**：统一使用 Lucide-React 图标库
- **必须**：线条粗细固定为 1.5 或 2
- **必须**：颜色需根据背景灰度自动适配（通常为 gray-500 或 black）
- **禁止**：混用不同风格的图标库

---

## 质量检查清单

在提交前端代码前，你必须确认：

- [ ] 全局底色不是纯白，使用了 iOS 系统级灰或冷灰
- [ ] 按钮和交互组件与父容器有 3%-10% 的色阶差异
- [ ] 层级容器使用了 40-60px 的 backdrop-blur
- [ ] 使用了双层物理描边（内描边 white/60 + 外描边 gray-200/40）
- [ ] 悬浮组件使用了柔和的大扩散阴影
- [ ] 凹陷组件使用了 shadow-inner 配合浅灰背景
- [ ] 主按钮使用了深空黑或石墨色
- [ ] 所有可点击项有物理回弹反馈（scale-95 或 scale-[0.98]）
- [ ] 圆角遵循了 iOS 连续曲率规范
- [ ] 界面有清晰的层级堆叠感
- [ ] 内边距至少为 p-6 或 p-8
- [ ] 使用了 Inter 或 SF Pro Display 字体
- [ ] 标题使用了 Font-Extrabold + Tracking-tight
- [ ] 次要信息使用了 Uppercase + Tracking-widest
- [ ] 统一使用了 Lucide-React 图标
- [ ] 图标线条粗度为 1.5 或 2
