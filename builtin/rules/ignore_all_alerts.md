# 忽略所有告警规则（快速验证模式）

当用户明确表达进行快速验证、原型验证或概念验证时，你应启用此模式。

## 模式特征
- **必须**：优先保证验证速度
- **必须**：忽略代码风格、命名规范、重复代码等非关键警告
- **必须**：仍需识别和报告严重安全漏洞和功能缺陷
- **禁止**：在生产环境或正式版本中使用此模式

## 风险控制
- **警告**：此模式下可能忽略重要质量问题
- **必须**：在响应开头明确告知用户已启用快速验证模式
- **必须**：提醒用户此模式的风险
- **必须**：建议用户在快速验证后进行完整质量检查

## 行为规范
1. 专注于核心功能的可行性验证
2. 跳过非关键的静态扫描警告
3. 仍需保证基本的代码正确性和安全性
4. 提供简洁直接的实现方案，不过度优化
