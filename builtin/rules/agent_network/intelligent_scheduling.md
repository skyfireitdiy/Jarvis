# 智能调度规则

## 架构感知

### 1. 网络拓扑发现
**发现流程：**
- **必须**：自动发现网络中的所有节点和Agent
- **必须**：维护网络拓扑图
- **必须**：识别节点间的连接关系
- **禁止**：忽视网络拓扑变化

### 2. 能力评估
**评估内容：**
- **必须**：评估每个节点的计算能力
- **必须**：评估每个Agent的能力标签
- **必须**：评估节点的资源使用情况
- **禁止**：忽视Agent能力差异

### 3. 负载监控
**监控内容：**
- **必须**：监控每个节点的负载情况
- **必须**：监控每个Agent的任务队列
- **必须**：预测负载趋势
- **禁止**：忽视负载变化

## 任务分析

### 1. 任务分解
**分解原则：**
- **必须**：将复杂任务分解为子任务
- **必须**：识别子任务间的依赖关系
- **必须**：评估每个子任务的资源需求
- **禁止**：任务分解不合理

### 2. 需求识别
**识别内容：**
- **必须**：识别任务所需的能力标签
- **必须**：识别任务的时间要求
- **必须**：识别任务的质量要求
- **禁止**：忽视任务需求

### 3. 复杂度评估
**评估方法：**
- **必须**：评估任务的技术复杂度
- **必须**：评估任务的协调复杂度
- **必须**：评估任务的风险复杂度
- **禁止**：复杂度评估不准确

## Agent匹配

### 1. 能力匹配
**匹配原则：**
- **必须**：匹配任务所需的能力标签
- **必须**：考虑Agent的能力熟练度
- **必须**：考虑Agent的学习能力
- **禁止**：能力不匹配强行分配

### 2. 资源匹配
**匹配原则：**
- **必须**：匹配节点的资源可用性
- **必须**：考虑资源使用效率
- **必须**：预留应急资源
- **禁止**：资源不足强行分配

### 3. 负载匹配
**匹配原则：**
- **必须**：考虑Agent的当前负载
- **必须**：预测任务执行时间
- **必须**：避免过载
- **禁止**：让Agent过载

## 资源优化

### 1. 负载均衡
**均衡策略：**
- **必须**：监控所有Agent的负载情况
- **必须**：动态调整任务分配
- **必须**：在Agent间迁移任务
- **禁止**：负载不均衡

### 2. 故障恢复
**恢复策略：**
- **必须**：检测Agent故障
- **必须**：自动重新分配任务
- **必须**：创建新Agent恢复服务
- **禁止**：故障后不采取行动

### 3. 性能优化
**优化策略：**
- **必须**：分析性能瓶颈
- **必须**：优化任务分配策略
- **必须**：调整网络拓扑
- **禁止**：性能问题不解决

## 检查清单

在执行智能调度后，你必须确认：

- [ ] 网络拓扑已发现并维护
- [ ] Agent能力已评估并更新
- [ ] 负载情况已监控并分析
- [ ] 任务已合理分解
- [ ] Agent已合理匹配
- [ ] 资源已优化分配
