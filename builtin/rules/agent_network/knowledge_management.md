# 知识管理规则

## 核心概念澄清

**Agent（智能体）**：

- 独立的智能实体，有自己的工作目录、模型、任务
- 使用 `gateway_manager` 工具的 `create_agent` 操作创建
- agent_type 必须是 `'agent'` 或 `'codeagent'`
- 示例：知识库Agent、监控Agent、调度Agent

**Tool（工具）**：

- 供Agent调用的功能模块
- 使用 `meta_agent` 工具创建（不在此规则范围内）
- 示例：memory工具、execute_script工具、read_code工具

**常见错误警示**：

- ❌ 混淆Agent和工具的概念
- ❌ 使用错误的agent_type（如'knowledge'、'monitor'等）
- ❌ 尝试用工具创建方式创建Agent

## 知识库Agent实现指南

### 知识库Agent职责

知识库Agent是负责管理Agent网络知识的核心角色，具体职责包括：

1. **知识存储**：存储所有Agent贡献的经验、最佳实践、解决方案
2. **知识检索**：提供高效的知识检索服务，支持相似度匹配和质量评分
3. **知识进化**：整理、分类、优化知识库，实现知识的持续进化
4. **知识传播**：确保知识及时到达需要的Agent，支持主动推送和被动检索

### 创建知识库Agent示例

```json
{
  "name": "gateway_manager",
  "arguments": {
    "action": "create_agent",
    "agent_type": "codeagent",
    "agent_name": "knowledge_base_agent",
    "working_dir": "/path/to/knowledge/work",
    "no_interaction_mode": false
  }
}
```

**关键注意事项**：

- `agent_type` 必须是 `codeagent`（不是 `agent`），知识库Agent需要代码能力来管理文件
- `agent_name` 必须显式指定，否则Agent的name字段为null
- `no_interaction_mode` 必须为 `false`（知识库Agent需常驻系统，不能设为无人值守，否则运行一轮后自动退出）
- 不要设置 `task` 参数，交互式Agent不需要初始任务

### 知识库路径配置

知识库由知识库Agent管理，存储在项目的 `knowledge_base/` 目录下，包含以下子目录：

- `experiences/`：经验分享
- `best_practices/`：最佳实践
- `solutions/`：解决方案
- `patterns/`：知识模式
- `collective_wisdom/`：集体智慧
- `index/`：索引文件

**⚠️ 强制规则：新知识必须保存到知识库Agent**

只要有新的知识产生（包括但不限于：Bug修复经验、最佳实践、架构决策、配置信息、解决方案等），**必须**将知识发送给知识库Agent保存。

**执行方式**：
- 使用 `gateway_manager send_to_agent` 将知识发送给知识库Agent
- 知识库Agent会自动将知识保存到 `knowledge_base/` 目录并更新索引

**原则**：
1. 知识库Agent是自进化网络的知识管理核心，所有知识应归它管理
2. `memory` 工具仅用于保存Agent运行上下文（如当前任务状态、临时信息），**不用于保存知识**
3. 无需用户提醒，主动将新知识发送给知识库Agent
4. 每次产生新知识时，在任务完成后主动发送给知识库Agent

### 知识条目格式规范

每个知识条目应包含：

- **任务类型**：标识知识适用的任务类型
- **问题描述**：清晰描述问题场景
- **解决方案**：详细的解决步骤
- **适用场景**：标注适用场景和限制条件
- **质量评分**：根据使用效果动态评分
- **时间戳**：记录创建和更新时间

## 知识贡献

### 1. 经验分享

**分享原则：**

- **必须**：任务完成后将执行经验存入知识库
- **必须**：经验包含任务类型、执行过程、遇到的问题
- **必须**：记录解决方案和最终结果
- **禁止**：只执行任务不分享经验

### 2. 最佳实践

**实践记录：**

- **必须**：记录成功的解决方案和方法
- **必须**：标注适用场景和限制条件
- **必须**：提供具体的实施步骤
- **禁止**：忽视成功经验的记录

### 3. 解决方案

**方案管理：**

- **必须**：记录问题的完整解决方案
- **必须**：包含问题描述、分析过程、解决步骤
- **必须**：标注方案的有效性和适用范围
- **禁止**：只记录结果不记录过程

## 知识检索

### 1. 相似度匹配

**匹配算法：**

- **必须**：基于任务类型和需求进行匹配
- **必须**：考虑知识的相关性和时效性
- **必须**：支持多维度相似度计算
- **禁止**：简单关键词匹配

### 2. 质量评分

**评分标准：**

- **必须**：根据知识的使用效果评分
- **必须**：考虑知识的完整性和准确性
- **必须**：定期更新评分结果
- **禁止**：忽视知识质量评估

### 3. 知识组合

**组合策略：**

- **必须**：支持多个知识片段的组合应用
- **必须**：识别知识间的互补关系
- **必须**：生成新的综合解决方案
- **禁止**：孤立应用单个知识

## 知识进化

### 1. 模式识别

**识别方法：**

- **必须**：分析知识库中的成功模式
- **必须**：识别通用解决方案和最佳实践
- **必须**：形成可复用的知识模板
- **禁止**：忽视模式识别和总结

### 2. 知识整合

**整合策略：**

- **必须**：合并相似的知识片段
- **必须**：消除重复和矛盾的内容
- **必须**：形成结构化的知识体系
- **禁止**：知识碎片化不整合

### 3. 知识淘汰

**淘汰标准：**

- **必须**：识别过时和无效的知识
- **必须**：根据使用频率和效果淘汰
- **必须**：保留高质量和高相关性的知识
- **禁止**：知识库无限膨胀

## 知识共享

### 1. 协作学习

**学习机制：**

- **必须**：Agent间相互学习成功经验
- **必须**：共同分析失败案例
- **必须**：形成集体学习氛围
- **禁止**：Agent间知识隔离

### 2. 集体智慧

**智慧形成：**

- **必须**：整合所有Agent的知识和经验
- **必须**：形成网络级的集体智慧
- **必须**：支持智慧的传承和发展
- **禁止**：知识只停留在个体层面

### 3. 知识传播

**传播机制：**

- **必须**：建立高效的知识传播渠道
- **必须**：确保知识及时到达需要的Agent
- **必须**：支持知识的主动推送和被动检索
- **禁止**：知识传播不畅
