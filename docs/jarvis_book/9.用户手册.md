# 用户手册

## 1. 这份手册适合谁

这份手册面向直接使用 Jarvis 的用户，重点回答三个问题：

- 我在什么场景下适合用 Jarvis
- 我应该用哪个命令或界面来完成任务
- 执行之后我能得到什么结果

如果你更关心快速上手、常见命令和实际效果，这份手册就是给你准备的。

---

## 2. 快速开始

### 2.1 安装后先做什么

首次安装完成后，建议按下面顺序开始：

1. 使用 `jqc` 快速配置大模型平台
2. 使用 `jvs` 体验通用 Agent
3. 使用 `jca` 体验代码任务
4. 如需浏览器操作，再启动 `jwg serve`

### 2.2 常用命令速览

| 命令                       | 快捷方式    | 用途                                   |
| -------------------------- | ----------- | -------------------------------------- |
| `jarvis`                   | `jvs`       | 通用 AI 代理，适合分析、规划、日常任务 |
| `jarvis-code-agent`        | `jca`       | 代码分析、修改、生成、审查             |
| `jarvis-quick-config`      | `jqc`       | 快速配置模型平台                       |
| `jarvis-platform-manager`  | `jpm`       | 管理模型与模型组                       |
| `jarvis-web-gateway serve` | `jwg serve` | 启动 Web Gateway 服务                  |
| `jarvis-tool`              | `jt`        | 查看和调用工具                         |
| `jarvis-git-commit`        | `jgc`       | 自动生成 Git 提交信息                  |
| `jarvis-memory-organizer`  | `jmo`       | 整理、导入、导出记忆                   |
| `jarvis-methodology`       | `jm`        | 导入、导出、提取方法论                 |
| `jarvis-smart-shell`       | `jss`       | 自然语言生成 Shell 命令                |
| `jarvis-sec`               | `jsec`      | 安全扫描与安全问题分析                 |
| `jarvis-c2rust`            | `jc2r`      | C→Rust 迁移工具                        |

### 2.3 三个最常见的起步示例

```bash
# 1) 快速配置模型平台
jqc

# 2) 让通用 Agent 帮你分析任务
jvs -T "分析当前项目结构并给出优化建议"

# 3) 让 CodeAgent 帮你修改代码
jca -T "为 user 模块增加忘记密码功能"
```

你会得到的效果：

- 能够快速接入自己的模型平台
- 能直接把自然语言任务交给 Jarvis 执行
- 能在代码场景下获得更完整的自动化支持

## 3. 通用 Agent：适合日常任务、分析任务和交互式协作

### 3.1 什么时候用

适合下面这些场景：

- 分析一个项目、文档或需求
- 规划一个任务的执行步骤
- 让 AI 帮你整理思路、生成方案
- 希望边对话边执行工具
- 需要在交互过程中人工介入和修正方向

### 3.2 怎么用

最常用入口：

```bash
jvs
```

如果你希望直接执行一条任务：

```bash
jvs -T "帮我梳理这个仓库的模块职责"
```

如果你希望使用非交互方式运行：

```bash
jvs -T "总结 README 的核心信息"
```

### 3.3 你可以期待什么效果

使用通用 Agent 时，你通常会得到这些体验：

- 直接通过自然语言描述任务
- 在需要时调用工具完成文件读取、命令执行、网页读取等操作
- 支持会话保存与恢复，方便长任务继续做
- 支持规则、方法论、记忆等能力，让回答更贴合当前任务
- 支持人工中断、确认执行、补充要求

### 3.4 常见使用方式

#### 方式一：交互式执行任务

适合任务还不够明确、需要边聊边收敛需求的时候。

使用方法：

```bash
jvs
```

效果：

- 你可以持续追加要求
- 可以让系统逐步分析、规划和执行
- 适合文档、研究、分析类工作

#### 方式二：一次性执行单条任务

适合任务目标已经明确的时候。

使用方法：

```bash
jvs -T "为当前项目生成测试补充建议"
```

效果：

- 启动更快
- 更适合脚本化或批处理场景
- 输出更聚焦当前目标

#### 方式三：使用任务派发器快速发起任务

适合希望快速把任务发到独立 tmux 面板中执行的场景。

使用方法：

```bash
jvsd "分析当前仓库并输出模块说明"
```

或者直接输入：

```bash
jvsd
```

效果：

- 支持单行任务快速派发
- 支持多行任务录入后自动派发
- 适合把通用任务和当前终端工作分离开

#### 方式四：恢复历史会话

适合昨天没做完、今天继续接着做。

效果：

- 不必重新解释全部背景
- 更适合长周期任务
- 对连续协作更友好
- 恢复前会检查 Git commit 是否一致，并在需要时提示你 reset 到旧 commit 或继续恢复

### 3.5 高级交互技巧

#### 内置输入标记

如果你经常与通用 Agent 长时间协作，可以使用内置标记来更快控制会话行为。

常见用途包括：

- `'<Summary>'`：立即生成阶段总结并压缩上下文
- `'<Clear>'`：清空当前历史
- `'<ToolUsage>'`：查看工具使用说明
- `'<ReloadConfig>'`：重新加载配置
- `'<ListRule>'`：查看规则及激活状态
- `'<Pin>'`：固定一段长期有效的补充要求

使用效果：

- 更适合长任务管理
- 能减少重复输入
- 在复杂任务中更容易维持上下文稳定

#### 文件引用输入

如果你希望 Agent 在理解任务时自动带上文件结构信息，可以在输入中直接引用文件路径。

使用示例：

```text
请分析 'README.md' 和 'src/jarvis/jarvis_agent/jarvis.py'
```

使用效果：

- Agent 会更容易抓住你关心的文件
- 在代码、文档、配置混合分析时更高效

#### 直接执行 Shell 脚本

在对话里，可以通过以 `!` 开头的方式直接执行脚本，并决定是否把结果回传给 Agent。

使用示例：

```text
!pwd
!rg "TODO" .
```

使用效果：

- 适合边查边问、边执行边分析
- 可以把命令执行结果回灌到后续对话中

### 3.6 如果你常用通用 Agent，建议关注这些能力

- 自动加载规则，让任务执行方式更稳定
- 方法论复用，减少重复摸索
- 会话保存与恢复，适合长任务
- 工具执行前确认，适合高风险操作
- 任务结束后的总结与记忆沉淀，适合长期使用
- 任务派发器、多行输入、文件引用和 `!Shell` 能明显提升复杂任务操作效率

## 4. CodeAgent：适合代码分析、修改、生成和审查

### 4.1 什么时候用

如果你的任务与代码直接相关，优先使用 `jca`。典型场景包括：

- 阅读和分析代码
- 增加功能
- 修复 Bug
- 修改配置或测试
- 执行代码审查
- 在改完代码后自动进行校验

### 4.2 怎么用

最常见的方式：

```bash
jca -T "修复登录流程中的空指针问题"
```

也可以用于分析类任务：

```bash
jca -T "分析 payment 模块的调用链和风险点"
```

### 4.3 你可以期待什么效果

CodeAgent 相比通用 Agent，更适合真正动代码的任务。你通常会获得：

- 自动读取相关代码上下文
- 自动分析影响范围
- 自动展示代码差异
- 在修改后执行构建验证或基础检查
- 自动进行静态检查、格式化或后处理
- 自动生成检查点提交，方便回退和追踪

### 4.4 它最适合解决的问题

#### 场景一：做小到中等规模的代码修改

使用方法：

```bash
jca -T "给用户服务增加邮箱格式校验"
```

效果：

- 自动定位相关代码
- 自动修改对应文件
- 输出修改结果和差异预览

#### 场景二：分析复杂代码并给出修改建议

使用方法：

```bash
jca -T "分析缓存模块为什么会出现重复写入"
```

效果：

- 帮你理解代码结构
- 定位关键调用链
- 产出风险点和修改建议

#### 场景三：让修改过程更安全

效果：

- 修改后会进行构建验证或检查
- 会显示 diff 和影响分析
- 在高风险改动时给出额外提醒

### 4.5 常见增强用法

#### 使用任务派发器启动代码任务

适合需要把代码任务派发到 tmux 中独立执行的场景。

使用方法：

```bash
jcad "修复订单模块中的空值判断问题"
```

或者直接运行：

```bash
jcad
```

效果：

- 支持单行任务快速派发
- 支持多行代码需求录入
- 更适合并行处理多个代码任务

#### 从任务文件读取需求

适合需求较长、需要保存任务说明的场景。

效果：

- 更适合复杂代码任务
- 方便复用或归档需求说明

#### 使用 worktree 和自动审查循环

适合希望把代码任务与主工作区隔离，或希望连续执行代码审查与修复的场景。

使用效果：

- 能减少主工作区污染
- 更适合高风险代码修改
- 更适合需要多轮修复的任务

### 4.6 什么时候优先选 CodeAgent 而不是通用 Agent

优先选 `jca` 的情况：

- 目标是改代码，而不是只讨论方案
- 你希望自动运行构建、检查、格式化
- 你希望自动生成代码变更提交
- 你需要更稳定的代码上下文推荐能力

## 5. 平台与配置：先把模型接通，再把使用体验调顺手

### 5.1 快速配置模型平台：`jqc`

#### 适用场景

- 第一次安装 Jarvis
- 想快速接入 OpenAI 或 Claude 平台
- 想快速选择模型并保存到配置文件

#### 使用方法

```bash
jqc
```

执行过程中，你会依次完成：

- 选择平台类型
- 填写 API 地址与密钥
- 拉取可用模型列表
- 选择 normal / smart / cheap 模型
- 测试模型连通性
- 保存配置

#### 使用效果

- 快速完成可用模型接入
- 减少手动编辑配置文件的成本
- 适合首次安装后的第一步操作

### 5.2 管理模型和模型组：`jpm`

#### 适用场景

- 已经接入多个模型
- 需要查看、增删、切换模型组
- 需要确认当前平台和模型配置

#### 常见命令

```bash
# 查看当前平台与模型信息
jpm info

# 查看 LLM 配置列表
jpm manage llm list

# 查看单个 LLM 配置
jpm manage llm show openai

# 添加或更新 LLM 配置
jpm manage llm add my_model
jpm manage llm update my_model

# 查看模型组
jpm manage group list
jpm manage group show default

# 切换当前模型组
jpm manage group set default

# 直接与当前模型聊天
jpm chat

# 启动 OpenAI 兼容服务
jpm service --host 127.0.0.1 --port 8000

# 按角色开始对话
jpm role
```

#### 使用效果

- 你的模型配置会更清晰
- 多模型切换会更方便
- 更适合长期使用和团队内部统一配置
- 直接从命令层面就能完成查看、管理、切换、聊天和服务启动

### 5.3 启动配置界面：`jcfg`

#### 适用场景

- 不想手动改 YAML
- 希望通过页面方式完成配置
- 想更直观看到配置结构

#### 使用方法

```bash
jcfg
```

#### 使用效果

- 通过 Web 页面管理配置
- 更适合首次配置或配置项较多的情况
- 降低手动编辑配置文件出错的概率

### 5.4 这一类功能能帮你解决什么

- 模型接入慢：用 `jqc`
- 模型配置杂乱：用 `jpm`
- 配置文件不想手改：用 `jcfg`

## 6. Web 界面与网关：适合希望通过浏览器操作 Jarvis 的用户

### 6.1 什么时候用

适合下面这些场景：

- 不想只用终端
- 需要同时查看消息、文件、终端、编辑器
- 想更方便地管理多个 Agent
- 想通过浏览器进行人工介入

### 6.2 怎么启动

```bash
jwg serve
```

如果你想给访问加密码：

```bash
jwg serve --gateway-password your-password
```

### 6.3 连接后可以做什么

Web 界面支持这些面向用户的操作：

- 创建、切换、复制、重命名和删除 Agent
- 浏览 Agent 工作目录的文件树
- 打开文件到编辑器并保存修改
- 创建、切换和关闭终端
- 向 Agent 发送消息
- 在执行过程中人工介入
- 复制消息内容
- 恢复会话
- 清理历史记录
- 查看和修改连接设置
- 使用输入缓冲区管理多段输入内容

### 6.4 使用效果

如果你更喜欢图形化操作，Web 界面会明显提升使用体验：

- 信息更集中，聊天、终端、编辑器在同一个界面中完成
- 多 Agent 管理更直观
- 更适合演示、多人协作或复杂任务观察

### 7.1 工具系统：`jt`

#### 适用场景

- 想知道当前有哪些工具可用
- 想查看某个工具需要什么参数
- 想直接从命令行调用工具

```bash

jt list
```

#### 使用效果

- 更清楚 Jarvis 能直接做哪些事情

- 适合高级用户或自动化集成场景

### 7.2 自动生成 Git 提交信息：`jgc`

#### 适用场景

- 改完代码后不想手写提交说明
- 希望提交信息更规范
- 希望统一团队提交风格

#### 使用效果

- 根据当前代码变更自动生成提交信息
- 可配合前缀或后缀规范化提交内容

### 7.3 记忆整理：`jmo`

```bash
jmo organize --type project_long_term
```

#### 使用效果

- 让长期记忆更整洁

### 7.4 方法论管理：`jm`

```bash

```

#### 使用效果

- 把“经验”变成可复用资产
- 适合团队沉淀标准做法

### 7.5 智能 Shell：`jss`

### 7.6 规则索引：`jri`

#### 适用场景

- 想先了解 Jarvis 当前有哪些规则可用
- 想查看规则目录，而不是逐个猜规则名
- 想用 JSON 或原始格式导出规则索引

#### 常见用法

```bash

jri show --raw
```

#### 使用效果

- 能更快发现可复用规则
- 在规则驱动任务中更容易选对入口
- 适合经常使用规则和内置专家的用户

## 8. 专项能力：面向特定专业场景

### 8.1 安全扫描与安全问题分析：`jsec`

#### 适用场景

- 想扫描项目中的安全问题
- 想分析外部工具导出的安全问题 JSON
- 想统一配置扫描目标、语言、输出格式和验证策略

#### 第一步：先做配置

```bash
jsec config --target .
jsec config --show
```

你可以配置：

- 扫描目标
- 语言过滤
- 排除目录
- 输出格式和输出文件
- 聚类限制
- 是否启用二次验证
- 是否启用强制记忆
- 使用哪个模型组

#### 第二步：执行扫描

配置完成后，真正开始扫描使用：

```bash
jsec scan
```

适合项目目录扫描，也适合先指定单个文件后再执行扫描。

#### 第三步：分析外部 JSON 结果

如果你已经有第三方工具导出的安全问题 JSON，可使用：

```bash
jsec analyze issues.json
```

#### 使用效果

- 安全问题分析流程更标准化
- 适合将扫描配置固定下来反复使用
- 既能处理项目扫描，也能处理外部输入结果
- 可以形成“配置 → 扫描/分析 → 报告输出”的完整闭环

### 8.2 C→Rust 迁移：`jc2r`

#### 适用场景

- 想把 C 项目逐步迁移到 Rust
- 想先配置根符号、禁用库和附加说明
- 想在迁移过程中反复执行和逐步推进

#### 第一步：先做配置

```bash
jc2r config --files bzlib.h
jc2r config --show
```

你可以配置：

- 根符号列表
- 禁用库列表
- 附加说明
- 是否启用 FFI 导出验证

#### 第二步：执行完整迁移流水线

```bash
jc2r run
```

这条命令会按顺序执行完整迁移流程，并支持断点续跑。

如果你希望从头开始重新跑：

```bash
jc2r run --reset
```

如果你希望在执行过程中开启交互模式：

```bash
jc2r run --interactive
```

#### 第三步：验证 Rust 与原 C 的功能对齐

```bash
jc2r verify
```

#### 使用效果

- 迁移任务更容易分阶段管理
- 配置可复用，适合多轮迁移
- 支持断点续跑，也支持从头执行
- 能形成“配置 → 运行流水线 → 对齐验证”的完整闭环

### 8.3 内置专家与规则驱动场景

除了通用命令和专项工具，Jarvis 还适合处理规则驱动的专项任务。常见场景包括：

- 生成项目架构图
- 从现有代码提取设计方案
- 研究开源项目部署步骤
- 生成可执行脚本和使用说明
- 检查重构前后逻辑是否一致
- 进行安全漏洞分析
- 切换不同对话风格或人格

使用效果：

## 10. 常见任务示例

### 10.1 分析项目

```bash
jvs -T "分析当前项目的模块结构并总结职责"
```

效果：快速得到项目概览和下一步建议。

### 10.2 修改代码

```bash
jca -T "为登录接口增加参数校验和错误提示"
```

效果：自动完成代码读取、修改建议、变更检查和结果展示。

### 10.3 配置模型

```bash
jqc
```

效果：一步步完成平台接入和模型选择。

### 10.4 浏览器中操作 Agent

```bash
jwg serve --gateway-password 123456
```

效果：通过浏览器管理 Agent、终端和编辑器。

### 10.5 生成提交信息

```bash
jgc
```

效果：根据当前变更自动生成更规范的提交说明。

### 10.6 执行安全扫描

```bash
jsec config --target .
jsec scan
```

效果：基于已保存配置执行完整扫描并输出报告。

### 10.7 分析外部安全问题文件

```bash
jsec analyze issues.json
```

效果：适合对第三方扫描结果进行进一步整理和分析。

### 10.8 执行 C→Rust 迁移并校验结果

```bash
jc2r config --files bzlib.h
jc2r run
jc2r verify
```

效果：完成从迁移准备到结果校验的完整闭环。

## 12. 推荐使用顺序

如果你是第一次使用 Jarvis，推荐这样逐步上手：

1. `jqc`：先把模型平台接好
2. `jvs`：先体验通用任务执行
3. `jca`：再体验代码场景自动化
4. `jwg serve`：如果需要浏览器界面，再启用 Web Gateway
5. `jgc` / `jt` / `jmo` / `jm` / `jss`：按需扩展自己的工作流
6. `jsec` / `jc2r`：进入专项专业场景

这样上手，通常会更顺。
