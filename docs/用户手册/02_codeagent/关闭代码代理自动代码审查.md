# 关闭代码代理自动代码审查

## 使用场景

当你只想让代码代理完成代码分析、修改和基础收尾流程，而不希望它在修改后继续进入自动代码审查与修复循环时，可以在启动时关闭自动代码审查。

## 操作步骤

1. 在项目目录中打开终端。
2. 输入类似命令：`jca --disable-review --task "你的任务"`
3. 按回车后，系统会按正常方式执行本次代码任务。
4. 等代码修改完成后，系统会跳过自动代码审查流程，直接进入后续结果整理与结束阶段。

## 你会看到的提示与反馈

- 关闭后，系统不会再进入“第几轮代码审查”的确认与循环提示。
- 任务仍然会继续进行代码修改、提交结果整理以及其他后续流程，不是完全停掉任务执行。
- 如果当前任务实际上没有产生代码变更，本来也不会进入审查环节。

## 注意事项

- 这个参数只影响“代码修改完成后的自动审查流程”，不影响你正常发起代码任务。
- 如果你当前处理的是高风险改动，关闭自动审查会减少系统的二次检查环节，建议只在你明确知道自己为什么要关闭时再使用。
- 这是一项启动参数，适合本次任务临时生效；下次不传时，仍会回到默认行为。
