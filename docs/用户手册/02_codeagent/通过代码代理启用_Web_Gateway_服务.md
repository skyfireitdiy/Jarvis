# 通过代码代理启用 Web Gateway 服务

## 使用场景

当你希望在启动代码代理时，同时打开一个本地 Web Gateway 入口，让外部页面或其他本地客户端通过 WebSocket 连接当前会话时，可以在启动时启用这个服务。

## 操作步骤

1. 在项目目录中打开终端。
2. 输入下面的命令，然后按回车：

   `jca --web-gateway`

3. 如果你想指定监听端口，可以改成：

   `jca --web-gateway --web-gateway-port 8001`

4. 如果你需要为网关设置密码认证，可以在启动时追加：

   `jca --web-gateway --gateway-password "你的密码"`

5. 启动完成后，系统会先准备代码代理环境，再在本机启动 Web Gateway 服务。
6. 之后你可以让支持该入口的本地客户端连接到终端里显示的 WebSocket 地址。

## 你会看到的提示与反馈

- 服务启动成功后，终端会显示类似 `ws://127.0.0.1:端口/ws` 的本地连接地址。
- 如果你设置了认证信息，系统会按当前启动方式启用相应认证。
- 如果网关启动失败，终端会显示警告信息，代码代理会继续按原流程运行或结束，具体取决于当次失败位置。
- 启用后，还会提供一个本地状态查询入口，用于查看当前执行状态。

## 注意事项

- 这是本机监听地址，默认不是对外网开放的公共地址。
- 如果端口已被占用，或者本机环境不满足启动条件，网关可能无法正常启动。
- `--web-gateway` 只是额外启动一个服务，不会替代你当前的终端交互方式；你仍然可以像平时一样在终端中运行任务。
- 如果你没有明确的 Web 接入需求，通常不必额外开启这个选项。
