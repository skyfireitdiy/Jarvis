# 设置代码代理审查最大迭代次数

## 使用场景

当你希望自动代码审查不要无限延长，或者你想明确限制“审查发现问题后最多再修几轮”时，可以在启动代码代理时设置审查最大迭代次数。

## 操作步骤

1. 在项目目录中打开终端。
2. 输入类似命令：`jca --review-max-iterations 3 --task "你的任务"`
3. 按回车后，系统会先执行正常代码任务。
4. 如果任务产生了代码修改，后续进入审查阶段时，系统会按照你设置的上限控制循环轮数。
5. 每一轮开始前，系统都会先询问你是否继续进行本轮代码审查。

## 你会看到的提示与反馈

- 每轮审查开始前，你会看到类似“是否进行第 1 轮代码审查？”这样的确认提示。
- 审查进行时，系统会显示当前轮次，例如“代码审查循环 - 第 1/3 轮”。
- 如果某一轮已经通过，系统会提示当前轮审查通过，并结束后续循环。
- 如果达到你设置的最大次数后仍有问题，系统会停止继续审查，不再无限追加下一轮。
- 当你把最大次数设为 `0` 时，表示不设上限，系统会按无限模式处理，但每一轮开始前仍然会先询问你是否继续。

## 注意事项

- 这个参数只有在自动代码审查启用时才有意义；如果你同时关闭了自动审查，这个上限设置就不会生效。
- 最大迭代次数控制的是“审查与修复循环”的上限，不是整个任务的总轮数。
- 如果你希望减少耗时，可以设置一个较小的上限；如果你更重视反复打磨结果，可以提高上限，或者使用 `0` 进入无限模式。
