# 为代码代理临时切换模型组或工具组

## 使用场景

当你希望这一次代码任务使用不同的模型能力，或者想临时切换到另一套工具权限范围时，可以在启动代码代理时临时指定模型组或工具组。

## 操作步骤

1. 在项目目录中打开终端。
2. 根据需要输入类似命令：
   - 切换模型组：`jca -g 你的模型组名称 --task "你的任务"`
   - 切换工具组：`jca -G 你的工具组名称 --task "你的任务"`
   - 同时切换：`jca -g 你的模型组名称 -G 你的工具组名称 --task "你的任务"`

3. 按回车后，系统会按你这次指定的组别启动并执行任务。
4. 任务结束后，如需恢复默认设置，下一次直接不传这些参数即可。

## 你会看到的提示与反馈

- 这类切换通常不会要求你在运行中再次确认，而是在启动时直接按本次命令生效。
- 如果你同时指定了模型组或工具组，当前命令会优先使用你这次输入的值，而不是继续沿用原来的默认设置。
- 后续任务执行时，你会看到与当前任务对应的正常运行输出；组别切换本身不会单独生成一篇额外结果页。

## 注意事项

- 这里的切换是“本次启动时临时覆盖”，适合针对某个任务单独调整，不等于永久修改默认配置。
- 只有当前环境里已经存在对应模型组或工具组时，这种写法才有意义；如果名称写错，任务可能无法按预期运行。
- 如果你同时还要控制附加工具、规则或审查行为，可以把这些参数一起写在同一条命令中。
