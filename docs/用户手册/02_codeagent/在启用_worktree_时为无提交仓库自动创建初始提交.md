# 在启用 worktree 时为无提交仓库自动创建初始提交

## 使用场景

当你的 Git 仓库刚初始化、还没有任何提交记录，但你又想直接使用 worktree 模式执行任务时，系统会先补齐初始提交，再继续创建独立工作目录。

## 操作步骤

1. 进入一个已经初始化为 Git 仓库、但还没有任何提交记录的项目目录。
2. 输入 `jca -w`，然后按回车。
3. 系统在准备创建 worktree 时，如果发现当前仓库还没有提交历史，会先自动补齐初始提交。
4. 初始提交完成后，系统再继续后面的 worktree 创建流程。

## 你会看到的提示与反馈

- 当仓库没有任何提交记录时，系统不会直接报错退出，而是会先尝试补齐起始提交。
- 补齐完成后，终端会继续显示后续的 worktree 创建信息。
- 如果初始提交失败，worktree 创建通常也会失败，终端会直接给出失败提示。

## 注意事项

- Git worktree 依赖现有提交历史，所以无提交仓库必须先补齐至少一次提交。
- 这一步是 worktree 模式下的自动前置处理，不需要你手动先执行额外命令。
- 如果你不想让当前仓库立刻生成第一次提交，就不要在这种状态下直接使用 worktree 模式。
