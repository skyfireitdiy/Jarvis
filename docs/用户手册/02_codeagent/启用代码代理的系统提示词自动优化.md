# 启用代码代理的系统提示词自动优化

## 使用场景

当你的任务描述比较复杂、目标很多，或者你希望系统在正式执行前先自动整理任务表达方式，再进入代码分析与修改流程时，可以启用系统提示词自动优化。

## 操作步骤

1. 在项目目录中打开终端。
2. 输入下面的命令，然后按回车：

   `jca --optimize-system-prompt`

3. 如果你已经准备好明确任务，也可以直接一起输入，例如：

   `jca --optimize-system-prompt --task "梳理订单模块异常分支，补齐日志并修复回滚问题"`

4. 交互模式下，启动后继续输入你的任务内容并提交；非交互模式下，系统会直接使用 `--task` 后面的内容开始执行。
5. 后续执行过程中，系统会先根据你的需求自动优化当前任务所使用的提示词，再继续后面的代码处理流程。

## 你会看到的提示与反馈

- 启动方式和普通 `jca` 基本一致，仍会先完成环境检查和仓库准备。
- 这个能力不会额外要求你输入新的命令；它是在当前启动参数已开启的前提下自动发生的。
- 如果你同时提供了任务内容，系统会直接进入执行；如果你没有提供任务内容，交互模式下仍然会等待你输入需求。

## 注意事项

- 这是一个启动时开关，不是在任务执行到一半时临时输入的命令。
- 它的作用是让系统更贴合当前代码任务目标，不等于替你补充缺失需求；你的原始任务说明仍然要尽量写清楚。
- 如果你只是处理非常简单、非常明确的小任务，不一定需要专门开启这个选项。
