# 在启用 worktree 时自动创建独立分支和工作目录

## 使用场景

当你希望一次代码任务拥有独立分支和独立目录，便于并行处理、隔离改动和后续回看时，worktree 模式会自动帮你准备好这些基础环境。

## 操作步骤

1. 在目标仓库中打开终端。
2. 输入 `jca -w`，或者输入 `jca -w --task "你的任务"`。
3. 启动后，系统会自动为当前任务生成独立分支，并创建对应的独立工作目录。
4. 创建完成后，系统会切换到新的工作目录，再继续本次代码任务。

## 你会看到的提示与反馈

- 终端会先提示 worktree 模式已启用。
- 接着会显示当前分支信息，帮助你确认本次任务是从哪个分支分离出去的。
- 创建完成后，终端会明确提示已经切换到新的 worktree 目录。
- 如果创建分支或目录失败，系统会立即报错并终止本次启动。

## 注意事项

- 这一步是 worktree 模式的核心价值之一：任务不直接在你当前目录里改，而是在单独目录里进行。
- 非 worktree 模式不会自动创建独立分支和目录。
- 如果你的仓库状态异常、分支不可创建或权限不足，本次流程会失败，不会自动降级成普通模式。
