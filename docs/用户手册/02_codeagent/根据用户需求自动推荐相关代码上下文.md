# 根据用户需求自动推荐相关代码上下文

## 使用场景

当你的任务涉及多个文件、多个模块，或者你给出的需求比较长时，系统会尝试在正式处理前自动补充更相关的代码上下文，帮助后续分析和修改更快聚焦到关键位置。

## 操作步骤

1. 在项目目录中打开终端。
2. 使用交互模式或非交互模式启动代码代理，例如输入 `jca`，或者输入 `jca -n --task "修复订单超时取消逻辑"`。
3. 正常提交你的任务内容，不需要额外再输入“推荐上下文”之类的命令。
4. 系统收到任务后，会根据你当前的需求自动尝试推荐更相关的代码上下文，再继续后续分析与修改流程。

## 你会看到的提示与反馈

- 这是自动发生的，不需要额外输入命令。
- 你看到的任务说明区域中，可能会带上系统补充的上下文信息，用来帮助后续处理更准确地聚焦相关文件和模块。
- 如果当前任务很简单，或者系统没有找到特别有价值的补充上下文，后续流程也会照常继续，不会因为没有推荐结果而中断。

## 注意事项

- 这个能力的作用是帮助系统更快找到相关代码，不等于替你补齐业务目标；你的任务描述仍然应该尽量写清楚。
- 它属于执行前的自动辅助行为，不是你在运行中随时单独触发的命令。
- 如果你已经明确给出了很具体的文件、模块或目标，系统推荐的上下文通常只起补充作用。
