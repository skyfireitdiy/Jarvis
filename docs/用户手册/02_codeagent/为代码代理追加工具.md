# 为代码代理追加工具

## 使用场景

当默认工具集不够用，而你又希望当前这次代码任务额外具备某些工具能力时，可以在启动代码代理时临时追加工具。

## 操作步骤

1. 在项目目录中打开终端。
2. 输入类似命令：`jca --append-tools "工具A,工具B" --task "你的任务"`
3. 如果你只追加一个工具，也可以只写一个名称，例如：`jca --append-tools "tool_x" --task "你的任务"`
4. 按回车后，系统会在代码代理默认工具集的基础上，把你额外指定的工具一起带入本次任务。
5. 等待任务执行完成。

## 你会看到的提示与反馈

- 追加工具后，系统会继续按正常方式运行，不需要你在进入任务后再次手动启用这些工具。
- 同一个工具如果本来就在默认工具集中，系统会自动避免重复追加。
- 追加成功后，本次任务可用工具范围会比默认情况更大，但终端里通常仍以正常任务执行输出为主。

## 注意事项

- 多个工具名称之间要用英文逗号分隔。
- 这里只影响当前这一次启动，不等于永久修改默认工具集。
- 追加工具前，最好确认这些工具确实是当前环境支持的；如果工具名称写错，可能无法达到预期效果。
- 如果你不确定是否真的需要额外工具，优先保持最小工具集，通常更容易控制任务边界。
