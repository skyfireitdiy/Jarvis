# 在代码修改后自动展示增强版 diff 预览和按文件补丁摘要

## 使用场景

当你希望在代码代理完成一轮修改后，马上看到这次改动改了哪些文件、每个文件大概改了多少、哪些内容被删除或新增时，这个自动预览功能会很有用。

## 操作步骤

1. 在项目目录中启动代码代理，并提交一个会产生代码改动的任务。
2. 等待系统完成本轮修改。
3. 修改一旦真正产生，系统会自动展示本次改动的 diff 预览，并整理出按文件显示的补丁摘要。
4. 查看终端中的文件级变更信息，再决定是否继续后续流程。

## 你会看到的提示与反馈

- 这是代码发生修改后自动触发的，不需要额外输入命令。
- 终端会先展示本轮变更的整体 diff 预览。
- 同时还会整理出“按文件”的补丁内容或摘要，方便你快速判断到底改了哪些文件。
- 如果某个文件变更过大，系统通常不会把完整内容全部展开，而是改用更紧凑的摘要方式显示。
- 如果文件已经被删除，系统通常会提示该文件已删除，而不是继续展示完整 diff 内容。

## 注意事项

- 这个功能只有在确实产生代码变更时才会出现；如果本轮没有改动，就不会有补丁预览。
- 大文件或超大改动通常不会完整展开，这是为了让终端输出保持可读。
- 补丁摘要适合快速总览；如果你还需要逐行核对，请继续用 Git 工具或编辑器查看完整差异。
