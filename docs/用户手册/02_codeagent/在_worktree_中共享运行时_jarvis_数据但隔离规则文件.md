# 在 worktree 中共享运行时 .jarvis 数据但隔离规则文件

## 使用场景

当你在 worktree 模式下并行处理多个代码任务时，系统会尽量让各个 worktree 共享运行时数据，但把更容易发生分支冲突的规则文件隔离开，减少相互干扰。

## 操作步骤

1. 使用 `jca -w` 启动代码代理。
2. 等待系统完成 worktree 创建和目录切换。
3. 后续在新的 worktree 中照常执行代码任务，不需要你手动复制或重新准备 `.jarvis` 目录。
4. 如果你的任务会依赖运行时数据，系统会自动复用共享部分；如果涉及规则文件，系统会按独立方式处理。

## 你会看到的提示与反馈

- worktree 准备阶段，终端会显示 `.jarvis` 目录相关的准备信息。
- 某些运行时目录或文件会以共享方式接入，不需要你重复配置。
- 容易引起分支冲突的规则内容会被单独隔离，避免多个 worktree 互相覆盖。
- 如果 `.jarvis` 目录准备失败，终端会提示失败并尝试回滚。

## 注意事项

- 这一步完全属于 worktree 启动过程中的自动处理，不需要你手动操作。
- 共享运行时数据的好处是减少重复准备；隔离规则文件的好处是减少不同任务之间的规则冲突。
- 如果你的项目高度依赖自定义规则，建议在进入 worktree 后再确认当前规则是否符合这次任务预期。
