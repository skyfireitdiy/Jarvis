# 把代码代理任务派发到 tmux 窗口

## 使用场景

当你的代码任务执行时间较长，或者你想把当前任务丢到新的 tmux 窗口中异步运行，再回到当前窗口继续别的工作时，可以使用派发模式。

## 操作步骤

1. 先进入已经运行在 tmux 会话里的终端窗口。
2. 在当前项目目录中输入类似下面的命令，然后按回车：

   `jca --dispatch --task "重构缓存清理逻辑，并检查相关测试"`

3. 系统会立即尝试把这条任务派发到新的 tmux 窗口中运行。
4. 当前命令成功派发后会直接退出，不会留在原窗口继续执行任务。
5. 你可以切换到新创建的 tmux 窗口查看后续运行过程。

## 你会看到的提示与反馈

- 派发开始时，终端会提示正在派发任务到新的 tmux 窗口。
- 如果派发成功，你会看到任务已成功派发到新的 tmux 窗口的提示，然后当前进程结束。
- 如果你只写了 `--dispatch`，却没有同时提供 `--task`，系统会直接报错并提示这个参数必须和 `--task` 一起使用。
- 如果当前环境不能创建 tmux 窗口或窗格，系统会提示派发失败，并要求你检查 tmux 配置。

## 注意事项

- 这个功能只在 tmux 环境中才有意义；如果你不是在 tmux 会话里启动，通常不适合这样用。
- `--dispatch` 需要和 `--task` 一起使用，不能先启动后再补输入任务。
- 派发成功后，当前窗口不会继续展示任务执行过程；你需要切换到目标 tmux 窗口查看运行状态。
- 如果任务内容很长，不适合直接写在命令行里，可以改用任务文件，或者直接使用 `jcad`。
