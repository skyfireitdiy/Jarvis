# 使用极速模式快速执行任务

## 使用场景

当你希望代码代理快速启动并执行任务，不需要系统进行任务分类、规则自动加载、上下文推荐等耗时操作时，可以使用极速模式。极速模式适用于：

- 你已经明确知道要做什么，不需要系统进行智能分析
- 你希望快速启动代码代理，减少等待时间
- 你已经在命令中提供了足够的上下文信息
- 你需要批量执行多个简单任务

## 操作步骤

1. 先在你的项目目录中打开终端。
2. 使用 `-q` 或 `--quick` 参数启动代码代理：

   - **交互模式**：输入 `jca -q`，然后按回车。
   - **非交互模式**：输入类似下面的命令，然后按回车：

     `jca -q -n --task "修复配置加载失败的问题"`

3. 系统会直接启动代码代理，跳过任务分类、规则加载和上下文推荐等步骤。
4. 在交互模式下，直接输入你的任务需求；在非交互模式下，系统会直接执行 `--task` 后面的内容。

## 你会看到的提示与反馈

- 启动后会看到欢迎信息，表示代码代理已经准备就绪。
- 极速模式下，系统会显示提示信息：`⚡ 极速模式已启用：跳过任务分类、规则加载、上下文推荐`。
- 系统会保留 Git 环境初始化和安全回退点设置，确保代码安全。
- 交互模式下，你会看到输入需求的提示，并可继续输入任务内容。
- 非交互模式下，系统会直接使用 `--task` 后面的内容开始执行。

## 注意事项

- 极速模式会跳过以下功能：
  - 任务分类（系统不会自动识别你的任务类型）
  - 规则自动加载（不会根据任务类型加载特定规则）
  - 上下文推荐（不会自动推荐相关代码上下文）
  - 方法论加载（不会加载相关的方法论文档）
- 极速模式会保留以下功能：
  - Git 环境初始化
  - 安全回退点设置
  - 代码修改、审查和提交功能
- 如果你需要系统进行智能分析和推荐，建议不使用极速模式。
- 极速模式可以与其他参数组合使用，如 `-n`（非交互模式）、`-T`（任务描述）等。
- 如果你的任务比较复杂，建议不使用极速模式，让系统进行完整的分析和准备。
