# 在代码修改后自动执行检查点提交

## 使用场景

当你希望代码代理在每轮真实修改后都留下一个可回退的检查点，方便后续审查、撤回或整理提交历史时，系统会在改动完成后自动进行检查点提交。

## 操作步骤

1. 启动代码代理并提交一个会修改代码的任务。
2. 等待系统完成本轮修改。
3. 一旦检测到有未提交的代码改动，系统会自动创建检查点提交，再继续后续审查、分析和提交整理流程。
4. 在终端中查看自动提交结果和对应的提交信息。

## 你会看到的提示与反馈

- 这是自动触发的，不需要额外输入 Git 命令。
- 修改成功后，终端会显示“代码已修改完成”和“已自动提交”等提示。
- 同时还会展示本次自动提交的 Commit ID、提交信息以及补丁摘要。
- 如果本轮修改没有真正生效，系统会提示修改没有生效，而不是继续显示成功提交结果。

## 注意事项

- 检查点提交的主要目的是留存当前可回退状态，后面还有机会继续整理为更规范的最终提交。
- 如果你的任务最终没有形成有效改动，就不会产生这类自动提交。
- 这一步是系统自动完成的，不代表最终提交历史已经是你想保留的最终版本。
