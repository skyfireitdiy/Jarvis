# 通过全局或项目级 after_change_tools 配置自定义修改后处理命令

## 使用场景

当默认的修改后处理规则还不够用，例如你希望某类文件在修改后自动执行特定格式化命令、自动修复命令或团队自定义脚本时，可以通过 after_change_tools 配置扩展这一步。

## 操作步骤

1. 确认你想扩展的是“全局规则”还是“当前项目专用规则”。
2. 如果是全局规则，在你的 Jarvis 数据目录中找到 `after_change_tools.yaml`；如果是项目专用规则，在当前项目的 `.jarvis` 目录中准备 `after_change_tools.yaml`。
3. 按“文件扩展名或文件名 → 命令列表”的方式写入规则。
4. 保存配置后，重新运行一次代码代理，让它修改到匹配这些规则的文件。
5. 当本轮流程进入修改后处理阶段时，系统会自动读取这些配置，并对匹配文件执行你定义的命令。

## 你会看到的提示与反馈

- 命令真正匹配并执行时，终端会像普通后处理一样显示某个文件已处理。
- 如果项目级和全局规则同时存在，当前项目里的规则会优先覆盖全局规则。
- 如果配置里的命令执行失败、超时或工具不存在，终端会显示相应警告。

## 注意事项

- 这类配置适合放你已经确认稳定、可重复执行的处理命令，不适合写高风险或会破坏文件内容的临时脚本。
- 项目级规则只影响当前项目，更适合团队规范；全局规则会影响你之后处理的其他项目。
- 命令会在修改后自动执行，所以写入配置前应先确认这些命令可以无交互运行，并且不会误改文件。
