# 自动调整 Git 换行符配置以减少无意义 diff

## 使用场景

当你在不同平台上处理同一个仓库，或者仓库里经常因为换行符差异出现大量无意义 diff 时，系统会在准备 Git 环境时自动调整相关设置，尽量减少这类干扰。

## 操作步骤

1. 在 Git 仓库中启动代码代理。
2. 系统准备 Git 环境时，会自动检查当前仓库的换行符相关设置。
3. 如果当前设置和推荐值不一致，系统会直接更新这些设置，然后继续后续流程。
4. 完成后，继续正常执行你的代码任务。

## 你会看到的提示与反馈

- 当系统发现当前换行符配置与推荐值不一致时，终端会先给出提醒，并列出将要调整的设置项。
- 设置完成后，终端会提示 Git 换行符敏感设置已更新。
- 如果当前配置本来就符合要求，系统通常不会重复提示。

## 注意事项

- 这一步不需要你手动确认，系统会直接执行调整。
- 它的目标是减少因为平台差异导致的无意义改动，不是为了替代你项目中的换行符规范。
- 如果你的团队已经有明确的 Git 换行符策略，运行后最好核对一下自动设置是否与团队要求一致。
