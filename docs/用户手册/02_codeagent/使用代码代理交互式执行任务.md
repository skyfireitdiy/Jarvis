# 使用代码代理交互式执行任务

## 使用场景

当你想一边描述代码需求、一边在当前仓库中让系统继续分析、修改和验证代码时，可以直接以交互模式启动代码代理。

## 操作步骤

1. 先在你的项目目录中打开终端。
2. 输入 `jca`，然后按回车启动代码代理。
3. 如果当前目录还不是 Git 仓库，系统会询问是否要在当前目录初始化一个新的 Git 仓库。
4. 进入交互界面后，系统会提示你输入需求。
5. 直接输入本次代码任务，例如“帮我修复登录接口的空指针问题，并补充必要测试”。
6. 输入完成后按回车提交；如果需要结束本次输入或退出，按提示使用 `Ctrl+C`。
7. 系统会开始执行当前任务，并在任务完成后结束本轮运行。

## 你会看到的提示与反馈

- 启动后会看到欢迎信息，表示代码代理已经准备就绪。
- 如果当前目录不是 Git 仓库，你通常会看到是否初始化仓库的确认提示；同意后会继续自动初始化，并补充默认忽略规则。
- 输入需求前，会看到类似“请输入你的需求（Ctrl+C 退出）”的提示。
- 如果当前仓库里存在与当前提交一致的历史会话，系统可能会先让你选择是否恢复其中一个会话，再继续输入新的需求。
- 如果你没有输入任何内容直接结束输入，本次运行会直接退出，不会继续执行任务。

## 注意事项

- 交互模式适合临时说明、边想边改的代码任务；如果你的任务很长，改用任务文件方式通常更方便。
- 代码代理会在当前仓库中运行，因此建议先进入你要处理的项目目录再启动。
- 如果当前目录不是 Git 仓库，而你又不希望初始化仓库，可以在确认时选择不继续，本次运行会结束。
- 如果你希望直接恢复旧会话而不是等待系统自动提示，可以改用带 `--restore-session` 的启动方式。
