# 使用 jcad 交互输入多行代码任务并派发到 tmux

## 使用场景

当你的任务说明很长，包含多段背景、限制条件和目标，而且你希望输入完成后直接把任务派发到 tmux 中执行时，可以直接启动 `jcad` 的交互输入模式。

## 操作步骤

1. 先进入项目目录，并确保你当前使用的终端环境可以正常派发 tmux 任务。
2. 输入 `jcad`，然后按回车。
3. 终端会出现多行输入提示，按提示直接输入完整任务内容。
4. 在输入过程中，普通回车用于换行；输入完成后，按提示中的确认快捷键提交整段内容。
5. 提交后，系统会先把这段多行任务写入临时文件，再自动派发给代码代理执行。
6. 派发完成后，到对应的 tmux 窗口或面板中查看后续运行过程。

## 你会看到的提示与反馈

- 进入交互输入后，终端会明确提示：`请输入任务内容（Ctrl+J/Ctrl+] 确认，Enter 换行）`。
- 提交后，终端会显示临时文件已创建的提示，表示多行任务已经被保存并准备派发。
- 如果你什么都没输入就直接结束，系统会提示本次输入为空，并直接退出，不会继续派发任务。
- 如果派发失败，终端会直接显示错误信息。

## 注意事项

- 这种方式适合长任务说明；如果你只是派发一句短任务，直接写 `jcad "你的任务"` 会更省事。
- 提交多行输入时，不是按普通回车结束，而是按提示中的确认快捷键；普通回车只会继续换行。
- 系统会临时保存你的任务内容后再派发，因此你会看到临时文件创建提示，这是正常现象。
- 如果当前环境无法正常派发 tmux 任务，这种方式也会失败；此时请先确认 tmux 环境可用。
