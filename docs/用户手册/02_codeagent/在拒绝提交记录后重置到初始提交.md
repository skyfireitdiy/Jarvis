# 在拒绝提交记录后重置到初始提交

## 使用场景

当你查看完自动生成的提交记录后，确认这些结果不是你想保留的版本，并且希望直接回到本次任务开始前的基线状态时，可以在拒绝后选择重置。

## 操作步骤

1. 完成代码任务后，先查看终端列出的提交记录。
2. 当系统询问是否接受以上提交记录时，选择不接受。
3. 随后系统会继续询问你是否要重置到初始提交。
4. 如果你确认重置，系统会把当前仓库恢复到本次任务开始前的提交状态。

## 你会看到的提示与反馈

- 拒绝提交记录后，系统不会立即自动重置，而是继续询问你是否真的要回到初始提交。
- 如果你确认重置，终端会提示已经重置到初始提交。
- 如果你不确认重置，当前流程会停在现有状态，不会继续强制回退。

## 注意事项

- 这是一个明确的回退动作，适合你确认本次中间结果完全不想保留的场景。
- 一旦完成重置，本次任务产生的后续提交结果就会被清掉，所以在确认前要先看清楚终端里的提交摘要。
- 如果你只是想调整最终提交信息，而不是彻底回退，通常应该选择接受并让系统继续整理最终提交，而不是走重置分支。
