# 使用 jcad 直接派发代码任务

## 使用场景

当你只是想快速把一条代码任务丢给代码代理执行，并且希望系统自动按适合的派发方式启动，而不想手动拼接较长的 `jca` 参数时，可以直接使用 `jcad`。

## 操作步骤

1. 先在项目目录中打开终端。
2. 输入下面的命令，然后按回车：

   `jcad "修复导出接口在空数据下的异常，并补充回归测试"`

3. 如果这段内容是一条单行任务，系统会直接把它派发给代码代理执行。
4. 如果你传入的内容本身就是一个已经存在的文件路径，系统会把它当作任务文件处理，而不是普通文本。
5. 派发后，等待系统把任务交给代码代理运行。

## 你会看到的提示与反馈

- `jcad` 会调用代码代理的派发流程，因此你通常会看到后续由代码代理输出的执行信息。
- 如果系统找不到 `jca` 命令，终端会直接报错并提示你先确认 Jarvis 已正确安装。
- 如果你提供的是多行内容，`jcad` 会自动改用临时文件方式再派发，而不是强行把整段文本塞进一条命令里。

## 注意事项

- `jcad` 更适合快速启动一次性任务，不适合需要你在启动后继续长时间互动补充说明的场景。
- 如果你传入的是文件路径，系统会把它当作任务文件读取；如果你本来想传普通文本，请确认这个文本不是当前目录里真实存在的文件名。
- 想直接输入较长多段任务时，建议使用 `jcad` 的交互输入方式，而不是在一条命令里硬写多段文本。
