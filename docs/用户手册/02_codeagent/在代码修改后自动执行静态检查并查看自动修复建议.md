# 在代码修改后自动执行静态检查并查看自动修复建议

## 使用场景

当你希望在代码改完后立刻检查格式、语法、风格或静态问题，并优先拿到可直接执行的自动修复建议时，代码代理会在合适条件下自动执行静态检查。

## 操作步骤

1. 启动代码代理并提交会修改代码的任务。
2. 等待系统完成本轮改动和相关验证。
3. 如果当前文件类型支持静态检查工具，系统会自动运行这些检查，不需要你手动逐个执行。
4. 查看终端里的静态检查结果；如果系统给出了自动修复建议，优先按建议处理。

## 你会看到的提示与反馈

- 这是修改后自动触发的，不需要额外输入命令。
- 如果静态检查通过，结果中会显示静态扫描通过。
- 如果发现问题，系统会把问题整理成后续修复提示，并在可用时优先给出自动修复命令建议。
- 如果当前环境里缺少对应工具，你可能会看到相关工具无法运行的结果；这不会一定中断主流程。
- 如果前面的完整构建验证已经失败且你还没有切换到基础静态检查路线，系统可能会跳过这一步，避免重复堆叠错误。

## 注意事项

- 自动修复建议通常是优先处理方式，但不代表一定能一次修好所有问题。
- 不同语言和文件类型会匹配不同的静态检查工具，不是所有文件都会得到同一种检查。
- 如果你的环境没有安装对应工具，终端里可能会看到“工具未安装”或命令不可用之类的情况，这属于环境问题，不一定是代码问题。
