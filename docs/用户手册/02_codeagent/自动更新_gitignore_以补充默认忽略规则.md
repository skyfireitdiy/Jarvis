# 自动更新 .gitignore 以补充默认忽略规则

## 使用场景

当你在代码代理中处理一个新仓库，或者当前仓库的 `.gitignore` 还不完整时，系统会自动补充一套默认忽略规则，减少无意义文件被纳入版本控制。

## 操作步骤

1. 在项目目录中启动代码代理。
2. 如果当前目录还不是 Git 仓库，先按提示完成仓库初始化，或者在已有 Git 仓库中直接继续。
3. 系统在准备 Git 环境时，会自动检查 `.gitignore` 是否已经包含默认忽略规则。
4. 如果缺失，系统会自动创建或更新 `.gitignore`，然后继续后续流程。

## 你会看到的提示与反馈

- 如果当前仓库还没有 `.gitignore`，系统会提示已创建 `.gitignore` 并添加默认忽略规则。
- 如果已有 `.gitignore` 但缺少默认规则，系统会提示已更新 `.gitignore`。
- 如果原文件已经包含这类默认规则，系统通常不会重复追加，也不会反复提示。

## 注意事项

- 这是 Git 环境准备阶段的自动行为，不需要你单独输入命令。
- 自动补充默认忽略规则的目的，是减少临时文件、运行时数据或常见构建产物被误加入版本控制。
- 如果你的项目已经有非常严格的 `.gitignore` 规范，运行后最好再检查一下自动追加的内容是否符合团队约定。
