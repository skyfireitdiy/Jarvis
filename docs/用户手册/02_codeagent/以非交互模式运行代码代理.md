# 以非交互模式运行代码代理

## 使用场景

当你希望把代码任务写成一条命令直接执行，或者在脚本、自动化流程、远程任务环境中运行代码代理时，可以使用非交互模式。

## 操作步骤

1. 先在项目目录中打开终端。
2. 输入类似下面的命令：

   `jca -n --task "修复配置加载失败的问题，并检查相关文件"`

3. 按回车后，系统会直接开始执行任务，不再进入交互式输入界面。
4. 等待任务执行完成，并查看终端中的执行结果。

## 你会看到的提示与反馈

- 非交互模式下，系统不会再打开多行输入界面，而是直接使用 `--task` 后面的内容作为任务描述。
- 如果你只写了 `jca -n`，没有同时提供 `--task`，系统会直接报错并提示：`❌ 非交互模式已启用：必须使用 --task 传入任务内容，因多行输入不可用。`
- 如果当前目录不是 Git 仓库，系统会自动按“同意初始化”的方式继续尝试初始化，而不会再等待你手动确认。

## 注意事项

- 非交互模式必须和 `--task` 一起使用，不能指望启动后再手动输入需求。
- 这种方式更适合单次明确任务，不适合需要多轮补充说明的长对话场景。
- 如果你的任务文本很长、包含多段说明，建议改用 `--task-file`，比在命令行里直接写长文本更稳定。
