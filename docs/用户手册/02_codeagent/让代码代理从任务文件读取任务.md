# 让代码代理从任务文件读取任务

## 使用场景

当你的代码任务很长，或者希望把同一份任务说明重复交给代码代理执行时，可以把任务写进文件，再让代码代理直接从文件读取。

## 操作步骤

1. 先在项目目录中准备一个文本文件，把任务说明写进去。
2. 在终端中输入：`jca --task-file 你的任务文件路径`
3. 按回车后，系统会读取文件内容，并把文件里的内容作为本次任务说明。
4. 如果你还想同时用非交互模式运行，可以在命令中加上 `-n`。
5. 等待任务执行完成，并在终端查看结果。

## 你会看到的提示与反馈

- 如果文件能正常读取，系统会直接开始执行，不需要你再手动粘贴同样的任务内容。
- 如果文件无法读取，系统会报错并提示类似：`❌ 错误: 无法从文件读取任务描述: ...`
- 如果你同时传入 `--task` 和 `--task-file`，系统会直接拒绝执行，并提示：`❌ 错误: 不能同时使用 --task 和 --task-file 参数`
- 如果任务文件中保存的是结构化任务内容，系统会自动把其中的任务说明、背景信息和附加信息整理后再执行。

## 注意事项

- `--task-file` 适合长任务、模板化任务和批处理场景。
- 不要同时使用 `--task` 和 `--task-file`，这两种输入方式只能二选一。
- 如果文件里除了主体任务外还包含背景和补充说明，系统会一并带入当前任务；因此写文件时最好把这些内容整理清楚。
