# 在代码审查结果格式错误时自动修复 JSON 格式

## 使用场景

当代码审查阶段已经跑起来了，但审查结果输出格式不符合系统要求时，系统会自动尝试修复结果格式，尽量避免因为格式问题直接中断整个审查循环。

## 操作步骤

1. 启动代码代理并让任务进入代码审查流程。
2. 如果某一轮审查结果格式不正确，系统会自动进入格式修复步骤。
3. 这一步不需要你手动重试，也不需要你自己整理 JSON。
4. 如果修复成功，系统会继续使用修复后的结果判断本轮是否通过；如果连续多次修复都失败，系统会重新开始这一轮审查。

## 你会看到的提示与反馈

- 格式有问题时，终端会提示正在尝试修复 JSON 格式，并显示当前是第几次修复尝试。
- 如果修复成功，终端会提示 JSON 格式修复成功，本轮流程继续。
- 如果多次修复仍然失败，终端会提示 JSON 格式修复失败，并重新进行当前轮的代码审查。
- 这类问题通常不会直接把整个任务终止，而是优先尝试自恢复。

## 注意事项

- 这是审查流程内部的自动恢复机制，不是普通输入阶段给用户手动调用的命令。
- 如果你看到这类提示，说明当前问题主要是审查结果格式异常，不一定代表代码本身一定有更严重的问题。
- 如果同一任务频繁出现这种情况，通常意味着本轮审查内容过于复杂，后续可以考虑把任务拆得更小一些。
