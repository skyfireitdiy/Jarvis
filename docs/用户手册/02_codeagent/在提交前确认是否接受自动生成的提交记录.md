# 在提交前确认是否接受自动生成的提交记录

## 使用场景

当系统已经根据本次任务自动生成了一组提交记录，但你还想在最终定稿前看一眼并决定是否接受时，可以直接在提交确认阶段做选择。

## 操作步骤

1. 启动代码代理并完成会修改代码的任务。
2. 等系统自动整理出本轮提交记录后，查看终端中列出的提交历史摘要。
3. 当终端询问你是否接受以上提交记录时，根据需要输入确认或拒绝。
4. 如果你接受，系统会继续整理最终提交；如果你拒绝，系统会进入后续拒绝分支。

## 你会看到的提示与反馈

- 系统会先列出检测到的提交记录，让你看到每条提交的简要信息。
- 随后会出现是否接受以上提交记录的确认提示。
- 如果你接受，系统会继续后面的最终整理与重新提交流程。
- 如果你不接受，系统不会强行直接采用这些提交记录，而是转入后续重置分支。

## 注意事项

- 这里确认的是“是否接受本次自动整理出的提交记录”，不是简单确认任务是否完成。
- 如果你对提交粒度、提交信息或中间结果不满意，可以在这一步拒绝。
- 在拒绝之前，建议先认真看一下终端里列出的提交摘要，避免误点通过。
