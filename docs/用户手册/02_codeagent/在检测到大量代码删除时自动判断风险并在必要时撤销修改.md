# 在检测到大量代码删除时自动判断风险并在必要时撤销修改

## 使用场景

当一次修改中出现大规模删除代码的情况时，系统会自动追加风险判断，避免因为误删关键逻辑而把危险改动直接保留下来。

## 操作步骤

1. 正常启动代码代理并提交你的代码任务。
2. 如果本轮修改后出现了大量代码删除，系统会自动进入额外风险判断流程。
3. 这一步不需要你手动输入命令；系统会根据当前补丁预览自动判断这些删除是否合理。
4. 如果判断结果认为风险过高，系统会自动撤销本轮修改；如果判断结果认为合理，则继续后续流程。

## 你会看到的提示与反馈

- 检测到大量删除时，系统会明确提示正在判断这些删除是否合理。
- 如果系统判断删除合理，会给出确认提示，然后继续后续提交流程。
- 如果系统判断删除不合理，终端会提示本次修改已被撤销，并附带按文件补丁预览，帮助你理解刚才被撤销的内容。
- 如果判断过程本身失败，系统会按更保守的方式处理，优先避免把高风险删除直接保留下来。

## 注意事项

- 这是自动保护机制，不是普通输入阶段可以随时手动调用的命令。
- 被撤销的情况通常意味着系统认为当前删除规模过大或不够可信，你应重新描述任务范围，避免让系统误删大量代码。
- 如果你的任务本来就是大规模精简、删除旧模块或移除废弃逻辑，请在任务说明里写清楚删除目的，这样更容易让后续判断与真实目标对齐。
