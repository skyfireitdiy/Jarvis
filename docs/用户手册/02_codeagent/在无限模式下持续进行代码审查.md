# 在无限模式下持续进行代码审查

## 使用场景

当你希望系统不预先限制审查轮数，而是持续进行“审查 → 修复 → 再审查”的循环，直到审查通过或你主动停止时，可以使用无限模式。

## 操作步骤

1. 在启动代码代理时，把代码审查最大迭代次数设置为 `0`。
2. 正常提交一个会修改代码的任务。
3. 修改完成后，系统会进入代码审查循环，并按无限模式继续运行。
4. 每轮开始前，你仍然可以根据提示决定是否继续下一轮审查。
5. 当审查通过，或者你主动拒绝继续时，无限模式才会结束。

## 你会看到的提示与反馈

- 进入这种模式后，终端会明确显示当前轮次处于无限模式。
- 每轮开始时，你会看到带有“无限模式”说明的轮次提示。
- 审查通过后，系统会像普通模式一样提示通过并结束。
- 如果你中途选择不再继续，系统会立刻停止后续轮次。

## 注意事项

- 无限模式不是“无人值守地永远运行”，因为每轮开始前通常仍会出现是否继续的确认。
- 适合复杂重构、质量要求高、需要多轮逐步收敛的问题。
- 如果你只是处理很小的改动，一般不必专门使用无限模式。
