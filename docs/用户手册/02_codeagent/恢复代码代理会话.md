# 恢复代码代理会话

## 使用场景

当你上一次的代码任务还没处理完，或者你希望继续沿用之前的任务上下文、最近记忆、规则状态和部分代码代理配置时，可以在启动代码代理时恢复历史会话。

## 操作步骤

1. 先进入之前处理过的项目目录。
2. 在终端中输入 `jca --restore-session`，然后按回车。
3. 如果系统找到了可恢复的会话，会先进入恢复流程，再继续本次运行。
4. 恢复完成后，继续输入你这次要补充的代码需求，或者直接沿用之前的上下文继续处理。
5. 如果你不想显式传参，也可以在已经开启自动恢复会话的启动方式中直接运行代码代理，让系统自动尝试恢复。

## 你会看到的提示与反馈

- 恢复成功后，你通常会看到类似 `✅ 已从 xxx 恢复会话。` 的提示。
- 如果恢复了最近记忆、记忆标签、规则状态或代码代理配置，系统还会继续显示对应的恢复结果，例如已恢复多少条最近记忆、已重新激活哪些规则。
- 在普通交互启动且你没有显式指定 `--restore-session` 时，如果当前提交对应的历史会话存在，系统也可能先提示你是否从匹配会话中恢复。
- 如果恢复失败，系统会给出失败提示，然后继续按普通新会话方式运行。

## 注意事项

- 会话恢复更适合在同一个项目、相近代码状态下继续工作。
- 恢复后，代码代理会尽量带回之前的工具组、审查配置、前后缀等运行状态，但这不代表所有外部环境都一定与上次完全一致。
- 如果当前仓库状态已经明显变化，即使有历史会话，也可能不适合直接继续沿用之前的任务结论。
- 如果你只是想重新执行同一类任务，而不是继续历史上下文，直接新开一次会话通常更干净。
