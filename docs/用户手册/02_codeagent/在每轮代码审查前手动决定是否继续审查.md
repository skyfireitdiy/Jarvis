# 在每轮代码审查前手动决定是否继续审查

## 使用场景

当你不想让系统一口气跑完整个审查循环，而是希望每轮开始前都自己决定要不要继续时，可以直接在轮次确认提示里做选择。

## 操作步骤

1. 启动代码代理并提交一个会修改代码的任务。
2. 进入代码审查流程后，留意每一轮开始前的确认提示。
3. 当终端询问是否进行下一轮代码审查时，根据你的需要选择继续或停止。
4. 如果你选择继续，系统会开始该轮审查；如果你选择不继续，当前审查循环会立刻结束。

## 你会看到的提示与反馈

- 每轮审查开始前，系统都会出现类似“是否进行第 N 轮代码审查？”的确认提示。
- 如果你确认继续，系统会马上显示本轮代码审查开始的信息。
- 如果你拒绝继续，终端会提示你已经停止在当前轮次继续代码审查，本轮流程随即结束。
- 在非交互模式下，这个确认的默认选择通常与交互模式不同，所以自动化运行时更容易继续执行。

## 注意事项

- 这里决定的是“是否继续下一轮审查”，不是“是否接受最终代码结果”。
- 如果你已经看过上一轮结果并确认没必要再继续，可以直接停止，避免多跑无效轮次。
- 交互模式下，系统通常不会默认替你一直继续，所以要留意终端中的确认提示。
