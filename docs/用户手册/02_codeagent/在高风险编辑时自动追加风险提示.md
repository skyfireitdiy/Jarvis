# 在高风险编辑时自动追加风险提示

## 使用场景

当本轮改动影响范围较大、接口变更多、相关测试受波及较广时，系统会自动把这次编辑标记为高风险，并提醒你重点复核。

## 操作步骤

1. 在项目目录中运行代码代理并提交代码修改任务。
2. 如果本轮修改被判定为高风险编辑，系统会在后续结果中自动追加提醒信息。
3. 根据这些提醒，优先检查被点名的受影响文件、接口变更和相关测试。

## 你会看到的提示与反馈

- 这是自动发生的，不需要额外输入命令。
- 高风险时，系统会在后续结果中增加明显的风险提示，提醒你这次编辑影响面较大。
- 提示内容通常会告诉你受影响文件数量、接口变更数量以及相关测试数量，帮助你判断要重点复核哪些部分。

## 注意事项

- 这类提示的重点不是阻止你继续，而是提醒你不要只看眼前改动，要一起检查周边影响。
- 如果你本来就在做跨模块重构、大规模接口调整或公共能力替换，看到高风险提示属于正常现象。
- 风险提示出现后，最好不要只依赖“代码能运行”这一点，最好再做针对性的回归检查。
