# 执行多轮代码审查与修复循环

## 使用场景

当你希望代码代理在完成第一轮修改后，不是立刻停下，而是继续检查本次改动是否真正满足需求、是否还有明显问题时，可以使用默认的代码审查与修复循环。

## 操作步骤

1. 在项目目录中启动代码代理，并提交一个会产生代码改动的任务。
2. 等待系统完成第一轮代码修改。
3. 如果当前没有关闭自动代码审查，系统会在修改后进入代码审查流程。
4. 审查发现问题时，系统会列出问题并继续进入修复阶段。
5. 修复完成后，如果还允许继续审查，系统会再次进入下一轮审查，直到通过、达到上限或你主动停止。

## 你会看到的提示与反馈

- 代码审查是在修改完成后自动进入的，不需要你单独输入“开始审查”命令。
- 每轮开始时，终端会显示当前是第几轮代码审查。
- 如果审查通过，终端会提示本轮代码审查通过，并给出简要总结。
- 如果审查未通过，系统会把发现的问题逐条列出来，包括问题类型、位置和修复建议，然后继续后面的修复流程。

## 注意事项

- 这项能力默认会在代码改动后发生；如果你不想使用，可以在启动时关闭自动代码审查。
- 它的目标是不断逼近“需求完成且改动合理”，所以适合需要质量把关的代码任务。
- 如果本轮实际上没有产生代码改动，审查循环通常不会继续进行。
