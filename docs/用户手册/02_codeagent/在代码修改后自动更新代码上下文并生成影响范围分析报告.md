# 在代码修改后自动更新代码上下文并生成影响范围分析报告

## 使用场景

当你修改的是接口、公共函数、跨模块逻辑或其他容易牵连多个文件的内容时，系统会在改动后自动更新代码上下文，并生成影响范围分析，帮助你判断哪里可能还需要继续检查。

## 操作步骤

1. 在项目目录中启动代码代理并提交会修改代码的任务。
2. 等待系统完成本轮修改。
3. 一旦检测到文件已被修改，系统会先自动更新当前代码上下文，再继续分析这些改动可能影响到哪些文件、符号或测试。
4. 查看终端里的影响范围分析结果，继续判断是否需要补充修改或追加验证。

## 你会看到的提示与反馈

- 这是代码修改后自动触发的，不需要单独输入命令。
- 终端会先提示正在更新代码上下文，然后提示正在进行变更影响分析。
- 分析结果会汇总到最终输出中，通常会提示可能受影响的文件、接口或测试范围。
- 如果当前修改过小、无法提取有效变更，或者分析失败，系统可能不会输出完整报告，但主流程仍会继续。

## 注意事项

- 影响范围分析是帮助你缩小复查范围，不等于已经自动完成所有验证。
- 如果你看到报告中提到了额外的受影响文件或测试，最好手动再检查这些位置。
- 只有确实发生代码改动时，这个功能才有意义；纯说明类任务不会产生这类分析结果。
