# 在代码修改后自动执行构建验证

## 使用场景

当你希望系统在代码修改完成后立刻尝试构建或编译，尽早发现明显的编译错误、构建失败或环境不匹配问题时，代码代理会在合适条件下自动执行构建验证。

## 操作步骤

1. 在项目目录中启动代码代理并提交一个会修改代码的任务。
2. 等待系统完成本轮改动和自动提交。
3. 如果当前会话允许构建验证，系统会自动开始这一步，不需要你手动再输入编译命令。
4. 查看终端中的构建验证结果，再决定是否继续修复或结束本轮任务。

## 你会看到的提示与反馈

- 构建验证开始时，终端会提示正在进行编译检查，并带上本轮涉及的文件摘要。
- 如果验证通过，最终结果中会显示构建验证通过，并标出耗时；有时还会显示识别到的构建系统类型。
- 如果构建验证被关闭或当前项目已经记住“只做基础静态检查”，系统不会继续做完整构建验证，而是走后续替代流程。
- 如果构建验证过程本身出错，主流程不会因此直接中断。

## 注意事项

- 这是代码修改后的自动验证步骤，不是在输入阶段随时手动调用的命令。
- 是否执行构建验证还取决于当前环境、配置和项目状态，不是所有仓库都一定会完整执行这一步。
- 构建验证通过并不等于所有功能都没问题，它主要用于尽快发现编译或构建层面的错误。
