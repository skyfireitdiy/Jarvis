# 在代码修改后自动执行格式化或变更后处理

## 使用场景

当你希望代码代理在修改文件后顺手完成格式化、自动修复或其他变更后处理步骤，减少你手动补跑工具的次数时，这项自动处理能力会很有帮助。

## 操作步骤

1. 启动代码代理并提交会修改代码的任务。
2. 如果后续流程进入最终提交整理阶段，系统会自动检查本轮变更文件是否匹配可用的后处理命令。
3. 对于匹配上的文件，系统会自动执行格式化或其他后处理动作。
4. 处理完成后，系统会继续整理最终提交结果。

## 你会看到的提示与反馈

- 这是自动发生的，不需要你在任务里再单独写“请再格式化一下”。
- 某个文件处理成功时，终端会提示该文件已处理，并带上所使用的工具名。
- 如果本轮有多个文件被处理，最终还会提示总共处理了多少个文件。
- 如果某个处理命令失败、超时或工具不存在，系统会给出警告，但通常不会因此直接中断整个主流程。

## 注意事项

- 这一步通常发生在最终整理提交流程中，不是每次生成一点改动就立即执行。
- 自动后处理是否会运行，取决于当前文件类型是否匹配已有规则，以及当前环境是否具备对应工具。
- 如果你的项目对格式化或自动修复有特殊要求，可以继续用自定义配置扩展这一步的行为。
