# 配置安全扫描

## 使用场景

当你准备使用 Jarvis 的安全扫描能力，但希望先明确扫描目标、语言范围、排除目录、输出格式和分析参数时，可以先配置安全扫描设置，而不是每次手工重新输入。

## 操作步骤

1. 在终端中输入 `jsec config`，并按需补充参数。例如：`jsec config --target . --languages c,cpp,rust --output-format markdown`
2. 根据你的目标，按需设置以下内容：

   - 扫描目标，可以是文件路径或目录路径
   - 语言过滤
   - 排除目录
   - 输出格式
   - 输出文件
   - 聚类限制
   - 是否启用二次验证
   - 是否强制保存记忆
   - 运行时模型组

3. 如果你只想确认当前配置，可直接运行：`jsec config --show`
4. 如果你希望恢复默认配置，可运行：`jsec config --clear`
5. 修改完成后，系统会把配置保存到本地配置文件，后续扫描命令会自动读取这些设置。

## 你会看到的提示与反馈

- 设置某个选项后，系统会回显该项已更新。
- 查看模式下，系统会显示当前配置文件路径和完整配置内容。
- 清空配置后，系统会提示配置已清空，并回显默认配置。
- 如果输出格式不是支持的类型，系统会直接报错。
- 如果你没有提供任何有效修改参数，系统会提示你补充配置参数。

## 注意事项

- 这是命令行入口，不是专家型任务入口。
- 输出格式只支持 `csv` 或 `markdown`。
- 配置会写入本地 `.jarvis/jsec/config.json`，后续 `jsec scan` 会直接使用它。
- 如果你还没配置过，首次扫描前建议先完成这一步。
