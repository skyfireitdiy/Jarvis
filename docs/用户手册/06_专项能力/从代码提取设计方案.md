# 从代码提取设计方案

## 使用场景

当你面对一个已有实现，想快速把它整理成“设计方案”而不是逐行讲代码说明时，可以使用内置代码反向设计专家，从现有代码中提取语言无关的设计内容。

## 操作步骤

1. 在任务中明确告诉系统：使用代码反向设计专家。
2. 指定需要分析的代码范围，可以是某个模块、某些文件，或一段关键实现。
3. 说明你更关注哪些方面，例如：

   - 架构与模块
   - 数据流
   - 功能点
   - 异常处理
   - 外部接口

4. 提交任务后，等待专家阅读代码并提取设计信息。
5. 让专家将结果整理为 Markdown 设计方案。
6. 如有需要，再要求按功能拆分成多份设计文档。

## 你会看到的提示与反馈

- 输出结果通常是结构化的设计方案文档，而不是源码逐行注释。
- 设计方案会覆盖模块职责、接口、数据流和关键行为。
- 内容会尽量采用语言无关表达，便于继续做新实现或跨语言重构。
- 如果范围过大或目标不明确，专家可能先要求你缩小范围或补充目标。

## 注意事项

- 这是专家型任务入口，不是固定 CLI 命令。
- 重点是提炼“设计抽象”，不是把现有代码逐句翻译成说明文。
- 如果你后续要做重实现，建议要求专家特别强调功能完整覆盖和边界条件。
- 当分析对象较大时，按模块拆分任务通常更容易得到清晰结果。
