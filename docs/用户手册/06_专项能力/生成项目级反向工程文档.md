# 生成项目级反向工程文档

## 使用场景

当你希望对整个项目做系统化反向工程，获得模块清单、依赖关系、数据流、控制流和整体架构文档时，可以通过反向工程规则发起项目级分析任务。

## 操作步骤

1. 在任务中明确说明：你要做的是“项目级反向工程”，而不是单段代码解释。
2. 指定项目根目录或需要分析的范围。
3. 要求输出内容覆盖：

   - 模块划分
   - 依赖关系
   - 数据流和控制流
   - 关键功能路径
   - 语言无关设计表达

4. 如果你想直接给出一段完整描述，可以使用类似任务：`请对当前项目做项目级反向工程，输出整体架构说明、模块划分、关键数据流和功能路径，并整理成适合新人阅读的 Markdown 文档。`
5. 提交任务后，等待系统分层阅读项目结构并整理结果。
6. 根据需要让系统输出整体架构文档、模块说明文档，或便于新成员理解的项目分析说明。

## 你会看到的提示与反馈

- 结果通常不是单一一页说明，而是可扩展的项目级分析材料。
- 输出会强调完整性、层次化和可追溯性。
- 系统可能会区分整体文档和模块文档，帮助你按层阅读项目。
- 如果你的目标更适合“单文件/单模块设计提取”，系统可能会引导你改用更小范围的分析方式。

## 注意事项

- 这是规则/专家驱动入口，不是固定 CLI 子命令。
- 在发起任务前，要先说明你要的是“项目反构”还是“代码反向设计”，两者范围不同。
- 输出目标应尽量避免语言绑定表达，这样更适合后续迁移、重写和培训新人。
- 项目越大，越建议先分模块再合并整体文档。
- 如果你只给出“分析整个项目”而不说明重点，输出通常会偏总览，难以直接满足迁移、培训或汇报等特定用途。
