# 代码漏洞分析并生成报告

## 使用场景

当你希望让 Jarvis 从安全视角深入阅读代码，并输出结构化漏洞分析报告时，可以使用内置安全漏洞分析师来完成这项任务。

## 操作步骤

1. 在任务中明确告诉系统：使用安全漏洞分析师。
2. 指定需要分析的代码范围、项目模块或重点风险区域。
3. 如有需要，可额外说明你特别关心的安全方向，例如：

   - 输入处理
   - 认证授权
   - 会话管理
   - 不安全 API 使用
   - 逻辑缺陷

4. 提交任务后，等待系统先阅读代码并理解整体架构、模块和数据流。
5. 要求系统输出完整漏洞分析报告，并包含修复建议。

## 你会看到的提示与反馈

- 报告通常会包含：

  - 问题位置
  - 问题描述
  - 严重程度
  - 触发场景
  - 影响
  - 修复建议

- 输出结果更适合使用 Markdown 归档和分享。
- 如果任务较复杂，系统可能会先拆分分析范围，再分别汇总结果。
- 分析强调基于证据，而不是无依据猜测。

## 注意事项

- 这是专家型任务入口，不是固定 CLI 子命令。
- 如果分析范围过大，建议先聚焦高风险模块，避免报告过于泛化。
- 报告价值在于“问题 + 证据 + 修复建议”的完整链条，而不仅是列出风险名称。
- 对复杂项目，分阶段分析再汇总，通常比一次性粗扫更可靠。
