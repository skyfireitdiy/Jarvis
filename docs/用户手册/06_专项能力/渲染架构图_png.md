# 渲染架构图 PNG

## 使用场景

当你已经有了 Graphviz DOT 架构图，希望把它进一步渲染为 PNG 图片，便于插入文档、分享给团队或直接查看时，可以继续让架构图专家完成渲染步骤。

## 操作步骤

1. 先确保已经生成 `.dot` 架构图文件。
2. 在任务中要求系统把该 DOT 文件渲染为 PNG 图片。
3. 如果你想直接描述任务，可以使用类似表达：`请把 docs/arch/system.dot 渲染成 PNG，输出到 docs/arch/system.png，并告诉我最终文件路径。`
4. 确认当前环境中可用 Graphviz 的 `dot` 命令。
5. 提交任务后，等待系统调用 `dot` 完成渲染；渲染完成后，检查生成的 `.png` 文件路径。

## 你会看到的提示与反馈

- 渲染成功后，系统会反馈 PNG 图片的保存路径。
- 最终通常会同时保留：

  - `.dot` 源文件
  - `.png` 图片文件

- 如果渲染失败，通常会出现与 Graphviz 环境相关的错误提示。

## 注意事项

- 这是基于已生成 DOT 图继续处理的步骤，不是从零生成架构图。
- 前提是本地环境中可用 `dot` 命令。
- 如果渲染失败，优先检查 Graphviz 是否安装完整、命令是否可执行。
- PNG 是展示产物，源 DOT 文件仍建议一并保留，便于后续调整和重新渲染。
- 最好在任务中同时写明输入 DOT 路径和期望输出 PNG 路径，避免系统需要反复确认文件位置。
