# 切换不同对话风格

## 使用场景

当你希望 Jarvis 在同样的任务中使用不同的表达风格、风险提示方式和建议呈现方式时，可以通过加载不同的人格规则来切换对话风格。

## 操作步骤

1. 在你的任务或会话中明确说明要切换到某种人格风格。
2. 根据需要选择不同人格规则，例如：

   - Jarvis 风格
   - 专业严谨风格

3. 在该人格规则生效后，再继续提出你的分析、编写或决策任务。
4. 观察系统输出的表达方式、结构和风险提示风格是否符合预期。
5. 如有需要，可再切换到另一种人格风格做对比。

## 你会看到的提示与反馈

- 切换到不同人格后，同一类任务的回复语气和组织方式会发生变化。
- Jarvis 风格通常更偏英式绅士、专业高效，并强调数据与风险提醒。
- 专业严谨风格通常更偏正式、结构化、强调规范、风险和可验证性。
- 切换后变化主要体现在表达方式，而不是底层功能能力本身。

## 注意事项

- 这是规则系统入口，不是固定 CLI 子命令。
- 人格规则改变的是交互风格，不等于更换底层模型或功能模块。
- 如果你需要更正式或更适合汇报的输出，可以优先选择专业严谨风格。
- 如果你希望同一任务做风格对比，最好在任务目标保持不变的前提下切换人格规则。
