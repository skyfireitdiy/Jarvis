# 执行安全扫描

## 使用场景

当你已经完成安全扫描配置，想直接对当前项目、某个目录或指定文件执行安全扫描并生成报告时，可以使用安全扫描命令。

## 操作步骤

1. 先确认已经通过配置命令保存好扫描参数。
2. 在终端中输入下面的命令，然后按回车：`jsec scan`
3. 这个命令本身不需要额外参数；提交后会自动读取本地安全扫描配置文件。
4. 如果配置中的目标是一个目录，系统会对该目录执行扫描。
5. 如果配置中的目标是一个文件，系统会先扫描所在目录，再过滤出该文件相关的问题结果。
6. 扫描完成后，查看终端输出，或到指定输出文件中查看报告。

## 你会看到的提示与反馈

- 如果配置文件不存在，系统会提示你先执行配置命令。
- 扫描成功后，系统会输出 Markdown 或 CSV 报告内容。
- 如果你配置了输出文件，系统会把报告写入该文件，同时通常也会在终端中显示内容。
- 当主扫描流程无法产出结果时，系统会回退到直接扫描模式继续生成报告。
- 如果目标文件不存在，系统可能退回按当前目录继续扫描。

## 注意事项

- 这个命令本身不接收完整扫描参数，而是依赖已保存的配置。
- 首次使用前应先执行“配置安全扫描”。
- 输出文件后缀会根据输出格式自动调整。
- 如果你只想分析外部 JSON 文件，不应使用这个命令，而应使用外部安全问题分析入口。
