# 研究官方文档并生成部署步骤

## 使用场景

当你要部署某个开源项目，但不希望只凭经验硬试，而是希望基于官方文档、官方仓库和社区资料整理出清晰可执行的部署步骤时，可以使用内置开源部署专家。

## 操作步骤

1. 在任务中明确告诉系统：使用开源部署专家。
2. 提供待部署的项目名称、版本、目标环境和特殊要求。
3. 要求系统优先研究：

   - 官方仓库
   - README
   - 官方部署文档
   - 社区最佳实践

4. 提交任务后，等待系统先理解项目和环境要求，再整理部署步骤。
5. 获取输出结果后，按步骤执行部署，并根据文档中的验证方式逐步检查。

## 你会看到的提示与反馈

- 系统通常会先补充确认项，例如版本、操作系统、运行环境或部署方式。
- 输出结果会包含结构化部署步骤，而不是零散建议。
- 文档中通常会附带：

  - 关键配置说明
  - 验证方法
  - 注意事项
  - 常见问题或排查建议

- 如果部署过程中遇到问题，系统会继续结合资料进行排查。

## 注意事项

- 这是专家型任务入口，不是固定 CLI 命令。
- 重点是优先参考官方资料，而不是跳过调研直接给出经验性命令。
- 目标环境越明确，输出越可执行，例如本地开发、服务器、容器或特定系统版本。
- 如果项目本身文档分散，部署专家通常会先帮助你汇总入口资料。
