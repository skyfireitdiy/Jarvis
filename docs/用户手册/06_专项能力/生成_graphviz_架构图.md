# 生成 Graphviz 架构图

## 使用场景

当你希望让 Jarvis 阅读一段代码或一个项目，并输出可直接渲染的 Graphviz 架构图时，可以使用内置架构图生成专家发起任务。

## 操作步骤

1. 在任务中明确告诉系统：你要使用架构图生成专家。
2. 说明你要分析的代码范围，以及希望输出的视图类型，例如：

   - 逻辑视图
   - 实现视图
   - 进程视图
   - 部署视图
   - 用例视图

3. 如果你还不确定视图类型，先让专家帮助你澄清目标。
4. 提交任务后，等待专家完整阅读相关代码并识别核心组件与关系。
5. 让专家把结果输出为 Graphviz DOT 文件。
6. 生成完成后，再根据需要继续渲染 PNG 图片。

## 你会看到的提示与反馈

- 如果你的需求没有说明视图类型，专家通常会先追问或澄清。
- 输出结果会是可直接保存和渲染的 `.dot` 内容。
- 图中会突出组件职责、关系和层次，而不是直接堆叠源码细节。
- 最终你会得到一份适合继续渲染或归档的架构图源文件。

## 注意事项

- 这是专家型任务入口，不是固定 CLI 子命令。
- 产出重点是抽象架构表达，而不是把函数名、文件路径原样画进图里。
- 如果分析范围太大，建议先指定模块或目标视图，避免输出过于笼统。
- 生成 `.dot` 后，如需图片，还需要继续执行渲染步骤。
