# 重置并重跑迁移流水线

## 使用场景

当你已经执行过一次 C→Rust 迁移流水线，但希望清除已有阶段进度、重新从扫描阶段开始完整执行时，可以使用重置并重跑的方式。

## 操作步骤

1. 在终端中输入下面的命令，然后按回车：`jc2r run --reset`
2. 如果你还想同时指定模型组、最大重试次数或交互模式，也可以组合使用，例如：`jc2r run --reset --llm-group 你的模型组名 --max-retries 3 --interactive`
3. 状态清除后，流水线会重新从扫描阶段开始执行。
4. 按正常流程等待扫描、库替换、准备、转译和优化阶段依次完成。
5. 重跑完成后，再根据结果决定是否继续执行功能对齐验证。

## 你会看到的提示与反馈

- 重置时，系统会删除旧的阶段状态记录。
- 后续执行中，不会再把之前完成的阶段当作可跳过阶段。
- 你会看到整个流水线从头开始重新输出各阶段状态。
- 最终完成后，会重新给出新的整体完成提示。

## 注意事项

- 重置的是流水线进度状态，不等于自动恢复你之前生成的代码、文件或外部产物。
- 如果你只是想继续未完成的流程，不需要加重置选项。
- 重跑前最好确认当前工作区和迁移产物状态，避免把旧结果和新流程混在一起。
- 这是完整重新执行流程的入口，不只是“重新执行失败阶段”。
