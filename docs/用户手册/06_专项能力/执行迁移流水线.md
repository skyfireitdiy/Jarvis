# 执行迁移流水线

## 使用场景

当你已经准备好 C→Rust 迁移配置，希望按固定阶段自动执行从扫描到转译、优化的完整流水线时，可以使用迁移流水线命令。

## 操作步骤

1. 先完成迁移配置，确保根符号、禁用库和附加说明已准备好。
2. 在终端中输入下面的命令，然后按回车：`jc2r run`
3. 根据需要设置模型组、最大重试次数，以及是否使用交互模式，例如：`jc2r run --llm-group 你的模型组名 --max-retries 3 --interactive`
4. 提交命令后，系统会按固定顺序执行流水线阶段：

   - 扫描
   - 库替换
   - 准备
   - 转译
   - 优化

5. 如果之前已经执行过一部分，系统会根据状态文件自动跳过已完成阶段。
6. 等待所有阶段完成后，再检查整体输出结果和后续验证需求。

## 你会看到的提示与反馈

- 每个阶段开始时，系统会显示正在执行的阶段。
- 已完成的阶段会被识别并跳过，而不是重复执行。
- 每个阶段结束后，会显示完成或跳过状态。
- 优化阶段完成后，系统会输出摘要，例如扫描文件数、清理情况、补充说明和检查次数。
- 流水线状态会写入本地状态文件，用于后续续跑。
- 全部阶段完成后，会给出整体完成提示。

## 注意事项

- 这是命令行流水线入口，不会在运行时重新录入根符号等基础配置。
- 如果你想从头重跑，应使用带重置的运行方式，而不是直接再次执行普通 run。
- 断点续跑依赖状态文件，因此状态文件会影响系统判断哪些阶段需要跳过。
- 优化阶段只有在判定真正完成后，才会被标记为已完成。
