# 验证功能对齐

## 使用场景

当你已经完成 C→Rust 迁移，希望进一步确认转译后的 Rust 与原始 C 在接口、逻辑、边界处理和错误处理上是否保持一致时，可以执行功能对齐验证。

## 操作步骤

1. 先完成迁移流水线，确保已经产出可验证的 Rust 结果。
2. 在终端中输入下面的命令，然后按回车：`jc2r verify`
3. 根据需要设置模型组、最大迭代次数，以及是否采用交互模式，例如：`jc2r verify --llm-group 你的模型组名 --max-iterations 5 --interactive`
4. 提交命令后，系统会对原始实现和转译结果进行对齐检查。
5. 如果发现不一致，系统可能进入持续迭代优化，直到没有问题或达到最大迭代次数。
6. 查看最终验证结果，并据此决定是否继续人工检查或再次优化。

## 你会看到的提示与反馈

- 验证过程中，系统会输出当前检查进展。
- 检查内容会覆盖接口、类型、逻辑、边界情况、错误处理和内存安全等方面。
- 如果发现不一致，系统会继续进入后续优化与再验证循环。
- 成功时会输出验证结果；失败时会显示明确错误提示并退出。

## 注意事项

- 这一步通常应在 `run` 完成后再执行，不适合作为第一步直接运行。
- 这不是简单的文本比较，而是强调功能和行为层面对齐。
- 如果项目较复杂，验证过程可能会进行多轮检查与修正。
- 最大迭代次数会影响系统愿意持续优化到什么程度。
