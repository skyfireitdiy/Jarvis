# 生成并保存脚本

## 使用场景

当你已经明确要自动化什么任务，希望让 Jarvis 根据你的需求、运行环境和输入输出要求，生成一份完整可执行并保存到指定位置的脚本时，可以使用内置脚本生成专家。

## 操作步骤

1. 在任务中明确告诉系统：使用脚本生成专家。
2. 说明你的脚本目标，包括：

   - 要解决的问题
   - 运行环境
   - 输入输出方式
   - 希望保存到哪里

3. 如有需要，再说明你偏好的脚本语言或限制条件。
4. 提交任务后，等待系统选择合适语言并设计脚本结构。
5. 让系统把完整脚本写入指定文件，并同时给出必要说明。
6. 生成后先自行运行一次，验证脚本是否符合预期。

## 你会看到的提示与反馈

- 如果需求不清晰，系统通常会先向你提问澄清。
- 脚本生成完成后，你会得到：

  - 脚本文件
  - 关键说明或摘要

- 输出结果会尽量保证脚本是完整且可运行的，而不是只给片段。
- 如果你提供了保存路径，系统会按要求保存到目标位置。

## 注意事项

- 这是专家型任务入口，不是固定 CLI 子命令。
- 脚本目标越具体，生成结果越容易一次到位。
- 规则强调脚本应可执行、可维护，并包含必要错误处理。
- 生成后仍建议你在真实环境中做一次最小验证，确认输入输出和权限符合预期。
