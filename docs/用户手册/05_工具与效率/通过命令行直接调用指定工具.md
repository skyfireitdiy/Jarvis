# 通过命令行直接调用指定工具

## 使用场景

当你已经知道某个工具的名称和参数结构，想绕过其他界面，直接在命令行中执行该工具并查看执行结果时，可以使用工具系统的调用命令。

## 操作步骤

1. 先确认目标工具名称和参数要求。
2. 在终端中输入调用命令，并指定工具名称，例如：`jt call 工具名`。
3. 准备参数时，你可以二选一：

   - 直接通过 `--args` 传入 JSON 字符串
   - 通过 `--args-file` 从 JSON 文件读取参数

4. 如果工具没有必填参数，也可以只传工具名称直接执行。
5. 提交命令后，等待工具执行完成并查看结果。
6. 若执行失败，根据终端中的缺参提示、JSON 格式错误提示或工具错误输出修正后再试。

## 你会看到的提示与反馈

- 工具执行后，系统会明确显示：

  - 工具执行成功
  - 或工具执行失败

- 如果工具有标准输出，会显示“输出”内容。
- 如果工具有错误输出，会显示“错误”内容。
- 如果 `--args` 里的内容不是合法 JSON，会直接提示“参数必须是有效的 JSON 格式”。
- 如果缺少必填参数，系统会明确列出：

  - 缺少哪些参数
  - 每个必填参数的说明

- 如果工具名称不存在，系统会直接提示该工具不存在。

## 注意事项

- `--args` 和 `--args-file` 都要求参数内容是合法 JSON。
- 当参数较长、层级较深时，更适合先写到 JSON 文件，再用 `--args-file` 读取。
- 如果工具执行失败，不一定是工具系统本身有问题，也可能是你传入的业务参数不正确。
- 调用命令会真正执行目标工具，因此在涉及写文件、网络请求或状态变更的工具时要先确认影响范围。
