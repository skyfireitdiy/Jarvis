# 为 Git 提交信息添加前缀

## 使用场景

当你所在的团队要求提交信息带上统一前缀，或者你想为某次提交增加环境标记、模块标记、任务类型标记时，可以在自动生成提交信息时追加前缀。

## 操作步骤

1. 先准备好当前 Git 仓库中的改动。
2. 运行自动生成 Git 提交信息的命令，例如：`jgc --prefix "你的前缀"`。
3. 把你希望附加在提交信息前面的文本写到 `--prefix` 后面。
4. 提交命令后，系统会先生成主体提交信息，再自动把你的前缀拼接到前面。
5. 等待提交完成，并检查最终输出的提交信息是否符合预期。

## 你会看到的提示与反馈

- 如果有可提交改动，系统会继续生成提交信息并执行提交。
- 提交成功后，终端会显示最终提交信息。
- 最终显示出来的提交信息中，你设置的前缀会出现在最前面。
- 如果当前没有未提交改动，系统仍会直接提示“没有未提交的更改”。

## 注意事项

- 前缀是在自动生成的主体信息之前附加，不会替代系统生成的主描述。
- 如果你们团队对前缀格式有固定要求，建议先统一命名方式后再使用。
- 前缀内容过长会让提交标题变得臃肿，建议尽量简短。
- 前缀功能依赖自动提交流程本身，因此它不是一个单独的独立命令。
