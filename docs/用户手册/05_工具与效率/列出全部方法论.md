# 列出全部方法论

## 使用场景

当你想先看看当前环境中已经有哪些可复用的方法论，再决定是否导出、导入或继续补充时，可以先列出全部方法论标题。

## 操作步骤

1. 在终端中运行方法论列表命令。
2. 根据需要选择查看作用域：

   - global
   - project
   - all

3. 如果你查看的是 project 作用域，请确保当前目录位于 Git 仓库中。
4. 提交命令后，查看输出的方法论标题列表。
5. 如有需要，再根据结果继续做导出、导入或提取操作。

## 你会看到的提示与反馈

- 正常情况下，系统会列出当前作用域下可用的方法论标题，并按序号显示。
- 如果当前作用域没有任何方法论，系统会明确提示为空。
- 如果你查看 project 作用域但当前不在 Git 仓库中，系统会提示无法获取项目级上下文或直接结束。
- 如果 scope 参数无效，系统会直接提示错误。

## 注意事项

- 这个命令主要用于查看“有哪些方法论”，不是查看完整正文内容。
- `all` 适合快速总览；如果你只关注某个项目经验，建议直接看 project。
- project 作用域依赖 Git 仓库环境。
- 列出方法论不会修改任何内容，适合先做盘点再进行其他操作。
