# 从 URL 内容提取方法论

## 使用场景

当你希望把某个网页、在线文章或外部说明中的经验内容沉淀为本地方法论时，可以直接使用 URL 作为输入进行提取。

## 操作步骤

1. 准备你要提取的方法论来源 URL。
2. 在终端中运行从 URL 提取方法论的命令，例如：`jm extract-url "https://example.com/article" --scope global`。
3. 选择保存作用域：

   - global
   - project

4. 如果选择 project 作用域，请确认当前目录位于 Git 仓库中。
5. 提交命令后，系统会根据该 URL 提取方法论内容，并尝试整理成结构化结果。
6. 提取成功后，结果会保存到指定作用域，并与当前已有方法论合并。

## 你会看到的提示与反馈

- 提取开始时，系统会提示正在执行提取。
- 提取成功后，会显示成功提取并保存的方法论数量。
- 如果返回内容格式不符合要求，系统会提示“响应格式无效”。
- 如果解析阶段失败，会提示“YAML 解析失败”或其他明确错误信息。
- 如果作用域参数错误，或 project 作用域缺少 Git 仓库上下文，系统也会直接报错。

## 注意事项

- 这是“从 URL 内容提取方法论”，不是简单保存链接。
- 如果最终返回内容格式不正确，系统不会静默忽略，而是会直接提示失败。
- project 作用域仍然要求当前处于 Git 仓库中。
- 由于这是提取型操作，建议在保存后再用“列出全部方法论”或导出命令检查结果。
