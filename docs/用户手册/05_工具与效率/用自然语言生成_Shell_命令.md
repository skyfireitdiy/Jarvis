# 用自然语言生成 Shell 命令

## 使用场景

当你知道自己“想做什么”，但不确定应该写什么 Shell 命令时，可以直接输入一句自然语言，让系统返回对应的命令建议。

## 操作步骤

1. 如果你已经想好了需求，可以直接运行：`jss request "列出当前目录下所有 Python 文件"`。
2. 这种写法会把自然语言需求直接作为参数传入，适合快速生成一条建议命令。
3. 如果你没有在命令后直接提供请求内容，也可以只输入：`jss request`。
4. 运行 `jss request` 后，系统会提示你现场输入需求；输入完成后，它会生成命令，并继续进入执行确认流程。
5. 无论是哪种方式，系统都会结合你的描述、当前路径和当前 shell 环境生成命令建议。
6. 如果你是通过某些 Shell 的“命令未找到”处理器间接触发这项能力，后续是否立即执行，还会受当前 Shell 交互方式影响。

## 你会看到的提示与反馈

- 正常情况下，系统返回的结果应只包含命令本身。
- 如果需要组合多个命令，结果通常会使用 `&&` 连接。
- 生成时会结合当前目录和当前 shell 类型，因此同一句描述在不同环境中可能得到不同命令。
- 如果你运行的是 `jss request`（不带参数），系统会先提示你输入需求，再生成命令，并继续进入执行确认流程。
- 如果你运行的是 `jss request "..."`（直接带参数），更适合把输出当成建议命令查看。
- 如果你是从不同 Shell 的智能处理器间接触发，后续交互可能表现为回填、可编辑或直接执行。

## 注意事项

- 直接传入自然语言参数时，更适合把结果当成建议命令查看；不带参数运行 `jss request` 时，则会进入交互输入并继续走执行确认流程。
- 这条命令的输出目标是“返回命令本身”，不是解释说明。
- 如果你是通过 bash、zsh 或 fish 的智能处理器间接使用，它们的后续交互方式并不完全一样，应以对应 Shell 文档中的说明为准。
- 如果描述过于模糊，生成结果可能也会偏泛，建议尽量把目标写清楚。
