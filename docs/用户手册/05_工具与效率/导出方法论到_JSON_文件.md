# 导出方法论到 JSON 文件

## 使用场景

当你想备份当前方法论、分享给其他环境，或者把某个作用域中的方法论单独导出时，可以把方法论导出为一个 JSON 文件。

## 操作步骤

1. 在终端中运行方法论导出命令。
2. 指定导出文件路径。
3. 选择导出作用域：

   - global：仅导出全局方法论
   - project：仅导出当前项目方法论
   - all：导出全部方法论

4. 如果你要导出 project 作用域，请确认当前目录位于 Git 仓库中。
5. 提交命令后，等待系统把对应作用域的方法论写入 JSON 文件。
6. 导出完成后，检查目标文件是否已生成。

## 你会看到的提示与反馈

- 导出成功后，系统会提示成功导出的数量和目标文件路径。
- 使用 `all` 时，会把当前可获取的方法论整体写入一个文件。
- 如果 scope 参数无效，系统会明确提示参数错误。
- 如果你选择 project 作用域但当前不在 Git 仓库中，系统会提示无法获取项目级方法论目录。
- 如果目标路径不可写，系统会提示导出失败原因。

## 注意事项

- 导出生成的是 JSON 文件，更适合备份、迁移和共享。
- `all` 适合做完整备份；如果你只想迁移当前项目经验，通常更适合用 project。
- project 作用域必须有 Git 仓库上下文。
- 导出不会删除当前环境中的任何方法论，只会生成一个额外文件。
