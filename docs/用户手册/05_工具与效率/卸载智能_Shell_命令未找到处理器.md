# 卸载智能 Shell 命令未找到处理器

## 使用场景

当你不再希望 Shell 自动拦截“命令未找到”并生成建议命令，或者想恢复原始 shell 行为时，可以卸载已经安装的智能 Shell 处理器。

## 操作步骤

1. 在终端中运行 JSS 的卸载命令。
2. 指定你要卸载的 shell 类型：

   - fish
   - bash
   - zsh

3. 提交命令后，系统会从对应 shell 的配置文件中删除之前安装的 JSS 配置片段。
4. 卸载完成后，按照提示重新加载对应 shell 配置文件。
5. 重新加载后，Shell 会恢复普通的“命令未找到”行为。

## 你会看到的提示与反馈

- 卸载成功后，系统会提示对应 shell 的 completion 已卸载。
- 同时会提示重新 source 对应配置文件。
- 如果目标配置文件不存在，系统会直接提示没有可卸载内容。
- 如果之前根本没有安装过，系统也会明确提示未安装，而不是误报成功。
- 如果你指定了不支持的 shell 类型，系统会直接报错。

## 注意事项

- 卸载是按 shell 类型分别执行的，不会一次同时清理所有 shell。
- 卸载后如果不重新加载配置，当前终端会话中可能仍保留旧行为。
- 只支持 fish、bash、zsh 三种 shell 类型。
- 如果你之后又想恢复该能力，可以重新执行对应 shell 的安装命令。
