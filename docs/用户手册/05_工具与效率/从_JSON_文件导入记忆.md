# 从 JSON 文件导入记忆

## 使用场景

当你已经有一份导出的记忆 JSON 文件，想把这些长期记忆恢复到当前环境，或者从别的环境迁移回来时，可以使用记忆导入命令。

## 操作步骤

1. 准备一份合法的记忆 JSON 文件。
2. 在终端中运行记忆导入命令，并指定该文件路径。
3. 如有需要，可开启覆盖模式，用于覆盖当前已存在的同 ID 记忆。
4. 提交命令后，系统会读取文件中的记忆列表，并按记忆类型写入对应位置。
5. 导入结束后，查看终端中的成功、跳过或失败提示。

## 你会看到的提示与反馈

- 导入成功后，系统会提示已导入的记忆数量。
- 如果导入文件中某些记录的记忆类型无法识别，这些记录会被跳过。
- 如果文件不存在、不是合法 JSON，或者格式不符合预期，系统会明确提示失败原因。
- 未开启覆盖模式时，遇到同 ID 的已存在记忆，系统会跳过这些记录，而不是直接覆盖。
- 开启覆盖模式时，已有同 ID 记忆会被新数据替换。

## 注意事项

- 导入文件应当是“记忆列表”格式，而不是任意 JSON 结构。
- 覆盖模式适合做恢复或同步；如果你不确定是否需要替换现有数据，建议先不要开启覆盖。
- 导入会真正写入当前环境的长期记忆目录，属于状态变更操作。
- 导入前最好保留一份原始备份，避免误覆盖后无法恢复。
