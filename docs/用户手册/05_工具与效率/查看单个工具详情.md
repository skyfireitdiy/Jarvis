# 查看单个工具详情

## 使用场景

当你已经知道某个工具的名称，但还不清楚它具体做什么、需要哪些参数、参数结构长什么样时，可以单独查看这个工具的详细说明。

## 操作步骤

1. 先通过“列出所有可用工具”确认目标工具名称。
2. 在终端中输入类似命令：`jt show 工具名`。
3. 如需人工阅读，直接按回车运行默认输出即可。
4. 如果你需要把结果交给脚本、自动化流程或其他系统处理，可以改用 JSON 输出模式。
5. 确认参数结构后，再决定是否继续直接调用这个工具。

## 你会看到的提示与反馈

- 默认模式下，终端会显示该工具的详细信息，包括：

  - 工具名称
  - 描述
  - 协议版本
  - 参数定义

- 如果这个工具没有参数，界面会明确显示“无参数”。
- 使用 JSON 模式时，会输出完整的结构化工具信息。
- 如果工具名称不存在，系统会明确提示该工具不存在，并直接退出。

## 注意事项

- 工具名称必须和系统中已注册名称一致，输错名称会直接报错。
- 默认输出更适合人读；JSON 输出更适合做进一步处理。
- 如果你只想确认某几个必填参数，建议重点查看参数定义中的 required 和 properties 部分。
- 查看详情不会执行工具本身，只是帮助你在调用前确认输入方式。
