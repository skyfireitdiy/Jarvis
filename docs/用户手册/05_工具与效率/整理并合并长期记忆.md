# 整理并合并长期记忆

## 使用场景

当你的长期记忆越来越多、主题相近、标签重复，导致查找和复用变得混乱时，可以使用记忆整理命令，把相关长期记忆整理并合并成更综合的结果。

## 操作步骤

1. 在终端中运行长期记忆整理命令，例如：`jmo organize --memory-type project_long_term`。
2. 根据需要把 `--memory-type` 设为你要整理的类型：

   - 项目长期记忆
   - 全局长期记忆

3. 如果你想先预演而不真正修改数据，可以先开启 dry-run 模式，只看将要合并哪些记忆。
4. 正式执行时，系统会扫描当前类型下的长期记忆，并找出适合合并的记忆组。
5. 对每一组待整理的记忆，系统会生成新的综合记忆，并替换掉原来的多条重复记忆。
6. 执行结束后，查看终端中的统计信息和处理结果。

## 你会看到的提示与反馈

- 开始前，系统会提示当前正在整理哪一类记忆。
- 如果当前类型下没有记忆，系统会直接提示没有可处理内容。
- 检测到可合并内容时，系统会先列出准备合并的记忆组。
- 使用 dry-run 时，你能看到“将会合并哪些记忆”，但不会真正修改文件。
- 正式执行时，成功后会显示处理组数、合并数量和新建记忆数量等统计信息。
- 如果某些记忆文件本身损坏或无法读取，系统会直接提示读取失败的文件信息。
- 如果某一组合并失败，系统会提示该组跳过，而不会让整个过程无提示终止。

## 注意事项

- 整理命令面向长期记忆，不是临时记忆操作。
- 如果你对结果不确定，建议先用 dry-run 预演。
- 合并后原始记忆会被替换为新的综合记忆，因此正式执行前应确认你接受这种整理方式。
- 项目长期记忆和全局长期记忆是分开整理的，操作前先选对类型。
