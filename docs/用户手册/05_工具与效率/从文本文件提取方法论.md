# 从文本文件提取方法论

## 使用场景

当你手头已经有一份经验总结、复盘记录、操作说明或其他文本材料，想把它整理成可复用的方法论并保存下来时，可以使用“从文本文件提取方法论”命令。

## 操作步骤

1. 准备一份文本文件，内容可以是经验总结、流程文档或问题解决记录。
2. 在终端中运行提取方法论命令，例如：`jm extract 你的文本文件.txt --scope global`。
3. 选择保存作用域：

   - global
   - project

4. 如果选择 project 作用域，请确认当前目录位于 Git 仓库中。
5. 提交命令后，系统会读取文本内容，并尝试提取出结构化方法论。
6. 提取成功后，系统会把结果保存到指定作用域，并与现有方法论合并。

## 你会看到的提示与反馈

- 开始时，系统会提示正在提取方法论。
- 如果提取成功，会显示“提取到有效方法论”，以及成功保存的数量和目标位置。
- 如果返回结果格式不正确，系统会提示“响应格式无效”。
- 如果 YAML 解析失败，会明确提示“YAML 解析失败”以及具体错误信息。
- 如果整体提取流程异常，也会显示提取失败原因。

## 注意事项

- 输入必须是文本文件，不是 JSON 方法论导入文件。
- project 作用域依赖 Git 仓库；如果当前不在仓库中，这个作用域不可用。
- 提取结果会和现有方法论合并，同名内容会被新结果覆盖。
- 文本材料越清晰、越聚焦，通常越有利于得到可复用的方法论结果。
