# 自动生成 Git 提交信息

## 使用场景

当你已经完成一批代码或文档修改，想快速得到一条中文提交信息并直接完成 Git 提交时，可以使用自动生成提交信息的命令。

## 操作步骤

1. 在 Git 仓库中完成你的改动。
2. 在终端中进入仓库目录，输入：`jgc`。
3. 如果当前目录不是仓库根目录，也可以在仓库内其他位置执行，工具会尝试定位仓库根目录。
4. 提交前，工具会检查当前工作区是否存在未提交变更。
5. 如果存在新文件，按提示确认是否将它们加入提交范围。
6. 工具会自动整理改动、生成提交信息，并执行实际提交。
7. 提交完成后，查看终端返回的提交结果。

## 你会看到的提示与反馈

- 如果仓库没有未提交改动，系统会直接提示“没有未提交的更改”，并结束流程。
- 有改动时，系统会根据当前差异内容自动生成提交信息；如果改动内容过大，生成结果可能会更多依赖修改文件范围，因此描述会更偏概括。
- 提交成功后，会显示：

  - 新提交的哈希
  - 最终使用的提交信息

- 如果提交失败，系统会明确显示失败信息，而不是静默结束。

## 注意事项

- 这是会真正执行 `git commit` 的命令，不只是预览提交信息。
- 提交前最好先确认当前改动范围，避免把不想提交的内容一并带入。
- 如果改动量特别大，系统生成的提交信息可能更偏概括性。
- 如果自动提交在最后失败，工具会报错，并尽量降低暂存区状态被打乱的风险。
- 如果你希望控制最终提交文本格式，可以进一步配合前缀或后缀选项使用。
