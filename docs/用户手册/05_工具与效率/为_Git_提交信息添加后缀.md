# 为 Git 提交信息添加后缀

## 使用场景

当你希望在自动生成的提交信息后补充任务编号、额外说明、关联备注或其他附加信息时，可以在提交时追加后缀。

## 操作步骤

1. 先准备好当前 Git 仓库中的改动。
2. 运行自动生成 Git 提交信息的命令，例如：`jgc --suffix "你的后缀"`。
3. 把你希望附加在提交信息末尾的文本写到 `--suffix` 后面。
4. 提交命令后，系统会先生成提交主体，再把你的后缀追加到末尾。
5. 提交完成后，检查最终提交信息是否已带上后缀内容。

## 你会看到的提示与反馈

- 正常有改动时，系统会继续完成自动提交流程。
- 提交成功后，终端会显示最终提交信息。
- 你设置的后缀会出现在生成提交信息的末尾。
- 如果提交失败，系统会显示失败信息；如果无改动，仍会提示“没有未提交的更改”。

## 注意事项

- 后缀适合补充额外上下文，但不适合替代主体提交说明。
- 如果你同时设置前缀和后缀，系统会在生成主体两侧分别拼接它们。
- 后缀过长时，提交信息可读性会下降，建议只保留必要信息。
- 这项能力依附于自动生成并提交的主流程，不是单独执行的独立命令。
