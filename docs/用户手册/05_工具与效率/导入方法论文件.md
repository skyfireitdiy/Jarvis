# 导入方法论文件

## 使用场景

当你已经准备好一份方法论 JSON 文件，想把其中的方法论导入到当前环境，供后续复用或共享时，可以使用方法论导入命令。

## 操作步骤

1. 准备一份合法的方法论 JSON 文件。
2. 在终端中运行方法论导入命令，并指定输入文件路径。
3. 选择导入作用域：

   - global：导入到全局方法论
   - project：导入到当前项目的方法论

4. 如果你选择 project 作用域，请确保当前目录位于 Git 仓库中。
5. 提交命令后，系统会读取文件内容，并与当前已有方法论合并。
6. 导入完成后，查看成功提示和目标保存位置。

## 你会看到的提示与反馈

- 导入成功后，系统会提示成功导入的方法论数量，以及写入位置。
- 新导入内容会与现有方法论合并；同名方法论会用新内容覆盖旧内容。
- 如果作用域参数无效，系统会直接提示 scope 参数错误。
- 如果你选择 project 作用域但当前不在 Git 仓库中，系统会明确提示无法获取项目级方法论目录。
- 如果输入文件读取失败或内容格式不正确，系统会提示导入失败。

## 注意事项

- 方法论导入文件应是合法 JSON，而不是 Markdown 或普通文本。
- global 和 project 是两套不同作用域，导入前先确认写入目标。
- project 作用域依赖 Git 仓库上下文，脱离仓库时不能使用。
- 导入采用合并策略，不会先清空现有方法论再重建。
