# 管理输入缓冲区和历史输入

## 使用场景

当 Agent 现在还没有进入等待输入状态，但你已经提前想好了后续要补充的内容，或者你想重复使用之前输入过的一段文字时，可以使用 Web 界面的输入缓冲区和历史输入功能。

## 操作步骤

### 查看和使用缓冲区

1. 先切换到目标 Agent。
2. 观察输入区附近是否出现“缓冲区有内容，点击管理”之类的提示。
3. 如果出现这条提示，说明当前 Agent 已经存在可管理的缓存内容。
4. 点击这条提示，进入缓冲区管理面板。

### 打开并管理缓冲区

1. 当缓冲区里已经有内容时，输入区附近会显示“缓冲区有内容，点击管理”。
2. 点击这条提示，打开“输入缓存”面板。
3. 在面板中，你可以执行以下操作：

   - 点击“↙ 加载”把缓存内容放回输入框
   - 点击删除按钮清空缓存
   - 直接在缓存编辑区修改内容
   - 点击“保存修改 (Ctrl+Enter)”保存当前改动

4. 如果你只是想关闭面板，点击面板右上角关闭按钮即可。

### 使用历史输入

1. 在多行输入框中，把光标放在第一行时按上方向键，可以向前翻阅更早的输入记录。
2. 把光标放在最后一行时按下方向键，可以向后返回较新的输入记录。
3. 如果你已经翻到旧记录，再一路返回到最新状态，页面会恢复你翻阅前正在编辑但尚未发送的临时内容。

## 你会看到的提示与反馈

- 缓冲区有内容时，输入区附近会显示“缓冲区有内容，点击管理”。
- 在某些状态下，发送按钮文案可能显示为“发送缓冲区”。
- 缓冲区面板标题会显示为“📝 输入缓存”。
- 面板中会看到“↙ 加载”“保存修改 (Ctrl+Enter)”等操作按钮。
- 清空缓冲区后，相关提示会消失。
- 历史输入会按你最近的输入顺序保存，方便重复提问或重复下达相似指令。

## 注意事项

- 输入缓冲区是按 Agent 分开的；你切换到不同 Agent 时，看到的是不同的缓存内容。
- 用户界面明确提供了缓冲区提示、管理面板和发送缓冲区入口；至于更底层的触发时机，应以你当前页面实际显示的状态和按钮为准。
- 历史输入主要针对多行输入框；单行确认场景更适合直接快速回答。
