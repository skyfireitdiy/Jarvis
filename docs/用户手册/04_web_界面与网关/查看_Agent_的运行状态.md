# 查看 Agent 的运行状态

## 使用场景

当你想判断某个 Agent 现在是在继续处理任务、正在等待你补充多行内容、正在等你确认，还是已经结束运行时，可以通过 Web 界面的状态展示快速判断下一步该怎么做。

## 操作步骤

1. 先进入已经连接成功的 Web 页面。
2. 在左侧 Agent 列表中查看每个 Agent 名称右侧的状态文字。
3. 如果你已经切换到某个 Agent，也可以在顶部当前 Agent 信息区域再次查看它的状态。
4. 根据状态决定下一步动作：

   - 如果显示“运行中”，通常表示 Agent 正在继续处理任务
   - 如果显示“运行中（等待多行输入）”，说明现在适合在底部多行输入框补充内容
   - 如果显示“运行中（等待确认）”，底部输入区会切换成单行输入框，适合快速回答一句话或提交短内容
   - 如果显示“已完成”，说明这个 Agent 当前已结束运行，适合查看历史结果

5. 如果消息区另外出现确认框，再按确认框里的按钮决定继续或取消；这与底部单行输入状态是两种不同场景。

## 你会看到的提示与反馈

- 状态文字会直接出现在 Agent 列表中。
- 当前 Agent 的状态也会出现在页面顶部。
- 当 Agent 正在正常处理时，底部输入区上方会显示“Agent 正在思考...”。
- 当状态切换为等待多行输入时，“完成”和“@”补全按钮会变为可用。
- 当状态切换为“运行中（等待确认）”时，底部输入区会切换为单行输入框。
- 如果系统需要你在消息区做确认操作，会另外出现确认框，并提供“确认”或“取消”按钮。
- 当 Agent 完成后，状态会变成“已完成”，你仍可以继续查看它的历史消息。

## 注意事项

- “运行中”不一定表示此刻能立即输入；只有进入“等待多行输入”时，页面才真正处于等待多行补充的阶段。
- “运行中（等待确认）”会把底部输入区切换成单行输入，不适合继续输入长文本。
- 消息区确认框是另一类独立提示；只有页面真的弹出确认框时，你才需要点击“确认”或“取消”。
- 已完成的 Agent 不代表会从列表中消失；它通常仍会保留在列表里，供你后续查看历史。
- 如果状态长时间没有更新，优先检查当前页面顶部是否仍显示“已连接”。
