# 使用独立终端面板

## 使用场景

当你希望在 Web 界面里保留一个或多个独立终端，用来执行额外命令、查看输出，或者与消息区并行工作时，可以使用独立终端面板，而不必只依赖 Agent 任务中的嵌入式执行输出。

## 操作步骤

1. 先确保当前页面已经连接到 Web Gateway。
2. 点击顶部或移动端工具栏中的“终端面板”按钮。
3. 打开终端面板后，如果当前还没有终端，会看到“暂无终端，点击 + 创建”。
4. 点击终端面板中的“+”按钮，新建一个终端。
5. 创建后，页面会自动切换到这个新终端标签。
6. 如果已经有多个终端，可以点击不同标签在它们之间切换。
7. 如需关闭某个终端，点击对应标签上的关闭按钮。
8. 如需暂时隐藏终端面板，点击终端面板右上角关闭按钮；后续再次打开时，已有终端内容可以继续恢复显示。

## 你会看到的提示与反馈

- 打开终端面板后，面板标题会显示为“终端”。
- 新建终端后，会出现一个新的终端标签页。
- 终端标签会显示当前解释器名称。
- 终端面板中可以实时看到输出内容。
- 如果某个终端已关闭，页面会同步清理对应标签，而不会一直保留无效会话。

## 注意事项

- 独立终端面板和消息区中的执行终端不是同一个概念；这里说的是可单独保留、可多标签切换的终端面板。
- 如果主连接断开，页面会清理独立终端会话，因此你需要先恢复网关连接后再继续使用。
- 终端面板支持拖动、缩放、最大化和还原；移动端与桌面端的展示方式会有差异。
- 后端会限制独立终端会话数量，因此不适合无限制一直创建新终端。
