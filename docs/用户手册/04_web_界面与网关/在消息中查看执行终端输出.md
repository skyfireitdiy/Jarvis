# 在消息中查看执行终端输出

## 使用场景

当 Agent 执行某些任务时，网页消息区除了普通文本回复，还可能嵌入终端输出。这个能力适合你实时查看命令执行过程、标准输出和错误输出，并在任务结束后继续回看这段终端历史。

## 操作步骤

1. 切换到正在执行任务的 Agent。
2. 在消息区中查看新的执行消息。
3. 当某次执行开始产生终端流输出时，消息区里会出现一个嵌入式终端区域。
4. 在执行进行期间，直接在这个区域里观察输出滚动。
5. 当执行结束后，继续留在当前会话中查看结果；原来的实时终端会转成历史输出块。
6. 如果你之后刷新页面、重新进入同一个 Agent，或再次加载它的历史消息，也可以继续查看已保存的终端历史。

## 你会看到的提示与反馈

- 执行进行中时，消息区里会出现终端样式的嵌入区域。
- 执行结束后，这个区域会变成一个历史展示块，标题类似“Terminal Output”。
- 已结束的终端历史会保留完整文本，方便之后回看。
- 如果某个 Agent 有较多历史消息，向上滚动消息区时，旧的执行终端历史也会随着历史消息一起被加载出来。

## 注意事项

- 这里说的是消息区中的执行终端，不是顶部按钮打开的独立终端面板，两者用途不同。
- 未结束的执行终端主要用于实时查看；真正保存到历史中的，是执行结束后整理出的终端文本内容。
- 如果你切换到一个已经完成的 Agent，通常只能查看它保留下来的终端历史，而不是继续与该执行实时交互。
- 如果当前页面连接中断，实时终端流会受影响；恢复连接后，你仍可以优先查看已经保存的历史内容。
