# 恢复 Agent 会话

## 使用场景

当你切换到某个 Agent 后，希望继续之前已经存在但尚未恢复的历史会话时，可以在网页里使用会话恢复功能，而不是从头重新开始。

## 操作步骤

1. 在左侧 Agent 列表中点击你要切换的 Agent。
2. 页面切换到该 Agent 后，如果系统检测到这个 Agent 有可恢复会话，但当前历史消息为空，会自动弹出“选择会话恢复”窗口。
3. 在列表中选择你想恢复的会话。
4. 点击“恢复会话”完成恢复。
5. 如果你不想恢复，也可以点击“跳过”，直接进入当前 Agent 的历史查看或新会话状态。

## 你会看到的提示与反馈

- 会话恢复窗口标题为“选择会话恢复”。
- 每个可恢复会话会显示：

  - 会话名称
  - 会话时间

- 如果没有可恢复会话，窗口中会显示“没有可恢复的会话”。
- 恢复成功后，窗口会关闭，并重新加载该 Agent 的历史消息。
- 恢复失败时，页面会弹出“恢复会话失败”的提示。

## 注意事项

- 不是每次切换 Agent 都会出现恢复窗口，只有在满足可恢复条件时才会弹出。
- 你可以选择跳过恢复，不会强制必须恢复旧会话。
- 会话恢复依赖当前页面已经连接成功，并且当前 Agent 可正常访问相关数据。
- 如果恢复失败，优先检查网关连接状态和当前 Agent 是否仍可用。
