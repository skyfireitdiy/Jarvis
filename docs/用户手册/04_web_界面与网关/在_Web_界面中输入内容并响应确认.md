# 在 Web 界面中输入内容并响应确认

## 使用场景

当 Agent 在网页里处理任务时，你可能需要继续补充多行说明、回答某个单行输入请求，或者对某一步操作做确认。这篇文档适合帮助你判断该在什么时候输入、怎么发送，以及确认框出现后该怎么处理。

## 操作步骤

### 发送多行输入

1. 先切换到目标 Agent。
2. 观察底部输入区是否为多行输入框。
3. 在输入框中直接输入你的补充内容。
4. 发送时按 Ctrl+Enter，或点击“发送 (Ctrl+Enter)”。
5. 如果当前 Agent 正在等待多行输入，这段内容会直接发给 Agent。

### 发送单行输入

1. 当页面顶部或 Agent 列表显示“运行中（等待确认）”时，先查看底部输入区是否已经切换成单行输入框。
2. 在单行输入框中输入你的回答。
3. 按 Enter 发送。
4. 发送后，输入区通常会切回多行模式。

### 发送完成信号

1. 当页面处于“等待多行输入”状态，并且你已经输入完需要补充的内容时，点击“完成”。
2. 页面会先弹出确认提示“确定要发送完成信号吗？”。
3. 确认后，系统会告诉 Agent：你这次多行输入已经结束。
4. 如果当前正处于等待多行输入，而且输入框为空，也可以直接按 Ctrl+C 触发同样的完成动作。

### 处理确认框

1. 当消息区出现确认框时，阅读确认提示内容。
2. 按需要点击“确认”或“取消”。
3. 如果焦点已经停在按钮上，也可以直接按 Enter 提交当前焦点对应的按钮。
4. 出现确认框时，优先把它当作按钮确认场景处理，而不是当作底部单行输入框场景处理。

## 你会看到的提示与反馈

- 多行输入框默认提示类似“输入内容 (Ctrl+Enter 发送)”。
- 单行输入框默认提示类似“输入内容 (Enter 发送)”。
- 当页面显示“运行中（等待确认）”时，底部输入区会切换成单行输入框。
- 当当前没有可用 Agent，或 Agent 没有处于可输入状态时，输入框会显示“没有激活的 Agent 或 Agent 未运行”。
- 当 Agent 正在处理任务时，输入区上方会显示“Agent 正在思考...”。
- 你成功发送的普通输入会回显到消息区，方便你回看自己刚才输入了什么。
- 如果系统另外发起确认请求，消息区会显示确认框，并提供“确认”或“取消”按钮。

## 注意事项

- 多行输入和单行输入的发送键不同：多行是 Ctrl+Enter，单行是 Enter。
- “运行中（等待确认）”对应的是底部单行输入状态，不适合继续输入长文本。
- “完成”按钮只有在页面处于等待多行输入时才可用，它不是普通文本发送，而是告诉 Agent 当前这轮多行输入已经结束。
- 消息区确认框是独立出现的操作提示；只有真的看到确认框时，才需要用按钮处理。
