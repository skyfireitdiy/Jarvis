# 复制重命名和删除 Agent

## 使用场景

当你想快速复用一个已有 Agent 的配置，修改 Agent 显示名称，或者清理不再需要的 Agent 时，可以直接在 Agent 列表中完成复制、重命名和删除操作。

## 操作步骤

### 复制 Agent

1. 在左侧 Agent 列表中找到目标 Agent。
2. 点击该 Agent 右侧的复制按钮。
3. 系统会根据当前 Agent 的类型、工作目录、模型组等信息创建一个副本。
4. 复制成功后，副本会加入列表，并自动切换到这个新副本。

### 重命名 Agent

1. 在 Agent 列表中找到目标 Agent。
2. 点击右侧的重命名按钮。
3. 在“重命名 Agent”窗口中输入新名称。
4. 如果你想恢复默认显示方式，可以直接把名称清空。
5. 点击“确认”完成重命名。

### 删除 Agent

1. 在 Agent 列表中找到目标 Agent。
2. 点击右侧的删除按钮。
3. 页面会弹出确认提示。
4. 确认后，系统会删除该 Agent，并清理与它相关的本地历史记录和文件树状态。

### 批量复制或批量删除

1. 在 Agent 列表顶部点击批量选择按钮。
2. 勾选多个 Agent。
3. 根据需要点击批量复制或批量删除。
4. 批量删除时，同样会先弹出确认提示。
5. 批量操作完成后，页面会显示成功或失败数量提示，并退出批量模式。

## 你会看到的提示与反馈

- 重命名窗口标题为“重命名 Agent”。
- 重命名成功后，会看到“重命名成功”的提示。
- 删除前会看到明确提示：删除后将无法恢复，且会清除所有历史记录。
- 复制失败、重命名失败或删除失败时，页面会弹出相应错误信息。
- 批量复制和批量删除结束后，会显示成功数量和失败数量。

## 注意事项

- 删除是不可恢复操作，确认前请先检查是否仍需要保留该 Agent 的历史记录。
- 删除后，不只是列表项消失，和该 Agent 相关的本地历史与部分界面状态也会一起清理。
- 重命名时留空不是报错，而是恢复默认显示方式。
- 批量模式下，点击 Agent 列表项主要用于选择，不会像普通模式那样直接切换当前 Agent。
