# 管理历史记录与重新连接

## 使用场景

当你想查看当前浏览器里已经保存了多少历史消息、清理历史记录，或者修改网关地址、密码后重新连接到新的网关时，可以在网页设置中完成这些操作。

## 操作步骤

### 查看历史记录统计

1. 点击页面顶部的“设置”按钮。
2. 在设置窗口中查看历史消息统计信息。
3. 这里会显示当前历史消息数量和占用空间。

### 清除历史记录

1. 在设置窗口中点击“清除历史记录”。
2. 根据页面提示再次确认。
3. 清理完成后，当前 Agent 的消息区和历史分页状态会同步更新。

### 修改连接信息并重新连接

1. 在设置窗口中按需修改：

   - 密码
   - 网关地址
   - 连接锁定开关

2. 点击“重新连接”。
3. 页面会关闭当前连接，并按新的设置重新建立连接。
4. 如果连接成功，页面继续进入正常使用状态；如果失败，会重新显示连接窗口或错误提示。

## 你会看到的提示与反馈

- 设置窗口标题为“设置”。
- 历史消息区域会显示：

  - 历史消息数量
  - 存储空间

- 如果当前没有历史记录，“清除历史记录”按钮会不可用。
- 清除历史记录前会有确认步骤，因为这是不可撤销操作。
- 重新连接后，顶部连接状态会重新变化为“连接中”“已连接”或“未连接”。

## 注意事项

- 历史记录保存在当前浏览器本地，不是服务端全局历史。
- 清除历史记录是不可撤销操作，确认前请先判断是否还需要保留当前浏览记录。
- 修改密码或网关地址后，只有点击“重新连接”，新设置才会真正生效。
- 如果重新连接失败，优先检查网关地址格式、网关服务是否启动，以及密码是否正确。
