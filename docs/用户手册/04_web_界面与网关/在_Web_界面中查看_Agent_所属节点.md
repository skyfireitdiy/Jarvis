# 在 Web 界面中查看 Agent 所属节点

## 使用场景

当你已经启用了 node 模式，或者需要确认某个 Agent 实际运行在哪个节点上时，可以通过 Web 界面中的 Agent 列表与当前 Agent 信息区域查看节点归属。这有助于你判断请求会被转发到哪里，也方便排查某个子节点是否已经正常接入主节点。

## 操作步骤

1. 先确保当前浏览器已经连接到主节点 Gateway。
2. 在左侧 Agent 列表中查看每个 Agent 项旁边的节点标识。
3. 选中任意一个 Agent 后，再查看当前 Agent 的详细信息区域。
4. 如果你刚创建或复制了 Agent，可以创建完成后立即查看它的节点归属是否符合预期。

## 你会看到的提示与反馈

- 在 Agent 列表中，节点通常会以单独的标识展示，例如带有指南针图标的节点标签。
- 当前 Agent 区域也会显示该 Agent 的所属节点。
- 如果某个 Agent 没有显式标出子节点，一般表示它运行在主节点，或当前界面将其按默认主节点处理。

## 注意事项

- 节点信息是否可见，取决于当前 Gateway 是否返回了 Agent 的 `node_id`。
- 在 node 模式下，即使 Agent 实际运行在子节点，浏览器侧通常仍然只连接主节点；这是正常现象。
- 如果你预期某个 Agent 应该出现在子节点，但界面中仍显示为主节点，请优先检查创建时是否选择了目标节点，以及主子节点连接是否正常。
