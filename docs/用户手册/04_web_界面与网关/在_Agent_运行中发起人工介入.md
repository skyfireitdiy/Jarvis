# 在 Agent 运行中发起人工介入

## 使用场景

当 Agent 仍在继续执行任务，但你想在中途主动插入干预信号，让当前运行过程尽快响应你的人工处理意图时，可以使用 Web 界面的“人工介入”按钮。

## 操作步骤

1. 先切换到正在运行中的 Agent。
2. 观察顶部工具栏是否出现“人工介入”按钮。
3. 当这个按钮可见时，点击它。
4. 系统会立即把人工介入信号发给当前 Agent。
5. 之后继续观察消息区和状态变化，等待 Agent 对这次介入做出后续响应。

## 你会看到的提示与反馈

- “人工介入”按钮只会在当前 Agent 仍处于运行状态时出现。
- 在桌面端和移动端顶部工具栏里都可能看到这个按钮。
- 发送后，页面不会额外要求你再输入一条命令；这是一次直接的人工介入操作。
- 介入后，后续变化通常会体现在消息区输出、状态变化，或后续再次请求你输入内容。

## 注意事项

- 这不是普通文本输入，而是专门的人工干预信号。
- 如果当前 Agent 已经完成，或当前页面没有实时连接，这个按钮通常不会出现或不可用。
- 人工介入只作用于当前选中的 Agent；如果你切换错了 Agent，信号会发到错误的对象上。
- 发送人工介入后，建议继续留意消息区和状态栏，判断是否还需要你继续补充输入或确认。
