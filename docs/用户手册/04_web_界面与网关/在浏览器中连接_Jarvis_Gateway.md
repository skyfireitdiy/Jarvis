# 在浏览器中连接 Jarvis Gateway

## 使用场景

当你已经启动了 Web Gateway 服务，想在浏览器中进入 Jarvis 的网页界面并开始使用 Agent、消息区、终端和编辑器时，需要先完成浏览器连接。

## 操作步骤

1. 在浏览器中打开 Jarvis Web 界面。
2. 首次进入页面时，界面会自动弹出“连接到 Jarvis”窗口。
3. 在“网关地址”输入框中填写你的 Web Gateway 地址。可用格式包括：

   - `127.0.0.1:8000`
   - `example.com:8000`
   - 带协议的完整地址，例如 `ws://example.com:8080/ws`

4. 如果当前网关启用了密码认证，再在“密码”输入框中填写密码；如果没有启用密码，可以留空。
5. 点击“连接”。
6. 连接成功后，连接窗口会自动关闭，界面进入正常使用状态。
7. 如果后续连接断开，界面会重新显示连接窗口，这时你可以再次输入信息并重新连接。

## 你会看到的提示与反馈

- 初次打开页面时，会看到标题为“连接到 Jarvis”的连接弹窗。
- 连接过程中，按钮会显示“连接中...”。
- 连接成功后，顶部状态会从“连接中”切换为“已连接”。
- 连接失败时，连接窗口中会显示错误信息。
- 如果你输入了无效地址格式，界面会直接提示“无效的网关地址格式”。
- 连接断开后，顶部状态会恢复为未连接，同时连接弹窗会重新出现。

## 注意事项

- 只有在 Web Gateway 已启动的前提下，浏览器连接才有意义。
- 网关地址支持简写和完整 URL，两种都可以，但必须保证地址和端口是可访问的。
- 如果当前页面是 HTTPS 环境，界面会自动优先使用安全协议连接。
- 浏览器会记住最近一次成功连接的网关地址，下次打开页面时可继续沿用。
