# 配置连接锁定模式

## 使用场景

当你希望控制 Web Gateway 是否允许新页面或新客户端继续接入当前连接时，可以在网页设置中使用“锁定连接（拒绝新连接）”开关。

## 操作步骤

1. 先确保当前页面已经连接到 Web Gateway。
2. 点击页面顶部的“设置”按钮。
3. 在设置窗口中找到“锁定连接（拒绝新连接）”开关。
4. 根据需要切换开关状态：

   - 开启：当已有活跃连接时，新连接会被拒绝
   - 关闭：新连接会替换旧连接

5. 切换后，设置会立即保存在当前浏览器中。
6. 如果你已经连上网关，页面还会把这项设置同步发送给当前服务端连接。

## 你会看到的提示与反馈

- 设置窗口中会显示这项开关，以及对应说明文字。
- 说明文字会明确告诉你：开启后拒绝新连接，关闭后允许新连接替换旧连接。
- 如果当前已经连接成功，切换开关后新设置会立即对当前连接生效。
- 如果因为锁定连接导致新的页面无法接入，新的连接端通常会收到连接被拒绝或需要重新尝试的提示。

## 注意事项

- 这项设置保存在当前浏览器本地，下次打开同一个浏览器页面时通常会继续沿用。
- 这不是全局永久配置，而是当前浏览器侧的连接行为设置。
- 只有在 Web Gateway 已启动且页面已连接后，这项设置才会真正影响当前连接行为。
- 如果你在多设备或多个浏览器页面之间切换使用，建议先确认自己是否需要保留当前连接独占状态。
