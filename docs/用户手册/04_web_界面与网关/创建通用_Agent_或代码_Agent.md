# 创建通用 Agent 或代码 Agent

## 使用场景

当你想在 Web 界面里新建一个可执行任务的 Agent，并为它指定工作目录、模型组和类型时，可以使用创建 Agent 弹窗。它适合开始新的通用任务或代码任务。

## 操作步骤

1. 先确保当前页面已经连接到 Web Gateway。
2. 在左侧 Agent 列表顶部点击“+”。
3. 在“创建 Agent”弹窗中依次设置：

   - Agent 类型：通用 Agent 或代码 Agent
   - Agent 名称（可选）
   - 工作目录
   - 模型组

4. 如果你选择的是代码 Agent，还可以额外决定是否启用 worktree。
5. 工作目录有两种填写方式：

   - 直接手工输入路径
   - 点击“选择目录”，在目录选择窗口中浏览并确认

6. 目录选择窗口中，你可以使用：

   - 刷新当前目录
   - 返回上级目录
   - 搜索目录
   - 点击或双击目录进入子目录
   - 最后点击“确认”选中工作目录

7. 所有必填项准备完成后，点击“创建”。
8. 创建成功后，新 Agent 会加入列表，并自动切换到这个新 Agent。

## 你会看到的提示与反馈

- 创建弹窗标题为“创建 Agent”。
- Agent 类型会显示两种说明：

  - 通用 Agent：适用于日常任务和通用操作
  - 代码 Agent：专注于代码分析和开发任务

- 如果工作目录为空，“创建”按钮会保持不可用。
- 创建成功后，新 Agent 会出现在左侧列表中，并自动成为当前 Agent。
- 如果创建失败，页面会弹出失败提示。

## 注意事项

- 工作目录是必填项，没有设置时不能创建。
- worktree 开关只对代码 Agent 生效，通用 Agent 不使用这项设置。
- 模型组来自当前网关返回的可用模型组列表，因此需要连接成功后才能正常选择。
- 在移动端和桌面端，目录选择时的聚焦和交互细节会略有不同，但核心流程一致。
