# 启动 Web Gateway 服务

## 使用场景

当你希望通过浏览器使用 Jarvis 的 Web 界面，而不是只在终端里操作时，需要先启动 Web Gateway 服务。这个服务负责提供浏览器访问入口，并转发后续的网页连接、认证和操作请求。

## 操作步骤

1. 在终端中启动 Jarvis 的 Web Gateway 服务。
2. 如需使用默认配置，直接启动即可；默认会监听本地地址 `127.0.0.1` 和端口 `8000`。
3. 如果你想改监听地址或端口，可以在启动时显式指定地址和端口。
4. 如果你希望浏览器连接前先输入密码，可以在启动时一并设置网关密码。
5. 提交命令后，保持这个终端窗口不要关闭，让 Web Gateway 服务持续运行。
6. 服务启动后，再到浏览器中访问对应的 Web 页面并进行连接。

## 你会看到的提示与反馈

- 启动时，终端会显示：

  - 当前监听地址
  - 当前监听端口
  - 密码是否已设置

- 如果你设置了密码，终端会显示“已设置”。
- 如果你没有设置密码，终端会显示“未设置”。
- 启动成功后，服务会持续运行在当前终端，不会立即退出。

## 注意事项

- 这是终端中的启动步骤，不是浏览器页面里的操作。
- 不设置密码时，Web Gateway 不会启用密码认证；是否允许未设置密码访问，还会受到网关认证相关配置影响。
- Web Gateway 运行期间需要保持终端进程存活；如果你关闭了启动它的终端，浏览器连接通常也会中断。
- 如果你只监听 `127.0.0.1`，通常只适合本机访问。
