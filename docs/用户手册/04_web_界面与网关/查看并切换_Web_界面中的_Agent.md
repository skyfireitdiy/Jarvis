# 查看并切换 Web 界面中的 Agent

## 使用场景

当你已经在网页里创建了多个 Agent，想快速查看当前有哪些 Agent、分别使用什么工作目录和模型组，或者想在它们之间切换当前工作对象时，可以直接使用左侧 Agent 列表。

## 操作步骤

1. 先确保当前页面已经连接到 Web Gateway。
2. 查看左侧“Agent 列表”区域。
3. 在每个 Agent 条目中，你可以直接看到：

   - Agent 名称或类型图标
   - 当前状态
   - 模型组
   - 是否启用 worktree
   - 工作目录

4. 如果你只是想切换当前 Agent，直接点击该 Agent 的列表项。
5. 切换后，主消息区、底部输入区、顶部当前 Agent 信息，以及该 Agent 自己的历史消息都会切换到对应内容。
6. 如果你在移动端使用，点击 Agent 后，侧边栏会自动收起，让出主内容区域。
7. 如果你想浏览某个 Agent 的工作目录，而不是立即切换它，点击该 Agent 下方的工作目录文字。
8. 首次展开文件树时，如果页面显示“加载文件树”，再点击一次“加载文件树”开始读取目录内容。

## 你会看到的提示与反馈

- 左侧区域标题会显示为“Agent 列表”。
- 如果当前还没有任何 Agent，会显示“暂无 Agent，点击 + 创建”。
- 当前正在使用的 Agent 会在列表中显示为高亮状态。
- 点击工作目录后，会看到“（点击浏览）”对应区域展开为文件树。
- 如果当前 Agent 已经断开实时连接，但仍可恢复，页面会自动尝试重新连接；在新建 Agent 或切换到刚启动的 Agent 时，这个过程可能会有短暂等待。
- 如果当前 Agent 已经完成，页面仍会切换到它并展示历史内容，但不一定再建立实时连接。

## 注意事项

- 普通模式下，点击列表项会切换当前 Agent；批量模式下，点击列表项主要用于选择，不会直接切换。
- 切换 Agent 时，页面会清空当前输入框中尚未发送的内容；如果你还想保留这些文字，先发送，或先手动整理到缓冲区后再切换。
- 某些 Agent 在刚创建后需要一点启动时间，切换过去后页面可能会先加载历史，再等待连接稳定。
- 如果切换后没有实时消息，但能看到历史记录，这通常表示当前 Agent 已完成，可以把它当作历史会话查看。
