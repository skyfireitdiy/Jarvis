# 按角色配置开始对话

## 使用场景

当你已经准备好一份角色配置文件，希望按不同角色的系统设定快速开始对话，例如让当前模型以“产品经理”“代码审查员”或其他固定身份回应时，可以使用角色对话功能。

## 操作步骤

1. 先准备好角色配置文件。
2. 在终端中启动角色对话命令。
3. 如果你的角色配置文件不在默认位置，就在启动命令里显式指定配置文件路径；如果不指定，系统会默认读取 `~/.jarvis/roles.yaml`。
4. 如需临时换用别的模型组，也可以在这次命令启动时一并指定。
5. 命令启动后，系统会读取角色配置文件，并列出当前可选角色。
6. 你可以直接在角色选择列表中选中一个角色；如果没有使用列表选择，也可以按提示手动输入角色编号。
7. 选定后，系统会提示当前已选择的角色，并立即进入聊天状态。
8. 进入聊天后，继续在输入区域中提交问题即可与该角色设定下的模型对话。

## 你会看到的提示与反馈

- 如果配置文件不存在，你会看到“角色配置文件不存在”之类的提示。
- 如果配置文件内容无效，系统会提示“无效的角色配置文件”。
- 配置文件读取成功后，你会看到可用角色列表，以及每个角色的编号、名称和说明。
- 选择成功后，系统会提示：`已选择角色`。
- 随后系统会进入聊天流程，后续聊天反馈与普通平台聊天模式一致。
- 如果编号输入无效，你会看到“无效的选择”。

## 注意事项

- 这是终端交互能力，不是 Web 页面配置。
- 默认角色配置文件位置是 `~/.jarvis/roles.yaml`；如果你把文件放在别处，必须在启动时明确指定路径。
- 角色选择支持列表选择，也支持手动输入编号；如果你直接回车不选，当前流程会退出。
- 选择角色后进入的是聊天模式，因此你仍需要在后续输入内容后提交，模型才会开始回复。
