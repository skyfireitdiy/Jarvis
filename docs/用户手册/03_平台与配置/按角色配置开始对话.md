# 按角色配置开始对话

## 使用场景

当你已经准备好一份角色配置文件，希望让当前模型按某个固定角色设定开始对话，例如产品经理、代码审查员、方案顾问等，可以使用这个角色对话功能。

## 操作步骤

1. 先准备好角色配置文件。
2. 在终端中启动角色对话命令。
3. 如果你的角色配置文件不在默认位置，就在启动时显式指定配置文件路径；如果不指定，系统会默认读取 `~/.jarvis/roles.yaml`。
4. 如果你希望这次角色对话临时换用别的模型组，也可以在启动命令里一并指定。
5. 命令启动后，系统会先读取角色配置文件。
6. 如果角色文件有效，系统会列出当前可选角色。
7. 接下来你可以：
   - 直接在交互选择列表里选中一个角色，或
   - 按提示手动输入角色编号
8. 选定角色后，系统会提示当前已选择的角色，并立即进入聊天状态。
9. 进入聊天后，继续在输入区域中提交问题，就会按这个角色设定与当前模型对话。

## 你会看到的提示与反馈

- 如果配置文件不存在，你会看到：`❌ 角色配置文件 ... 不存在`。
- 如果配置文件存在但内容解析失败，你会看到角色配置文件解析失败提示。
- 如果文件结构不符合要求，你会看到：`❌ 无效的角色配置文件`。
- 加载成功后，你会看到：`✅ 可用角色`，以及一份按编号列出的角色列表，内容通常包含角色名称和说明。
- 如果你在选择列表里直接取消，或者在手动输入时直接回车，当前流程会退出，不会进入聊天。
- 选择成功后，你会看到：`✅ 已选择角色: 角色名`。
- 随后系统会进入聊天流程，后续聊天提示和普通平台聊天模式一致。
- 如果你输入了无效编号，会看到：`❌ 无效的选择`。

## 注意事项

- 这是终端交互能力，不是 Web 页面配置。
- 默认角色配置文件位置是 `~/.jarvis/roles.yaml`；如果你把文件放在别处，必须在启动时明确指定路径。
- 角色选择既支持交互列表，也支持回退到手动输入编号。
- 只有在角色文件有效并且你真正选中了某个角色后，系统才会进入聊天。
- 选择角色后进入的是聊天模式，因此你仍需要继续输入内容并提交，模型才会开始回复。
