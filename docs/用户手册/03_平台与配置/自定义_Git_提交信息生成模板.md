# 自定义 Git 提交信息生成模板

## 使用场景

当你希望 Jarvis 生成的提交说明更贴近团队规范，例如固定使用 Conventional Commits、要求写清模块名和风险说明，或者希望加入你们自己的提交格式提示时，可以配置 `git_commit_prompt`。

## 操作步骤

1. 找到 Jarvis 的全局配置文件，默认位置是：`~/.jarvis/config.yaml`。
2. 如果你不想手工编辑文件，也可以启动 Web 配置界面：
   - 命令：`jarvis-config web`
   - 默认输出文件也是 `~/.jarvis/config.yaml`
3. 在配置文件中加入或修改 `git_commit_prompt`。例如：

   ```yaml
   git_commit_prompt: |
     请按以下格式生成提交信息：
     1. 第一行使用 Conventional Commits 风格，例如 feat:, fix:, refactor:
     2. 必须写明影响模块
     3. 正文补充变更目的和潜在风险
   ```

4. 保存配置文件。
5. 重新执行会触发提交信息生成的流程，让系统重新读取配置。
6. 对比新生成的提交信息，确认输出风格是否已经按你的模板变化。

## 你会看到的提示与反馈

- 这个配置项本身不会弹出单独的“配置成功”窗口。
- 真正的反馈出现在后续生成提交信息时：生成结果会优先参考你写入的 `git_commit_prompt`。
- 如果你使用 `jarvis-config web`，启动时通常会看到：
  - 配置 Web 服务启动成功
  - 输出文件路径为 `~/.jarvis/config.yaml`
- 如果配置文件写错了 YAML 格式，后续读取配置时可能导致相关能力无法按预期生效。

## 注意事项

- 这里配置的是“提交信息生成提示模板”，不是“自动直接提交代码”。
- 配置项名称就是 `git_commit_prompt`，不是自然语言描述。
- 建议把格式要求写具体，例如标题格式、正文结构、是否要求风险说明；写得太泛，效果通常不稳定。
- 如果修改后感觉没变化，先检查三件事：
  1. 你改的是不是 `~/.jarvis/config.yaml`
  2. YAML 缩进是否正确
  3. 触发提交信息生成的流程是否重新读取了配置
