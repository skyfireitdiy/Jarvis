# 启动 Web 配置界面编辑配置

## 使用场景

当你不想直接手写配置文件，而是希望在浏览器中通过可视化页面编辑 Jarvis 配置时，可以启动 Web 配置界面。

## 操作步骤

1. 在终端中启动 Jarvis 配置工具的 Web 模式。
2. 如果你不额外指定参数，系统会：
   - 使用默认的配置 Schema
   - 把输出文件设为 `~/.jarvis/config.yaml`
   - 在 `8080` 端口启动服务
   - 启动后自动尝试打开浏览器

3. 如果你需要改用其他 Schema 文件、输出文件或端口，可以在启动命令时分别指定。
4. 如果你不希望系统自动打开浏览器，可以在启动时关闭自动打开浏览器的行为。
5. 命令启动后，终端会显示访问地址。
6. 在浏览器中打开对应地址后，按页面提示编辑配置并保存。
7. 使用完成后，回到启动服务的终端，按 `Ctrl+C` 停止服务。

## 你会看到的提示与反馈

- 启动时，你会看到：
  - 当前使用的 Schema 文件
  - 输出文件路径
  - 服务端口

- 如果 Schema 加载成功，你会看到加载成功提示。
- 服务启动后，你会看到“服务已启动”和浏览器访问地址，例如 `http://localhost:8080`。
- 如果系统成功自动打开浏览器，会看到“已自动打开浏览器”的提示。
- 如果自动打开失败，系统会提示你手动访问对应地址。
- 停止服务时，终端会显示服务已停止提示。

## 注意事项

- 这是“终端启动 + 浏览器编辑”的方式，不是纯终端交互流程。
- 默认输出文件是 `~/.jarvis/config.yaml`，因此你通过网页保存的结果会直接写入这份配置。
- 如果你使用自定义 Schema 或自定义输出文件，启动时必须显式传入对应参数。
- 服务启动后会持续占用当前终端，直到你按 `Ctrl+C` 停止。
