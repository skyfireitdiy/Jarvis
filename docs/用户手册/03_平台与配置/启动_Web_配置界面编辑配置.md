# 启动 Web 配置界面编辑配置

## 使用场景

当你不想直接手写配置文件，而是希望在浏览器中通过可视化页面编辑 Jarvis 配置时，可以启动这个 Web 配置界面。

## 操作步骤

1. 在终端中输入下面的命令，然后按回车：`jcfg`
2. 如果你不额外指定参数，系统会默认：
   - 使用内置的配置 Schema
   - 把输出文件设为 `~/.jarvis/config.yaml`
   - 在 `8080` 端口启动服务
   - 启动后自动尝试打开浏览器
3. 如果你需要改用其他 Schema 文件，可以在启动时额外指定 Schema 路径，例如：`jcfg --schema /你的/schema.json`
4. 如果你希望把结果保存到别的文件，也可以在启动时指定输出文件路径，例如：`jcfg --output /你的/config.yaml`
5. 如果你需要换端口，可以在启动时指定端口号，例如：`jcfg --port 3000`
6. 如果你不希望系统自动打开浏览器，可以在启动时加上 `--no-browser`，例如：`jcfg --no-browser`
7. 命令启动后，终端会显示访问地址。
8. 在浏览器中打开对应地址后，按页面提示编辑配置并保存。
9. 使用完成后，回到启动服务的终端，按 `Ctrl+C` 停止服务。

## 你会看到的提示与反馈

- 启动时，你会先看到一段启动信息，包括：
  - `Schema 文件: ...`
  - `输出文件: ...`
  - `服务端口: ...`
- 如果 Schema 加载成功，你会看到：`✓ Schema 加载成功: ...`。
- 如果 Schema 加载失败，系统会直接报错并退出。
- 如果输出目录无法创建，你会看到“无法创建输出目录”的错误提示。
- 如果 Web 应用创建失败，也会直接提示“创建应用失败”。
- 服务成功启动后，你会看到：
  - `✓ 服务已启动！`
  - 访问地址，例如 `http://localhost:8080`
  - `按 Ctrl+C 停止服务`
- 如果系统成功自动打开浏览器，会看到：`已自动打开浏览器...`。
- 如果自动打开失败，系统会提示你手动访问对应地址。
- 停止服务时，终端会显示：`⚠ 服务已停止`。

## 注意事项

- 这是“终端启动 + 浏览器编辑”的方式，不是纯终端交互流程。
- 默认输出文件是 `~/.jarvis/config.yaml`，因此你通过网页保存的结果会直接写入这份配置。
- 默认访问提示显示的是 `http://localhost:端口`，但实际服务会启动在可接受外部访问的监听地址上；如果你只在本机使用，直接按终端给出的地址打开即可。
- 如果你使用自定义 Schema 或自定义输出文件，启动时必须显式传入对应参数。
- 服务启动后会持续占用当前终端，直到你按 `Ctrl+C` 停止。
