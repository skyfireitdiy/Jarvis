# 配置中心工具仓库

## 使用场景

如果你们团队把共享工具集中放在一个公共仓库中，希望每个人的 Jarvis 都能自动接入这批工具，可以配置 `central_tool_repo`。

## 操作步骤

1. 准备中心工具来源，二选一即可：
   - 本地目录
   - 远端 Git 仓库地址
2. 打开 Jarvis 全局配置文件：`~/.jarvis/config.yaml`。
3. 添加或修改 `central_tool_repo`。例如：

   ```yaml
   central_tool_repo: "https://github.com/your-org/jarvis-tools.git"
   ```

   或者使用本地目录：

   ```yaml
   central_tool_repo: "/opt/team/jarvis-tools"
   ```

4. 保存配置文件。
5. 重新启动相关会话。
6. 如果配置的是本地目录，系统会直接把它作为中心工具来源。
7. 如果配置的是远端 Git 地址，系统会在本地没有缓存时先克隆，再加载工具；后续还会做后台更新检查。

## 你会看到的提示与反馈

- 首次使用远端仓库且本地没有缓存时，通常会先执行仓库克隆。
- 如果克隆失败，你会看到中心工具仓库克隆失败之类的错误提示。
- 配置成功后，你原本没有的共享工具会出现在工具来源中。

## 注意事项

- 配置项名称是 `central_tool_repo`。
- 这个值既可以是本地目录，也可以是 Git 仓库 URL。
- 即使工具已经成功加载，后续仍可能被 `tool_group`、`use`、`dont_use` 等过滤规则限制，导致当前会话中不可用。
- 如果你希望团队统一工具集合，最好配合工具组一起使用，避免“仓库里有，但当前会话没放开”的误解。
