# 配置默认自动加载的规则

## 使用场景

如果你希望某些规则在每次启动 CodeAgent 或相关 Agent 时都自动带上，而不是每次都手动输入规则名，可以在配置文件里设置 `default_rule_names`。

## 操作步骤

1. 先确认你要加载的规则名称是真实存在、可用的。
2. 打开 Jarvis 全局配置文件：`~/.jarvis/config.yaml`。
3. 添加或修改 `default_rule_names`，它是一个字符串数组。例如：

   ```yaml
   default_rule_names:
     - project:code_quality/jarvis_user_manual_writing
     - global:security/basic_safe_editing
   ```

4. 保存配置文件。
5. 重新启动会读取配置的会话。
6. 后续如果启动入口还额外传入了规则名，系统会把 `default_rule_names` 和命令行传入的规则合并，并去重。

## 你会看到的提示与反馈

- 这个配置项通常不会单独弹出“默认规则已加载”的提示框。
- 更实际的反馈是：后续任务中，系统会稳定遵循这些默认规则约束。
- 如果某个规则名写错，常见现象是这条规则没有生效，而不是系统替你自动修正。
- 如果你同时又在命令行传入 `--rule-names`，最终生效的是“配置规则 + 命令行规则”的去重合并结果。

## 注意事项

- 配置项名称是 `default_rule_names`。
- 这是数组，不是单个字符串。
- 适合写长期稳定使用的规则；临时规则更适合按单次任务指定。
- 如果你不确定规则名是否存在，先确认规则来源，再写入配置，避免文档看起来正确但运行时无效。
