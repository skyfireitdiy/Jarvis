# 配置额外的规则加载目录

## 使用场景

如果你的规则文件不只放在 Jarvis 默认来源里，而是还放在个人目录、团队共享目录或项目公共目录中，可以通过 `rules_load_dirs` 把这些目录加入规则加载来源。

## 操作步骤

1. 先准备好规则目录，并确保目录内容是你希望 Jarvis 扫描的规则文件。
2. 打开全局配置文件：`~/.jarvis/config.yaml`。
3. 添加或修改 `rules_load_dirs`。例如：

   ```yaml
   rules_load_dirs:
     - ~/jarvis-rules
     - /opt/team/jarvis-rules
   ```

4. 保存配置文件。
5. 重新启动相关会话，让系统重新扫描规则来源。
6. 之后这些目录会作为“额外规则来源”参与规则搜索和加载。

## 你会看到的提示与反馈

- 一般不会出现单独的“目录添加成功”提示。
- 更常见的现象是：之前找不到的规则，在重新启动后可以被枚举、检索或正常加载。
- 如果目录写错、目录不存在或里面没有可识别的规则文件，表现通常是这些规则仍然不可用。

## 注意事项

- 配置项名称是 `rules_load_dirs`。
- 这是“额外来源”，不会替代默认规则来源。
- 建议优先使用稳定的绝对路径；如果使用 `~`，要确认当前运行环境会正确展开。
- 如果多个来源中存在同名规则，使用时要特别注意自己加载的是哪一份。
