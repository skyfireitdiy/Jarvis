# 配置多行输入的提交快捷键

## 使用场景

如果你在终端的多行输入框里经常误触默认提交键，或者当前终端对默认组合键支持不好，可以通过 `submit_keys` 修改多行输入的提交快捷键。

## 操作步骤

1. 打开 Jarvis 全局配置文件：`~/.jarvis/config.yaml`。
2. 找到或新增 `submit_keys` 配置项。
3. 这个配置支持两种写法：
   - 数组格式
   - 逗号分隔的字符串格式
4. 推荐使用数组格式，示例：

   ```yaml
   submit_keys:
     - c-j
     - c-]
   ```

   或者写成字符串：

   ```yaml
   submit_keys: "c-j,c-]"
   ```

5. 保存配置后，重新启动终端会话。
6. 进入需要多行输入的界面，尝试用你配置的新快捷键提交内容。

## 你会看到的提示与反馈

- 配置生效后，多行输入区域通常会按新的快捷键规则响应。
- Schema 中默认值是 `Ctrl+J` 和 `Ctrl+]`，对应写法分别是 `c-j`、`c-]`。
- 常见可用写法包括：`c-j`、`c-]`、`c-enter`。
- 如果你按旧快捷键没有反应，而新快捷键能提交，说明配置已经生效。

## 注意事项

- 配置项名称是 `submit_keys`。
- 不同终端、tmux、远程 shell 对快捷键支持不完全一致，`c-enter` 在某些环境中可能无法正常传递。
- 如果你改完后无法提交，优先回退到更保守的组合，例如 `c-j`。
- YAML 中数组格式更清晰，也更不容易因为字符串分隔写错而失效。
