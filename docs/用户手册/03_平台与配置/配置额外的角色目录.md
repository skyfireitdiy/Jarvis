# 配置额外的角色目录

## 使用场景

如果你希望系统除了默认角色来源外，还能从其他目录发现更多角色设定，可以通过 `roles_dirs` 扩展角色搜索范围。

## 操作步骤

1. 准备好额外角色目录，并把角色设定内容放入其中。
2. 打开 Jarvis 全局配置文件：`~/.jarvis/config.yaml`。
3. 添加或修改 `roles_dirs`。例如：

   ```yaml
   roles_dirs:
     - ~/jarvis-roles
     - /opt/team/jarvis-roles
   ```

4. 保存配置文件。
5. 重新启动相关会话。
6. 之后系统在查找角色来源时，会把这些目录也纳入搜索范围。
7. 到角色相关入口中确认这些新增角色是否已经可见。

## 你会看到的提示与反馈

- 配置生效后，原本不在默认位置的角色内容更有机会被系统识别到。
- 如果目录路径写错，或者目录中没有有效角色内容，常见现象就是你仍然找不到以为已经配置好的角色。
- 如果你在具体命令中显式指定了角色文件，那次运行会优先以显式指定的文件为准。

## 注意事项

- 配置项名称是 `roles_dirs`。
- 这是扩展角色来源的配置，不等于“在某次命令里直接指定一个角色文件路径”。
- 长期复用的角色建议放在稳定目录中，避免临时目录失效后难以排查。
- 改完后最好重新进入相关入口确认角色是否真的被发现，而不是只看配置文件本身。
