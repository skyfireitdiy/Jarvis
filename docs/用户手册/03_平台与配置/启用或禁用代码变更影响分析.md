# 启用或禁用代码变更影响分析

## 使用场景

如果你希望系统在代码修改后自动判断变更可能影响哪些位置，需要了解当前影响分析行为。根据当前源码行为，影响分析默认开启。

## 操作步骤

1. 在进行代码修改任务时，观察系统是否在编辑后输出影响范围分析结果。
2. 当前源码中，影响分析默认返回开启状态。
3. 如果你的实际运行入口对这一步做了额外控制，应以当次运行入口的真实行为为准。

## 你会看到的提示与反馈

- 在相关流程里，编辑完成后通常会出现影响范围分析报告。
- 如果运行入口跳过了这一步，你可能只看到编辑结果本身，而不会看到影响分析结论。
- 影响分析通常用于帮助你判断是否还需要补测、补查相关文件。

## 注意事项

- 就当前 `config.py` 行为来看，这同样更接近“默认行为说明”，而不是已经明确暴露给用户的独立 config 开关。
- 文档不应过度承诺“你现在一定能通过某个 config 键自由关闭它”，除非后续源码明确提供该配置项。
- 如果你需要稳定控制这一步是否执行，应结合具体入口和任务运行模式一起判断。
