# 控制 diff 是否显示行号

## 使用场景

如果你在阅读代码变更时非常依赖行号定位，需要了解当前 diff 行号行为。根据当前源码行为，diff 默认显示行号。

## 操作步骤

1. 在一次会展示 diff 的代码修改流程中观察当前预览内容。
2. 当前源码中，diff 行号显示默认返回 `true`。
3. 如果你当前看到的 diff 中包含行号，说明这项默认行为已经生效。

## 你会看到的提示与反馈

- 在 diff 预览中，你通常会直接看到行号信息。
- 行号对定位修改位置、交叉核对文件内容更方便。
- 如果某些场景没有显示行号，更可能是该入口没有使用同一套 diff 展示路径，而不是这里的默认行为失效。

## 注意事项

- 当前实现更接近“固定默认行为说明”，不是已正式暴露给用户的独立配置项说明。
- 不要把这篇文档理解成“现在可以在 config.yaml 里自由开关行号”，除非后续版本明确提供对应配置键。
