# 调整代码 diff 的展示模式

## 使用场景

如果你觉得当前 diff 预览方式不适合阅读，例如希望更紧凑、强调语法高亮或并排查看，就需要先了解当前 diff 展示行为。根据当前源码行为，默认返回的可视化模式是 `side_by_side`。

## 操作步骤

1. 先在一次会显示 diff 的代码修改流程中观察当前展示效果。
2. 当前源码注释列出了可选模式概念：`unified`、`syntax`、`compact`、`side_by_side`、`default`。
3. 根据当前 `config.py` 实现，默认返回值是 `side_by_side`。
4. 如果后续版本把该能力暴露成正式配置项，再按正式键名配置；当前文档主要用于说明现状与边界。

## 你会看到的提示与反馈

- 在支持 diff 展示的场景里，你会直接看到当前使用的展示风格。
- 当前实现下，更接近并排展示效果。
- 如果后续入口没有启用增强 diff，可见效果可能与这里描述的模式名不完全一致。

## 注意事项

- 当前源码显示的是固定返回值，不应把它写成“已经稳定开放给用户自由配置”的能力。
- 这篇文档的重点是帮助用户理解当前 diff 展示偏好，而不是承诺现阶段一定能通过 `config.yaml` 切换。
- 如果你需要稳定修改展示模式，应等待对应配置项正式暴露或使用入口级参数（若未来提供）。
