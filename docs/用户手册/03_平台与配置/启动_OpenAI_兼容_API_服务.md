# 启动 OpenAI 兼容 API 服务

## 使用场景

当你希望把 Jarvis 当前默认使用的平台和模型以 OpenAI 兼容接口的形式暴露出来，供其他程序、脚本或调用方访问时，可以启动这个服务。

## 操作步骤

1. 先确认你当前默认模型组对应的平台和模型已经能正常使用。
2. 在终端中输入下面的命令，然后按回车：`jpm service`
3. 如果你不额外传参数，系统会使用默认主机地址 `127.0.0.1` 和默认端口 `8000`。
4. 如果你希望改成其他监听地址或端口，可以使用类似命令：`jpm service --host 0.0.0.0 --port 9000`
5. 提交后，系统会读取当前默认平台和默认模型，并以它们作为服务的默认后端启动 OpenAI 兼容 API 服务。
6. 服务启动后，让你的外部程序按这个地址访问即可。
7. 使用结束后，在当前终端按中断方式停止服务。

## 你会看到的提示与反馈

- 如果启动成功，当前终端会进入持续运行状态，表示服务已经开始监听。
- 默认情况下，监听地址是 `127.0.0.1:8000`。
- 如果你改了主机或端口，后续访问地址也会跟着变化。
- 如果当前默认平台或模型配置不完整，或者底层服务启动失败，你会看到对应的错误或异常提示，而不会进入正常监听状态。

## 注意事项

- 这个服务依赖你当前默认平台和模型配置可正常使用，因此最好先确认默认模型组已经配置完成，必要时先用“查看当前平台与模型信息”或“直接与当前模型对话”做检查。
- 不传参数时，只监听 `127.0.0.1:8000`，通常只适合本机访问。
- 如果你要改主机地址和端口，应在启动时一并传入。
- 这是一个持续运行的服务进程，不会像普通查询操作那样执行完就立刻退出。
- 这里启动的是 OpenAI 兼容接口服务，不是 Web 配置页面。
