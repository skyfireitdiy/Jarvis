# 启动 OpenAI 兼容 API 服务

## 使用场景

当你希望把 Jarvis 当前默认平台和模型以 OpenAI 兼容接口的形式暴露出来，供其他程序、脚本或调用方访问时，可以启动这个服务。

## 操作步骤

1. 在终端中启动平台服务命令。
2. 如果不额外传参数，系统会使用默认主机地址 `127.0.0.1` 和默认端口 `8000`。
3. 如果你需要让别的地址或端口提供服务，可以在启动时显式指定主机地址和端口。
4. 提交命令后，系统会读取当前默认平台和模型，并启动 OpenAI 兼容服务。
5. 服务启动后，让你的外部程序按这个地址访问即可。
6. 如需停止服务，在当前终端中中断运行即可。

## 你会看到的提示与反馈

- 启动时，系统会基于当前默认平台和模型启动服务。
- 如果服务启动成功，终端会进入持续运行状态，表示服务正在监听。
- 如果当前默认平台或模型配置不完整，或者服务启动过程中出现异常，你会看到相应错误提示。

## 注意事项

- 这个服务依赖你当前默认平台和模型配置可正常使用，因此最好先确认默认模型组已经配置完成。
- 不传参数时，只监听 `127.0.0.1:8000`，通常只适合本机访问。
- 如果你要改主机地址和端口，应在启动时一并传入。
- 这是一个持续运行的服务命令，不会像普通查询命令那样执行完就立刻退出。
