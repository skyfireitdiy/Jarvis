# 配置中心规则仓库

## 使用场景

如果你们团队把规则集中维护在一个共享仓库中，希望所有人都能自动接入同一批规则，可以配置 `central_rules_repo`。

## 操作步骤

1. 准备好中心规则来源，二选一即可：
   - 本地目录路径
   - 远端 Git 仓库地址
2. 打开 Jarvis 全局配置文件：`~/.jarvis/config.yaml`。
3. 添加或修改 `central_rules_repo`。例如：

   ```yaml
   central_rules_repo: "https://github.com/your-org/jarvis-rules.git"
   ```

   或者使用本地目录：

   ```yaml
   central_rules_repo: "/opt/team/jarvis-rules"
   ```

4. 保存配置文件。
5. 重新启动相关会话。
6. 如果你填的是本地目录，系统会直接把它作为中心规则来源。
7. 如果你填的是远端 Git 地址，系统在本地没有缓存时会先克隆，再纳入规则来源；后续还会做后台更新检查。

## 你会看到的提示与反馈

- 首次使用远端仓库且本地还没有缓存时，通常会看到中心规则仓库克隆相关提示。
- 如果克隆失败，会出现“中心规则仓库克隆失败”一类错误信息。
- 配置成功后，你会在规则枚举、加载或来源区分场景中看到来自中心仓库的规则。

## 注意事项

- 配置项名称是 `central_rules_repo`。
- 这个值既可以是本地目录，也可以是 Git 仓库 URL。
- 如果使用远端仓库，当前环境必须能访问仓库并执行 `git clone`。
- 如果中心仓库和其他来源中有同名规则，使用前要确认你实际加载的是哪一份。
