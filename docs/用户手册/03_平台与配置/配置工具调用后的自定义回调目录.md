# 配置工具调用后的自定义回调目录

## 使用场景

如果你希望在工具执行完成后接入自己的后置逻辑，例如记录额外日志、发通知、做二次处理，可以通过 `after_tool_call_cb_dirs` 配置自定义回调目录。

## 操作步骤

1. 准备好一个或多个回调实现目录。
2. 把你的工具调用后处理逻辑放入这些目录。
3. 打开 Jarvis 全局配置文件：`~/.jarvis/config.yaml`。
4. 添加或修改 `after_tool_call_cb_dirs`。例如：

   ```yaml
   after_tool_call_cb_dirs:
     - ~/jarvis-callbacks
     - /opt/team/jarvis-callbacks
   ```

5. 保存配置文件。
6. 重新启动相关会话。
7. 执行一次工具调用，观察你的后处理逻辑是否真的发生。

## 你会看到的提示与反馈

- 这个功能通常不会提供一个独立的“回调已加载成功”页面。
- 更常见的验证方式，是看回调带来的后续效果，例如额外输出、额外记录或其他后处理行为。
- 如果目录路径无效，或者回调实现本身不符合要求，常见现象是你期待的后处理没有发生。

## 注意事项

- 配置项名称是 `after_tool_call_cb_dirs`。
- 这是扩展能力配置，不属于普通用户日常必须操作的主流程。
- 如果目录里放了多套逻辑，后续排查效果时要注意区分到底是哪一套在生效。
- 建议先用一个简单回调做验证，再逐步增加更复杂的后处理逻辑。
