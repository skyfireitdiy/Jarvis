# 配置额外的工具加载目录

## 使用场景

如果你写了自定义工具，或者团队把共享工具放在某个固定目录中，希望 Jarvis 在启动时一并扫描这些工具，可以配置 `tool_load_dirs`。

## 操作步骤

1. 准备好你的自定义工具目录。
2. 打开 Jarvis 全局配置文件：`~/.jarvis/config.yaml`。
3. 添加或修改 `tool_load_dirs`。例如：

   ```yaml
   tool_load_dirs:
     - ~/jarvis-tools
     - /opt/team/jarvis-tools
   ```

4. 保存配置文件。
5. 重新启动相关会话。
6. 系统启动时会在默认来源之外，再扫描 `tool_load_dirs` 中列出的目录。

## 你会看到的提示与反馈

- 配置生效后，原本不存在的自定义工具会出现在工具系统中。
- 如果目录不存在、路径写错，常见现象是你期待的工具没有出现。
- 如果目录存在但某个工具文件本身有问题，通常只会导致该工具加载失败，而不是整个系统都不能启动。

## 注意事项

- 配置项名称是 `tool_load_dirs`。
- 这是“额外来源”，不会替代内置工具来源。
- 如果你同时配置了工具白名单、黑名单或工具组，工具即使成功加载，也不一定会在当前会话中最终可用。
- 建议目录中只放准备给 Jarvis 扫描的工具文件，减少无关内容干扰。
