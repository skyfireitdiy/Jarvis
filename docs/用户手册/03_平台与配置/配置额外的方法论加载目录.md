# 配置额外的方法论加载目录

## 使用场景

如果你除了默认方法论目录外，还希望从个人知识库、团队共享目录或其他外部目录加载方法论，可以配置 `methodology_dirs`。

## 操作步骤

1. 准备好一个或多个方法论目录，并确保里面放的是你希望 Jarvis 加载的方法论文件。
2. 打开 Jarvis 全局配置文件：`~/.jarvis/config.yaml`。
3. 添加或修改 `methodology_dirs`。例如：

   ```yaml
   methodology_dirs:
     - ~/jarvis-methodologies
     - /opt/team/methodologies
   ```

4. 保存配置文件。
5. 重新启动相关会话。
6. 之后系统在加载方法论时，会把这些目录和默认目录一起纳入来源。
7. 观察方法论总数是否增加，或者相关任务是否能命中你新增的方法论。

## 你会看到的提示与反馈

- 系统加载方法论时，通常会有加载开始和加载完成的提示。
- 如果额外目录存在且内容有效，加载完成后的方法论数量通常会增加。
- 如果目录不存在或不是有效目录，系统会给出目录相关警告。
- 如果目录中的某个文件损坏或格式不正确，系统会提示该文件加载失败。

## 注意事项

- 配置项名称是 `methodology_dirs`。
- 这是附加来源，不会替代项目级和全局默认方法论目录。
- 如果多个目录里有相同问题类型的方法论，后续可能出现覆盖或并存效果，使用前应留意结果是否符合预期。
- 改完后建议重新启动一次会话，确认方法论数量或行为变化是否符合预期。
