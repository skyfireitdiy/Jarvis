# 跳过启动时的预定义任务选择

## 使用场景

如果你不希望 Agent 启动时先进入预定义任务选择步骤，而是希望直接进入手动输入需求的流程，可以配置 `skip_predefined_tasks`。

## 操作步骤

1. 打开 Jarvis 全局配置文件：`~/.jarvis/config.yaml`。
2. 添加或修改以下配置：

   ```yaml
   skip_predefined_tasks: true
   ```

3. 保存配置文件。
4. 重新启动对应的 Agent 入口。
5. 启动后观察是否直接进入手动输入需求，而不是先出现预定义任务选择。

## 你会看到的提示与反馈

- 配置生效后，启动流程会更直接，不再停留在预定义任务选择步骤。
- 如果你把它设为 `false` 或不配置，系统会按默认行为决定是否加载预定义任务。
- 如果修改后看起来没有变化，优先检查当前入口是否重新读取了配置。

## 注意事项

- 配置项名称是 `skip_predefined_tasks`。
- 这是布尔值，写法应为 `true` 或 `false`，不要写成字符串。
- 这个配置只影响“是否跳过预定义任务加载”，不影响你后续手动输入任务的能力。
