# 启用任务完成后的自动方法论提取

## 使用场景

如果你希望系统在任务结束后自动从执行过程里提炼可复用经验，并保存为后续可用的方法论，可以开启 `auto_methodology_extraction`。

## 操作步骤

1. 打开 Jarvis 全局配置文件：`~/.jarvis/config.yaml`。
2. 添加或修改以下配置：

   ```yaml
   auto_methodology_extraction: true
   ```

3. 保存配置文件。
4. 重新启动相关会话。
5. 完成一次具有代表性的任务。
6. 之后检查方法论相关内容是否新增，或观察后续加载方法论时数量是否变化。

## 你会看到的提示与反馈

- 开启后，任务完成时系统会尝试自动提取方法论并保存。
- 如果提取成功，后续方法论加载时可能出现新的条目。
- 如果没有提取出有效内容，可能不会有明显新增结果。

## 注意事项

- 配置项名称是 `auto_methodology_extraction`。
- 默认行为是关闭，只有设为 `true` 才会启用。
- 这项功能适合沉淀长期可复用经验，不适合把一次性临时结论都当成方法论保存。
