# 基于当前需求自动筛选相关方法论

## 使用场景

如果你已经准备了很多方法论，但不希望每次都手工挑选，Jarvis 会在需要时根据当前需求自动筛选更相关的方法论供后续使用。

## 操作步骤

1. 先确保方法论来源已经准备好，例如：
   - 默认方法论目录
   - 项目级方法论目录
   - `methodology_dirs`
   - `central_methodology_repo`
2. 启动相关 Agent 并输入你的任务需求。
3. 当系统进入方法论加载流程时，它会先加载所有可用方法论标题。
4. 然后系统会结合当前需求和可用工具，自动选择更相关的方法论。
5. 观察方法论加载提示和后续任务表现，确认筛选后的方法论是否已经参与当前任务。

## 你会看到的提示与反馈

- 系统通常会先提示“加载方法论文件”。
- 如果方法论成功加载，会提示总数量。
- 如果没有找到可用方法论，系统会给出相关警告。
- 真正的筛选结果通常体现在后续任务行为中：系统更可能引用与当前问题更相关的经验。

## 注意事项

- 这项能力依赖你先准备好方法论来源；没有方法论文件时，系统无法筛选。
- 自动筛选是相关性选择，不保证每次都完全符合你的主观预期。
- 如果你发现筛选结果不理想，优先检查方法论标题命名、内容质量，以及方法论来源是否真的被正确加载。
