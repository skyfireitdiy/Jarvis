# 配置中心方法论仓库

## 使用场景

如果你们团队把常用方法论集中维护在一个共享仓库里，希望每个人的 Jarvis 都能自动接入这批内容，可以配置 `central_methodology_repo`。

## 操作步骤

1. 准备好中心方法论来源，二选一即可：
   - 本地目录
   - 远端 Git 仓库地址
2. 打开 Jarvis 全局配置文件：`~/.jarvis/config.yaml`。
3. 添加或修改 `central_methodology_repo`。例如：

   ```yaml
   central_methodology_repo: "https://github.com/your-org/jarvis-methodologies.git"
   ```

   或者写成本地目录：

   ```yaml
   central_methodology_repo: "/opt/team/jarvis-methodologies"
   ```

4. 保存配置文件。
5. 重新启动相关会话。
6. 如果配置的是本地目录，系统会直接把它作为中心方法论来源。
7. 如果配置的是远端 Git 地址，系统在本地没有缓存时会先克隆，再纳入方法论来源；后续还会做后台更新检查。

## 你会看到的提示与反馈

- 首次使用远端仓库且本地没有缓存时，通常会看到中心方法论仓库克隆相关提示。
- 如果克隆失败，会看到中心方法论仓库克隆失败一类错误信息。
- 加载完成后，方法论总数通常会增加。
- 如果中心目录不可访问或文件格式有问题，也会在加载阶段看到警告或失败提示。

## 注意事项

- 配置项名称是 `central_methodology_repo`。
- 这个值既可以是本地目录，也可以是 Git 仓库 URL。
- 如果使用远端仓库，当前环境必须具备访问仓库和执行 Git 的能力。
- 中心方法论只是来源之一，不代表每条方法论都会自动在每次任务里被完整采用，后续仍会经过相关性筛选。
