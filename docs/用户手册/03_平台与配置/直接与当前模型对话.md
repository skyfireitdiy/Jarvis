# 直接与当前模型对话

## 使用场景

当你想先单独验证某个模型组对应的平台和模型是否能正常响应，或者只是想快速与当前模型聊几轮，确认连接、模型输出和回复风格是否正常时，可以直接进入平台聊天模式。

## 操作步骤

1. 在终端中启动平台聊天命令。
2. 如果你希望临时改用别的模型组，可以在启动这条命令时额外指定模型组；如果不指定，系统会使用当前默认模型组。
3. 命令启动后，系统会连接当前平台和模型。
4. 进入聊天状态后，在多行输入区域输入你的问题或指令。
5. 提交后，系统会返回当前模型的回复。
6. 如需继续对话，继续输入下一条内容并提交即可。
7. 如需退出聊天，直接输入 `/bye` 并提交。
8. 如果你提交空内容，当前聊天也会结束。

## 你会看到的提示与反馈

- 连接成功时，你会看到类似“已连接到某个平台某个模型”的提示。
- 进入聊天后，系统会提示你输入内容，并告知可以通过 `/bye` 退出。
- 每次模型返回内容时，回复会以带标题的消息块形式显示，标题中会带上当前平台和模型名称。
- 如果模型调用失败，你会看到“聊天失败”或初始化失败的提示。
- 输入 `/bye` 后，系统会提示“再见”。

## 注意事项

- 这是终端中的交互式聊天功能，不是通用代理或代码代理的任务执行界面。
- 这里主要用于验证当前平台与模型本身是否可正常响应。
- 退出方式有两种：输入 `/bye`，或者直接提交空内容。
- 如果你临时指定了模型组，这次聊天只对当前命令生效，不会自动改写默认配置。
