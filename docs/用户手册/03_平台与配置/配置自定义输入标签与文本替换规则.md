# 配置自定义输入标签与文本替换规则

## 使用场景

如果你经常输入重复的大段文本，例如“先分析再执行”“按团队规范输出”这类固定指令，可以把它们定义成短标签，在输入时由 Jarvis 自动展开。这个能力使用的配置项是 `replace_map`。

## 操作步骤

1. 打开 Jarvis 全局配置文件：`~/.jarvis/config.yaml`。
2. 在配置文件中添加或修改 `replace_map`。
3. 每个标签对应一个对象，常用字段有：
   - `template`：替换成的文本内容
   - `append`：是否在现有模板基础上追加
4. 例如：

   ```yaml
   replace_map:
     /spec:
       template: |
         请先给出任务理解、风险评估、执行步骤和验收标准。
       append: false

     /safe:
       template: |
         所有修改必须最小化，先读后写，不要臆测。
       append: false
   ```

5. 保存配置文件后，重新启动会话。
6. 在后续输入中使用你定义的标签，例如输入 `/spec` 或 `/safe`，观察它们是否被展开成对应文本。

## 你会看到的提示与反馈

- 这个功能通常不会弹出单独的“替换成功”提示。
- 最直接的验证方式，是看最终送入系统的内容是否已经按 `replace_map` 展开。
- 如果你定义了与内置规则同名的标签，自定义内容会覆盖同名内置项。
- 如果修改后没有效果，通常要先检查：
  - 配置文件是否写在 `~/.jarvis/config.yaml`
  - `replace_map` 的 YAML 缩进是否正确
  - 当前会话是否在修改后重新启动

## 注意事项

- 配置项名称是 `replace_map`，值是一个对象，不是数组。
- 标签建议带明显前缀，例如 `/spec`、`/safe`、`/jira`，避免和普通自然语言冲突。
- `template` 可以写多行文本，推荐用 YAML 的 `|` 写法保留格式。
- 如果替换内容很长，建议先在简单输入里测试一次，确认展开结果符合预期。
