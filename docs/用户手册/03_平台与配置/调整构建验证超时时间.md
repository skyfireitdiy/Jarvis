# 调整构建验证超时时间

## 使用场景

如果你的项目构建耗时较长，默认超时时间不够，或者你想让构建验证更快失败，可以调整 `build_validation_timeout`。

## 操作步骤

1. 打开 Jarvis 全局配置文件：`~/.jarvis/config.yaml`。
2. 添加或修改以下配置，例如改为 120 秒：

   ```yaml
   build_validation_timeout: 120
   ```

3. 保存配置文件。
4. 重新启动相关会话。
5. 触发一次会进行构建验证的代码修改流程。
6. 观察构建验证是否按新的超时时间执行。

## 你会看到的提示与反馈

- 如果构建在超时时间内完成，会正常返回验证结果。
- 如果超过你设置的时间，系统会终止验证并报告超时。
- 不配置时，默认超时时间是 30 秒。

## 注意事项

- 配置项名称是 `build_validation_timeout`。
- 这是数值型配置，单位是秒。
- 设置过短会导致本来能完成的构建频繁超时；设置过长会拖慢反馈速度。
