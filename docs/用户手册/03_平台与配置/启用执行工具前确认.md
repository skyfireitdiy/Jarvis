# 启用执行工具前确认

## 使用场景

如果你担心工具误执行带来风险，希望在真正执行工具前先人工确认，可以配置 `execute_tool_confirm`。

## 操作步骤

1. 打开 Jarvis 全局配置文件：`~/.jarvis/config.yaml`。
2. 添加或修改以下配置：

   ```yaml
   execute_tool_confirm: true
   ```

3. 保存配置文件。
4. 重新启动相关会话。
5. 在一次需要调用工具的任务中，观察系统是否在执行前先等待你的确认。

## 你会看到的提示与反馈

- 配置生效后，工具执行前会出现确认步骤，而不是直接执行。
- 这样可以降低误操作风险，尤其适合包含写文件、执行命令等高影响操作的场景。
- 如果你没有看到确认步骤，优先检查当前会话是否重新读取了配置，以及当前入口是否支持这套确认流程。

## 注意事项

- 配置项名称是 `execute_tool_confirm`。
- 这是布尔值，推荐使用 `true` 或 `false`。
- 开启确认后，交互成本会增加，但安全性更高。
