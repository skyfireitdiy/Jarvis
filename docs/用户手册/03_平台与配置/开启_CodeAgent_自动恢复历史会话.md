# 开启 CodeAgent 自动恢复历史会话

## 使用场景

如果你经常重复进入同一个项目，希望 CodeAgent 启动时自动尝试恢复最近一次会话，减少重复输入上下文，可以配置 `auto_resume_session`。

## 操作步骤

1. 打开 Jarvis 全局配置文件：`~/.jarvis/config.yaml`。
2. 添加或修改以下配置：

   ```yaml
   auto_resume_session: true
   ```

3. 保存配置文件。
4. 重新启动 CodeAgent。
5. 启动后，系统会尝试自动恢复最近的历史会话。
6. 观察启动提示，确认是否已经从某个会话文件恢复成功。

## 你会看到的提示与反馈

- 如果恢复成功，通常会看到“已从某个会话文件恢复会话”的提示。
- 如果没有可恢复会话，或恢复失败，系统会给出相应警告，然后继续进入正常输入流程。
- 如果你在命令行里显式指定恢复会话，显式参数会和自动恢复逻辑一起影响最终行为。

## 注意事项

- 配置项名称是 `auto_resume_session`。
- 默认行为是关闭，只有设为 `true` 才会自动恢复。
- 自动恢复并不保证每次都成功；如果会话文件不可用、恢复失败或当前环境不满足条件，系统会退回正常流程。
