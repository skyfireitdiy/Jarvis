# 扩展可用平台实现并让系统自动发现

## 使用场景

如果你希望 Jarvis 不只使用内置平台，还能接入你自己扩展的平台实现，可以把平台实现文件放到用户平台目录，让系统在启动时自动发现并注册。

## 操作步骤

1. 先准备好你的自定义平台实现文件。
2. 把它放到 Jarvis 的用户平台目录中。
3. 如果你不知道目录在哪，首次使用时系统会按自己的用户平台目录规则尝试创建；你可以通过平台相关功能启动后观察目录创建或扫描提示来确认。
4. 重新启动相关会话后，系统会在启动阶段同时扫描：
   - 用户平台目录
   - 内置平台目录
5. 如果你的实现满足平台要求，就会被注册进可用平台列表。
6. 之后再通过“查看当前平台与模型信息”或平台管理相关入口，确认新增平台是否已经出现。

## 你会看到的提示与反馈

- 如果用户平台目录不存在，系统会尝试自动创建这个目录。
- 如果目录创建失败，你会看到平台目录创建失败的错误提示。
- 如果某个平台实现缺少必要能力或接口不完整，系统会给出警告，并跳过该实现。
- 如果某个模块加载失败或注册失败，系统会输出相应错误，而不是默默当作成功。
- 配置正确时，你最终会在可用平台列表或平台相关功能中看到新增的平台名称。

## 注意事项

- 这个能力不是通过 `config.yaml` 里某一个键打开的，而是通过“把实现文件放到用户平台目录”生效。
- 不是把任意脚本丢进目录里就能被识别；只有满足平台实现要求的文件才会被注册。
- 自定义平台加载失败通常不会阻止内置平台继续使用，但你的扩展平台本身不会生效。
- 建议先在一个简单场景下验证新增平台能被发现，再继续把它接入更完整的会话或服务流程。
