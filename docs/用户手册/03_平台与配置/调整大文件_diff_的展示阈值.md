# 调整大文件 diff 的展示阈值

## 使用场景

如果你的项目经常修改大文件，希望了解在什么规模下 diff 会改用摘要展示而不是完整预览，需要关注大文件阈值行为。根据当前源码行为，这个阈值默认是 300 行。

## 操作步骤

1. 在大文件修改场景中观察 diff 是否已经从完整预览切换为更简化的展示。
2. 当前源码里，大文件阈值默认返回 `300`。
3. 当文件变更规模超过该阈值时，系统更倾向于只显示统计或摘要信息，而不是完整内容。

## 你会看到的提示与反馈

- 小规模修改时，通常能看到更完整的 diff。
- 超过阈值后，更容易看到摘要化或统计化展示。
- 这样做的目的是避免在大文件场景里输出过长内容，影响阅读和性能。

## 注意事项

- 当前源码体现的是默认阈值行为，不应写成“当前已经稳定开放给用户通过 config.yaml 任意修改”的能力，除非后续明确暴露配置项。
- 如果你需要精细控制这一阈值，应关注后续是否提供正式配置入口。
