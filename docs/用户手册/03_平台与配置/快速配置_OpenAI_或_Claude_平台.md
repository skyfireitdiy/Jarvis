# 快速配置 OpenAI 或 Claude 平台

## 使用场景

当你第一次接入 Jarvis，或者想把一套新的 OpenAI / Claude 平台配置快速写入本机配置文件时，可以使用这个交互式快速配置流程。它适合不想手动编辑配置文件、希望系统一步一步提示输入的人。

## 操作步骤

1. 在终端里启动快速配置命令，进入交互流程。
2. 按提示依次输入：
   - 平台类型：只支持 `claude` 或 `openai`
   - API 基础 URL
   - API 密钥
3. 系统会尝试读取可用模型。
   - 如果成功拿到模型列表，你会先看到一份可选模型清单。
   - 如果当前拿到多个模型，系统会先问你是否要一次配置全部模型。
4. 如果你不想全部配置，就按提示输入模型序号。多个模型可以用逗号分隔。
5. 如果模型列表里没有你要的模型，或者模型列表根本没有成功读取：
   - 系统会让你改为手动输入模型名称。
   - 这时你可以直接输入一个或多个模型名，多个模型之间用逗号分隔。
6. 选定本次要配置的模型后，系统会继续让你为这次模型组选择三类角色模型：
   - Normal：大多数普通场景使用
   - Smart：更偏向复杂任务或代码生成
   - Cheap：更偏向低要求、大数据量场景
7. 如果本机原来的配置文件里已经有其他模型，系统还可能先询问你：是否也允许从已有配置中的模型里挑选 Normal / Smart / Cheap。
8. 选完模型后，系统会先测试你选中的 Normal 模型是否能连通当前 API。
9. 如果连通性测试失败，系统会继续问你是否仍要保存配置：
   - 如果你选择不保存，流程会直接结束。
   - 如果你选择继续，系统仍会往下走并写入配置。
10. 接着按提示输入模型组名称；如果你直接回车，系统会使用一个基于 Normal 模型自动生成的默认名称。
11. 然后系统会逐个询问这次实际使用到的模型最大 token 数：
    - 直接回车时，会使用默认值或已有值。
    - 如果同一个模型同时被多个角色复用，系统只会问一次。
12. 全部完成后，系统会：
    - 复用已有的相同模型配置，或为新模型创建配置
    - 创建模型组
    - 把这个模型组设为当前默认模型组
    - 把结果保存到 `~/.jarvis/config.yaml`

## 你会看到的提示与反馈

- 一开始会看到类似：`🚀 开始配置 OPENAI 平台` 或 `🚀 开始配置 CLAUDE 平台` 的提示。
- 如果平台类型不是 `claude` 或 `openai`，系统会直接提示不支持该平台并结束。
- 成功拿到模型后，你会看到可用模型摘要，以及按编号列出的模型列表。
- 如果模型列表获取失败，你会看到失败提示，然后看到“是否手动输入模型名称”的确认。
- 当你完成模型选择后，会看到：`✅ 已选择 X 个模型: ...`。
- 在选择 Normal / Smart / Cheap 时，系统会逐项列出候选模型，并要求你输入编号。
- 选定后会看到类似：
  - `🎯 Normal模型: ...`
  - `🧠 Smart模型: ...`
  - `💰 Cheap模型: ...`
- 连通性测试时，你会看到：`🔍 正在测试模型 ... 的API连通性...`。
- 测试通过时会看到：`✅ API连通性测试通过`。
- 测试失败时会看到具体失败原因，并继续询问是否仍要保存配置。
- 保存成功后，你会看到这几类结果提示：
  - 已复用已有模型配置，或已为若干模型创建配置
  - 已创建模型组
  - 已设置默认模型组
  - `✅ 配置已保存到 ~/.jarvis/config.yaml`

## 注意事项

- 这个流程只明确支持 `claude` 和 `openai` 两种平台类型。
- 这是纯终端交互流程，不会打开浏览器页面。
- OpenAI 一类平台会尝试读取在线模型列表；Claude 这类流程会直接提供一组已知模型候选，因此“可用模型列表”不一定完全等于你账号下的全部模型。
- 如果模型列表没有成功获取，你仍可以手动输入模型名称继续配置。
- 连通性测试失败并不强制阻止保存；如果你选择继续保存，后续真正使用该模型时仍可能失败。
- 如果本地已经有相同模型的配置，系统可能直接复用原配置，而不是重复创建一份新的。
- 这里写入的是持久配置，并且会改写当前默认模型组，不等于某次会话里临时切换模型组。
