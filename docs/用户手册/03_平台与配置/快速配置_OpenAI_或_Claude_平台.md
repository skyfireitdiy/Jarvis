# 快速配置 OpenAI 或 Claude 平台

## 使用场景

当你第一次使用 Jarvis，或者想把 OpenAI、Claude 这类大模型平台快速接入到本地配置中时，可以使用快速配置流程。它适合还没有手动整理配置文件、希望系统一步步提示输入的人。

## 操作步骤

1. 在终端中启动 Jarvis 的快速配置命令，进入交互式配置流程。
2. 按提示依次输入以下内容：
   - LLM 平台类型：`claude` 或 `openai`
   - API 基础 URL
   - API 密钥

3. 如果系统成功获取到可用模型列表：
   - 你会先看到可用模型列表
   - 然后决定是否一次配置全部模型
   - 如果不想全部配置，可以按提示输入模型序号，多个序号用逗号分隔
   - 如果列表里没有你要的模型，也可以按提示切换为手动输入模型名称

4. 如果系统没有成功获取模型列表：
   - 系统会询问你是否手动输入模型名称
   - 这时你可以直接输入一个或多个模型名，多个模型用逗号分隔

5. 选定模型后，系统会继续提示你为当前模型组选择三类模型：
   - Normal：大多数普通场景使用
   - Smart：更偏向复杂任务或代码生成
   - Cheap：更适合低要求、大批量场景

6. 系统会测试你选择的 Normal 模型是否能连通当前 API。
7. 如果测试通过，继续按提示输入模型组名称；如果测试失败，系统会先询问你是否仍要保存配置。
8. 最后按提示为实际使用到的模型填写最大 token 数；直接回车时会使用默认值。
9. 配置完成后，系统会把结果保存到默认配置文件，并把这次创建的模型组设为当前默认模型组。

## 你会看到的提示与反馈

- 刚开始时，你会看到类似“开始配置平台”的提示。
- 如果模型列表获取失败，你会看到是否改为手动输入模型名称的确认提示。
- 选择模型后，你会看到已选择多少个模型，以及当前选中的模型名称。
- 在选择 Normal、Smart、Cheap 模型时，系统会把候选模型按编号列出。
- 测试 API 连通性时，你会看到“正在测试模型 API 连通性”的提示。
- 如果测试通过，会看到连通性测试成功提示。
- 如果测试失败，系统会告诉你失败原因，并询问是否仍要保存配置。
- 保存成功后，你会看到：
  - 已为模型创建配置
  - 已创建模型组
  - 已设置默认模型组
  - 配置已保存到 `~/.jarvis/config.yaml`

## 注意事项

- 这个快速配置流程只明确支持 `claude` 和 `openai` 两种平台类型。
- 这是终端里的交互式流程，不是浏览器页面。
- 如果你已经有旧配置，系统可能会复用已有模型配置，而不是重复创建完全相同的条目。
- 连通性测试失败时，你仍可以选择继续保存，但后续实际调用模型时仍可能失败。
- 这里保存的是持久配置，和在某次任务启动时临时传入模型组参数不是一回事。
