# 管理 LLM 配置

## 使用场景

当你已经接入过一个或多个模型平台，想查看当前都有哪些 LLM 配置、补充新的配置、修改已有配置，或者删除不再使用的配置时，可以使用平台管理器里的 LLM 配置管理功能。

## 操作步骤

### 查看 LLM 配置列表

1. 在终端中进入平台管理命令，并打开 LLM 配置管理。
2. 执行查看列表操作后，系统会列出当前所有 LLM 配置。
3. 列表里会直接显示每个配置名称，以及它对应的平台和模型。

### 查看单个 LLM 配置详情

1. 进入 LLM 配置详情查看操作。
2. 如果你在命令里已经直接带了配置名称，系统会直接展示这个配置。
3. 如果你没有直接带名称，系统会先弹出一个交互选择列表，让你从现有配置中挑选。
4. 选中后，系统会显示这个配置的主要信息，包括：
   - 平台
   - 模型
   - 最大 token 数
   - 其他附加连接配置

### 新增 LLM 配置

1. 进入新增 LLM 配置操作，并先给这条配置起一个名称。
2. 然后按提示依次输入：
   - 平台名称，例如 `openai`、`claude` 或其他自定义平台名
   - 模型名称
   - 最大 token 数
   - API 基础 URL
   - API 密钥
3. 输入完成后，系统会立即写入配置文件。
4. 如果新增成功，这条配置之后就可以被模型组引用。

### 更新 LLM 配置

1. 进入更新操作。
2. 如果没有直接指定名称，系统会先让你从现有配置中选择一个。
3. 选中后，系统会先显示当前值，再逐项让你输入新值。
4. 你可以只修改你想改的项；不想改的项直接留空即可。
5. 提交后，系统会保存更新结果。

### 删除 LLM 配置

1. 进入删除操作。
2. 如果没有直接指定名称，系统会先让你从现有配置中选择目标配置。
3. 如果这条配置仍被某个模型组引用，系统会先给出警告。
4. 随后系统会要求你确认删除。
5. 只有你明确确认后，这条配置才会真正从配置文件中移除。

## 你会看到的提示与反馈

- 查看列表时，你会看到类似：`✅ LLM 配置列表:`，后面跟着每个配置的 `名称 (平台/模型)`。
- 查看详情时，你会看到类似：
  - `✅ LLM 配置: 配置名`
  - `平台: ...`
  - `模型: ...`
  - `最大token数: ...`
  - `其他配置: ...`
- 新增开始时，系统会提示：`📝 添加 LLM 配置: 配置名`。
- 新增成功后，会提示：`✅ 已添加 LLM 配置: 配置名`。
- 更新开始时，系统会提示：`📝 更新 LLM 配置: 配置名`，并显示当前平台和模型。
- 更新成功后，会提示：`✅ 已更新 LLM 配置: 配置名`。
- 删除前如果该配置正被模型组使用，系统会先给出警告，而不是直接静默删除。
- 删除成功后，会提示：`✅ 已删除 LLM 配置: 配置名`。
- 如果你输入了不存在的名称，系统会提示：`❌ 未找到 LLM 配置: ...`。
- 如果你在选择列表中直接取消，当前操作会直接结束，不会继续往下执行。

## 注意事项

- LLM 配置保存在 `~/.jarvis/config.yaml` 中，属于持久配置。
- 新增时，平台名称、模型名称、API 基础 URL、API 密钥都不能为空。
- 新增时如果 token 数输入无效，系统会回退到默认值 `128000`。
- 更新时“留空不变”表示保持当前值，不是把该字段清空。
- 删除 LLM 配置前，最好先确认它是否正在被某个模型组使用，否则会影响已有模型组。
- 这里管理的是底层 LLM 配置，不等于在某次会话里临时切换模型组。
