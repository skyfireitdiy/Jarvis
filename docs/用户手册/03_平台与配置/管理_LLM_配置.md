# 管理 LLM 配置

## 使用场景

当你已经在 Jarvis 中接入了一个或多个模型平台，想查看现有配置、补充新的配置、修改旧配置，或者删除不再使用的配置时，可以使用平台管理命令里的 LLM 配置管理功能。

## 操作步骤

### 查看 LLM 配置列表

1. 在终端中进入平台管理命令。
2. 打开 LLM 配置管理后，执行查看列表操作。
3. 系统会列出当前已保存的 LLM 配置名称，并显示每个配置对应的平台和模型。

### 查看单个 LLM 配置详情

1. 在终端中进入 LLM 配置管理。
2. 选择查看详情操作。
3. 如果你没有在命令后直接带上配置名称，系统会进入交互选择列表，让你从现有配置中选择一个。
4. 选中后，系统会显示这个配置的：
   - 平台
   - 模型
   - 最大 token 数
   - 其他附加配置项

### 新增 LLM 配置

1. 在终端中进入 LLM 配置管理，并启动新增操作。
2. 输入新配置名称。
3. 按提示继续输入：
   - 平台名称，例如 `openai`、`claude` 或其他平台名
   - 模型名称
   - 最大 token 数
   - API 基础 URL
   - API 密钥

4. 输入完成后，系统会立即尝试保存配置。

### 更新 LLM 配置

1. 在终端中进入 LLM 配置管理，并启动更新操作。
2. 如果没有直接指定配置名称，系统会先弹出选择列表。
3. 选中目标配置后，系统会显示当前值。
4. 按提示逐项输入新值；如果某一项不想改，可以直接留空并提交。
5. 全部输入完成后，系统会保存更新结果。

### 删除 LLM 配置

1. 在终端中进入 LLM 配置管理，并启动删除操作。
2. 如果没有直接指定配置名称，系统会先让你从列表中选择。
3. 系统会显示确认删除提示。
4. 只有你明确确认后，系统才会真正删除该配置。

## 你会看到的提示与反馈

- 查看列表时，你会看到“LLM 配置列表”以及每个配置对应的平台和模型。
- 查看详情时，你会看到当前配置的详细字段。
- 新增成功后，系统会提示：`已添加 LLM 配置`。
- 更新成功后，系统会提示：`已更新 LLM 配置`。
- 删除成功后，系统会提示：`已删除 LLM 配置`。
- 如果名称不存在，你会看到“未找到 LLM 配置”之类的失败提示。
- 如果你在删除一个仍被模型组引用的配置，系统会先给出警告，再进入确认流程。

## 注意事项

- LLM 配置保存在 `~/.jarvis/config.yaml` 中，属于持久配置。
- 很多查看、更新、删除操作在你没有直接提供名称时，会进入交互选择模式。
- 更新时留空表示保持原值，不是清空当前字段。
- 删除前最好先确认这个配置是否还被某个模型组使用，否则会影响已有模型组。
- 这里管理的是底层 LLM 配置，不等于“当前会话临时切换模型组”。
