# 查看当前平台与模型信息

## 使用场景

当你想确认 Jarvis 现在默认会用哪个平台、哪个模型，或者想检查某个模型组对应的平台和候选模型列表是否正常时，可以使用这个查看功能。

## 操作步骤

### 查看默认配置对应的平台与模型

1. 在终端中启动平台信息查看命令。
2. 如果你不额外指定模型组，系统会直接读取当前默认模型组对应的配置。
3. 提交后，系统会先显示当前默认平台和默认模型。
4. 然后系统会继续尝试读取这个平台的模型列表详情。
5. 如果读取成功，你就能继续看到该平台下可用模型的列表或说明。

### 查看指定模型组对应的平台与模型

1. 在终端中启动同一个平台信息查看命令。
2. 在这次命令里额外指定你想查看的模型组。
3. 提交后，系统会优先使用你这次传入的模型组，而不是继续使用原来的默认组。
4. 系统会先显示“从模型组获取的配置信息”，再显示这个模型组当前使用的平台和模型。
5. 随后系统会继续尝试读取这个平台的模型列表详情。

## 你会看到的提示与反馈

- 不指定模型组时，你会看到类似：
  - `✅ 默认配置信息:`
  - `平台: ...`
  - `模型: ...`
- 指定模型组时，你会看到类似：
  - `✅ 从模型组 '...' 获取的配置信息:`
  - `平台: ...`
  - `模型: ...`
- 如果平台详情读取成功，还会看到类似：
  - `✅ 某平台详情`
  - `• 模型名 - 描述`
- 如果平台实例创建失败、当前配置不完整，或者平台没有返回模型列表，你会看到警告提示，而不是完整详情。
- 如果当前没有任何可用平台，也会直接显示警告并结束。

## 注意事项

- 这个功能只负责查看当前会生效的配置，不会修改任何配置文件。
- 如果你在命令里临时指定模型组，这次查看结果会以这次传入的模型组为准。
- 能否成功看到模型列表，取决于当前平台是否支持提供模型列表，以及当前配置是否足够让平台实例正常创建。
- 这是终端查看能力，不会打开浏览器，也不会进入聊天。
