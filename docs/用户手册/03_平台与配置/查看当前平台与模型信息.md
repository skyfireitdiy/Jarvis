# 查看当前平台与模型信息

## 使用场景

当你想确认 Jarvis 当前默认会使用哪个平台、哪个模型，或者想检查某个模型组对应的平台与候选模型列表时，可以使用平台信息查看功能。

## 操作步骤

### 查看默认配置对应的平台与模型

1. 在终端中启动平台信息查看命令。
2. 如果你不额外指定模型组，系统会直接读取当前默认配置。
3. 提交后，系统会显示当前默认平台和默认模型。
4. 如果平台支持返回模型列表，系统还会继续显示这个平台下可用的模型详情。

### 查看指定模型组对应的平台与模型

1. 在终端中启动平台信息查看命令。
2. 在这次命令里额外指定你想查看的模型组。
3. 提交后，系统会优先读取你这次指定的模型组，而不是继续使用配置文件中的默认值。
4. 系统会显示该模型组当前使用的平台和模型。
5. 如果能成功读取平台模型列表，还会继续显示该平台可用模型。

## 你会看到的提示与反馈

- 查看默认配置时，你会看到“默认配置信息”以及：
  - 平台
  - 模型

- 查看指定模型组时，你会看到“从模型组获取的配置信息”之类的提示。
- 如果平台详情可以正常读取，你还会看到平台名称以及模型列表。
- 如果当前没有可用平台，或者平台详情读取失败，你会看到警告信息，而不是完整模型列表。

## 注意事项

- 这个功能主要用于查看当前会生效的默认平台与模型，不会修改你的配置。
- 如果你在命令中临时指定模型组，当前查看结果以这次命令传入的模型组为准。
- 平台模型列表是否能完整显示，取决于当前平台是否支持返回模型列表，以及当前配置是否可正常建立连接。
- 这是终端查看功能，不会打开浏览器页面。
