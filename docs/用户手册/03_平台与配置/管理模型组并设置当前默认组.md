# 管理模型组并设置当前默认组

## 使用场景

当你希望把多个模型配置组合成一套固定策略，例如把普通任务、复杂任务和低成本任务分别绑定到不同模型上，或者想切换 Jarvis 当前默认使用的模型组时，可以使用模型组管理功能。

## 操作步骤

### 查看模型组列表

1. 在终端中进入平台管理命令。
2. 打开模型组管理后，执行查看列表操作。
3. 系统会列出当前已有的模型组名称，并显示每个模型组对应的 Normal 配置。

### 查看模型组详情

1. 在终端中进入模型组管理。
2. 选择查看详情操作。
3. 如果没有直接指定模型组名称，系统会先弹出交互选择列表。
4. 选中后，系统会显示当前模型组中的：
   - `normal_llm`
   - `cheap_llm`
   - `smart_llm`

### 新增模型组

1. 在终端中进入模型组管理，并启动新增操作。
2. 输入模型组名称。
3. 系统会先列出当前可用的 LLM 配置名称。
4. 按提示输入：
   - Normal LLM 配置名称（必填）
   - Cheap LLM 配置名称（可选，留空时与 Normal 相同）
   - Smart LLM 配置名称（可选，留空时与 Normal 相同）

5. 输入完成后，系统会保存这个模型组。

### 更新模型组

1. 在终端中进入模型组管理，并启动更新操作。
2. 如果没有直接指定名称，系统会先让你从现有模型组中选择一个。
3. 系统会显示当前的 Normal、Cheap、Smart 值。
4. 你可以逐项输入新配置名称；不想修改的项可以直接留空。
5. 提交后，系统会保存更新结果。

### 删除模型组

1. 在终端中进入模型组管理，并启动删除操作。
2. 如果没有直接指定名称，系统会先进入交互选择。
3. 系统会弹出确认删除提示。
4. 只有你确认后，删除才会真正执行。

### 设置当前默认模型组

1. 在终端中进入模型组管理。
2. 启动设置当前模型组的操作。
3. 如果没有直接指定名称，系统会让你先从现有模型组中选择。
4. 确认后，系统会把这个模型组写入当前默认配置。

## 你会看到的提示与反馈

- 查看列表时，你会看到“模型组列表”以及每个组的 Normal 配置。
- 查看详情时，你会看到当前组对应的 Normal、Cheap、Smart 配置名称。
- 新增成功后，系统会提示：`已添加模型组`。
- 更新成功后，系统会提示：`已更新模型组`。
- 删除成功后，系统会提示：`已删除模型组`。
- 设置成功后，系统会提示：`已设置当前模型组`。
- 如果你输入了不存在的 LLM 配置名称，系统会立即报错并终止保存。

## 注意事项

- 创建模型组前，必须先有可用的 LLM 配置；如果当前没有任何 LLM 配置，系统不会让你继续新增模型组。
- 设置当前默认模型组属于持久配置，会影响之后默认启动时使用的模型策略。
- 更新时留空表示保持不变。
- 删除模型组前，建议先确认当前是否还需要用它作为默认模型组。
- 这里设置的是默认模型组，和某次启动命令里临时覆盖模型组不同。
