# 管理模型组并设置当前默认组

## 使用场景

当你想把多个 LLM 配置组合成一套固定策略，例如把普通场景、复杂任务和低成本场景分别绑定到不同模型上，或者想切换 Jarvis 后续默认使用的模型组时，可以使用模型组管理功能。

## 操作步骤

### 查看模型组列表

1. 在终端中进入平台管理命令，并打开模型组管理。
2. 执行查看列表操作后，系统会列出当前已有模型组。
3. 列表中会显示每个模型组名称，以及它当前的 Normal 配置引用。

### 查看模型组详情

1. 进入模型组详情查看操作。
2. 如果你没有直接传入模型组名称，系统会先弹出交互选择列表。
3. 选中后，系统会显示这个模型组当前引用的：
   - `normal_llm`
   - `cheap_llm`
   - `smart_llm`

### 新增模型组

1. 进入新增模型组操作，并先给模型组起一个名称。
2. 系统会先列出当前所有可用的 LLM 配置名称，供你参考。
3. 然后按提示输入：
   - Normal LLM 配置名称（必填）
   - Cheap LLM 配置名称（可选）
   - Smart LLM 配置名称（可选）
4. 如果 Cheap 或 Smart 留空，系统会按“与 Normal 相同”的方式处理。
5. 输入完成后，系统会把这个模型组写入配置文件。

### 更新模型组

1. 进入更新操作。
2. 如果没有直接指定模型组名称，系统会先让你从现有模型组中选择一个。
3. 选中后，系统会显示当前的 Normal / Cheap / Smart 值，并再次列出可用的 LLM 配置名称。
4. 你可以只修改其中某几项；不想改的项直接留空即可。
5. 提交后，系统会保存更新结果。

### 删除模型组

1. 进入删除操作。
2. 如果没有直接指定模型组名称，系统会先进入交互选择。
3. 系统会要求你确认删除。
4. 只有你明确确认后，删除才会真正执行。

### 设置当前默认模型组

1. 进入设置默认模型组操作。
2. 如果没有直接指定模型组名称，系统会先让你从现有模型组里选择一个。
3. 选定后，系统会把这个模型组写入当前默认配置。
4. 从这之后，未显式覆盖模型组的相关功能都会默认使用它。

## 你会看到的提示与反馈

- 查看列表时，你会看到类似：`✅ 模型组列表:`，后面跟着每个组的 `名称 (normal: ...)`。
- 查看详情时，你会看到类似：
  - `✅ 模型组: 组名`
  - `normal_llm: ...`
  - `cheap_llm: ...`
  - `smart_llm: ...`
- 新增开始时，系统会提示：`📝 添加模型组: 组名`，随后列出可用的 LLM 配置。
- 如果你输入了不存在的 LLM 配置名，系统会立即提示：`❌ 未找到 LLM 配置: ...`，并结束这次保存。
- 新增成功后，会提示：`✅ 已添加模型组: 组名`。
- 更新开始时，会提示：`📝 更新模型组: 组名`，并显示当前值。
- 更新成功后，会提示：`✅ 已更新模型组: 组名`。
- 删除前会出现确认步骤；如果你取消，当前删除流程会直接结束。
- 删除成功后，会提示：`✅ 已删除模型组: 组名`。
- 设置默认组成功后，会提示：`✅ 已设置当前模型组: 组名`。

## 注意事项

- 创建模型组前，必须先有可用的 LLM 配置；如果当前一个都没有，系统不会让你继续新增模型组。
- 新增或更新模型组时，输入的配置名称必须是真实存在的 LLM 配置名。
- 更新时留空表示保持原值，不是清空当前字段。
- 设置当前默认模型组属于持久配置，会影响后续默认启动时的模型选择。
- 这里设置的是“默认模型组”；如果某次命令额外传入了模型组参数，那次运行仍会以临时传入的值为准。
