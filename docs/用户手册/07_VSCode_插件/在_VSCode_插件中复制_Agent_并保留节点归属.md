# 在 VSCode 插件中复制 Agent 并保留节点归属

## 使用场景

当你已经有一个配置合适的 Agent，希望快速复制一个副本继续工作，同时尽量保留原 Agent 所属节点时，可以使用插件列表中的复制功能。

## 操作步骤

1. 在 VSCode 插件侧边栏中找到目标 Agent。
2. 点击该 Agent 右侧的复制按钮。
3. 插件会根据原 Agent 的类型、工作目录、模型组和节点信息向主节点发送复制请求。
4. 复制成功后，列表会刷新，并自动切换到新副本。

## 你会看到的提示与反馈

- 新副本通常会保留原 Agent 的节点归属。
- 刷新后，你可以通过节点标识确认副本所在节点。
- 如果复制失败，插件会弹出错误提示。

## 注意事项

- 节点归属是否完全保留，最终取决于主节点的创建路由与节点可用性。
- 即使副本运行在子节点，插件仍然通过主节点访问它。
