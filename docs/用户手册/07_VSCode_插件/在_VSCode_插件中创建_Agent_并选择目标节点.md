# 在 VSCode 插件中创建 Agent 并选择目标节点

## 使用场景

当你想直接在 VSCode 插件中创建新的通用 Agent 或代码 Agent，并决定它应该运行在主节点还是某个子节点时，可以使用插件侧边栏中的创建 Agent 表单。

## 操作步骤

1. 先确保插件已经连接到主节点 Gateway。
2. 在侧边栏打开创建 Agent 面板。
3. 依次填写 Agent 类型、工作目录、名称、模型组等参数。
4. 如果当前主节点已发现子节点，在“目标节点”下拉框中选择目标节点；如果不选择，则默认创建到主节点。
5. 如有需要，再决定代码 Agent 是否启用 worktree。
6. 点击“创建”。
7. 创建成功后，插件会自动刷新 Agent 列表，并切换到新 Agent。

## 你会看到的提示与反馈

- 创建表单中会显示模型组与目标节点选项。
- 如果当前没有可用子节点，目标节点通常只表现为默认主节点语义。
- 创建成功后，新 Agent 会显示在侧边栏列表中，并带有对应节点标识。

## 注意事项

- 目标节点列表来自主节点的节点状态接口。
- 创建到子节点后，后续聊天、HTTP 请求和 WebSocket 通信仍由主节点转发，插件端不需要手工切换连接地址。
