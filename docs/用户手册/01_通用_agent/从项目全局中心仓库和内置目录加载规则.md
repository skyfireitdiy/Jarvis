# 从项目、全局、中心仓库和内置目录加载规则

## 使用场景

当你希望同时使用项目专属规则、个人长期规则、团队共享规则或系统内置规则时，Jarvis 会把多个来源里的规则统一纳入可用列表，供你后续查看和选择。

## 操作步骤

1. 先把规则放到对应来源中，例如项目规则目录、个人全局规则目录、团队共享规则仓库或额外配置目录。
2. 启动 Jarvis 进入交互对话。
3. 在输入框中输入 `'<ListRule>'`，按 `Ctrl+J` 或 `Ctrl+]` 提交。
4. 系统会把当前已识别到的规则统一列出来，你可以从列表里看到不同来源下的规则项。
5. 如需精确指定某个来源的规则，再使用带前缀的规则名继续后续操作。

## 你会看到的提示与反馈

- 列表里会统一显示不同来源的规则，而不是只显示某一个目录下的内容。
- 你可以通过 `规则名称` 和 `文件路径` 判断它来自哪里。
- 列表会同时展示 `内容预览` 和 `状态`，方便你先辨认再决定是否使用。
- 如果某个共享来源当前不可用，对应规则可能不会出现在列表里。

## 注意事项

- 这篇文档描述的是“统一被识别并列入可用列表”，不代表所有规则都会自动立即生效。
- 项目规则更适合当前仓库专属约束，全局规则更适合个人长期偏好。
- 团队共享规则依赖外部来源可访问；如果该来源没有同步成功，列表中可能看不到对应规则。
