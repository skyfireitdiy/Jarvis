# 在开启确认后手动决定是否执行工具调用

## 使用场景

当你不希望代理未经确认就直接操作文件、执行命令或调用外部能力时，可以打开确认流程，由你手动决定每次工具调用是否真正执行。

## 操作步骤

1. 先进入已经开启工具执行确认的会话或启动方式。
2. 正常提交任务，让代理规划并尝试调用工具。
3. 当系统展示完工具名和参数摘要后，会进一步询问你是否确认执行。
4. 根据当前情况选择继续或取消。

## 你会看到的提示与反馈

- 在工具真正执行前，系统会询问：`需要执行工具名确认执行？`
- 如果你确认，工具会继续真正执行。
- 如果你取消，本次工具调用不会执行，系统会直接结束这一轮工具执行流程。
- 如果工具执行时发生错误，系统会提示：`❌ 工具执行失败: ...`

## 注意事项

- 这项能力依赖当前入口已经开启工具确认，不是所有会话都会自动弹出这一步。
- 这个确认点适合拦截高风险操作，例如写文件、执行脚本、网络请求等。
- 如果你频繁取消关键工具，代理可能无法完成当前任务。
