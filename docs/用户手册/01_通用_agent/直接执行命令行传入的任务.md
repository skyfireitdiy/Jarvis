# 直接执行命令行传入的任务

## 使用场景

适合批处理、脚本调用，或者你已经把任务写完整，不想再进入交互式输入界面时使用。

## 操作步骤

1. 启动通用代理时，直接通过 `--task` 传入任务内容。
2. 提交命令后，系统不会再询问你输入任务。
3. 启动阶段会先输出一条提示，说明当前正在执行命令行传入的任务。
4. 随后代理立即开始处理。

## 你会看到的提示与反馈

- 系统会打印一条信息，表示正在执行你通过命令行传入的任务。
- 不会出现 `请输入你的任务（Ctrl+C 退出）` 这个多行输入提示。
- 如果任务正常开始，说明命令行任务已被成功接收。

## 注意事项

- 这种方式适合一次性明确任务，不适合需要边运行边补充需求的场景。
- 如果任务内容很长、很复杂，建议先检查命令是否正确，再执行。
- 后续若仍需频繁调整任务，建议改用交互式输入。
