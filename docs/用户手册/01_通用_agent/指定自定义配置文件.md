# 指定自定义配置文件

## 使用场景

当你需要临时改用另一套配置，例如切换默认行为、加载不同设置组合，或者在某个项目下使用专门配置时，可以在启动时指定配置文件。

## 操作步骤

1. 先准备好配置文件路径。
2. 启动通用代理时，使用 `-f` 或 `--config` 指定该文件。
3. 如果还需要加载代理定义，可另外配合对应启动参数一起使用。
4. 启动成功后，继续像平常一样输入任务或直接执行任务。

## 你会看到的提示与反馈

- 如果配置文件路径不存在，系统会直接提示配置文件未找到，并回退为空配置处理。
- 如果配置文件格式有问题，系统会提示配置解析失败。
- 如果启动流程继续进行，说明配置已被读取并应用到当前会话。

## 注意事项

- 配置文件路径建议写成你自己容易确认的绝对路径或明确的相对路径。
- 如果你发现启动后的行为和预期不一致，优先检查是否用了错误的配置文件。
- 配置解析失败时，先修正文件内容，再重新启动。
