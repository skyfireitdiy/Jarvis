# 在输入中引用文件并自动附加文件符号信息

## 使用场景

当你希望代理理解某个文件的大致结构，而不是只看到一个文件名时，可以在输入里直接引用文件路径，让系统自动补充文件中的结构化符号信息。

## 操作步骤

1. 在当前任务输入里，使用单引号包住文件路径，例如把目标文件写成 `'路径/文件名'` 的形式。
2. 按平时的方式提交任务。
3. 系统会检查这些路径是否指向真实存在的文本文件。
4. 如果识别成功，系统会自动把该文件的结构化符号信息附加到你的输入后面，再交给代理处理。

## 你会看到的提示与反馈

- 只有写在单引号里的路径才会被识别为待分析文件。
- 如果文件存在且可读取，系统会自动补充该文件的结构信息，而不需要你手动整理。
- 如果路径不存在、不是文本文件，或无法提取出结构信息，系统通常不会额外报错，只是不会附加这部分上下文。

## 注意事项

- 这种方式更适合源码、配置、脚本文本等可读取文件。
- 如果你没有用单引号包住路径，系统不会按文件引用处理。
- 这项能力是为了帮助代理更快理解文件结构，不等于自动替你完成全文分析。
