# 通过统一的用户交互封装切换多行输入与确认实现

## 使用场景

当同一套 Agent 需要同时适配 CLI、Web 或其他交互界面时，统一的用户交互封装可以让多行输入和确认逻辑保持一致。

## 操作步骤

1. 正常进入需要多行输入或确认的流程，例如输入任务、补充说明或回答确认问题。
2. 在 CLI 中，你通常会看到多行输入提示，并使用 `Ctrl+J` 或 `Ctrl+]` 提交；在其他界面中，可能会看到按钮、对话框或不同样式的输入框。
3. 不管界面外观怎么变化，系统都会把这些输入和确认动作按同样的任务逻辑继续处理。

## 你会看到的提示与反馈

- 多行输入仍会表现为同样的任务输入流程。
- 确认操作仍会表现为一致的确认问题和确认结果。
- 这意味着你在不同前端上看到的细节可能略有差异，但操作语义保持一致。

## 注意事项

- 这项能力更偏底层交互兼容，对普通用户来说通常是“无感”的。
- 如果你发现不同界面的输入行为略有差别，不代表任务逻辑不同，通常只是输入实现不同。
- 文档中的按键提示主要以 CLI 体验为准，其他界面可能用按钮或不同交互替代。
