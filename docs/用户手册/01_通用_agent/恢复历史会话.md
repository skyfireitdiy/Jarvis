# 恢复历史会话

## 使用场景

当你中断过某个任务，之后想在同一个项目里继续做，而不想重新解释背景时，可以恢复之前保存的会话。

## 操作步骤

1. 先回到原来的项目目录，再启动 Jarvis。
2. 如果当前入口支持会话恢复，你可以在对话输入框直接输入 `'<RestoreSession>'`，然后按 `Ctrl+J` 或 `Ctrl+]` 提交。
3. 系统会列出可恢复的历史会话，并提示你输入编号。
4. 按提示输入要恢复的序号；如果你暂时不想恢复，直接回车跳过即可。
5. 恢复成功后，继续在原来的上下文上接着工作。

## 你会看到的提示与反馈

- 选择前会看到提示：`是否恢复会话？（输入序号恢复，直接回车跳过）:`
- 列表中通常会包含：
  - 文件名
  - 时间戳
  - 会话名称
- 恢复成功后，会提示：`✅ 会话已成功恢复。`
- 恢复失败时，会提示：`❌ 恢复会话失败。`
- 如果当前入口不允许使用这个命令，你会先看到权限不足的提醒，而不会直接恢复。

## 注意事项

- 恢复会话依赖原项目环境，最好回到保存该会话时的同一项目目录。
- 直接回车跳过不会报错，只是继续当前新会话。
- 某些旧会话可能因为当前代码状态、模型限制或入口权限变化而无法完整恢复。
