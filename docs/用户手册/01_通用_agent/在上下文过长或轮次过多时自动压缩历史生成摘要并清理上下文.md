# 在上下文过长或轮次过多时自动压缩历史、生成摘要并清理上下文

## 使用场景

当你的会话已经非常长，或者多轮对话快要逼近模型的输入上限时，系统会自动压缩上下文，避免因为历史过长而无法继续工作。

## 操作步骤

1. 正常持续进行多轮对话。
2. 当上下文剩余空间不足，或对话轮次超过阈值时，系统会自动触发压缩，你不需要先手动输入某个控制命令。
3. 系统会优先尝试自适应压缩。
4. 如果自适应压缩不成功，系统会回退到完整摘要压缩方案，然后再继续后续对话。

## 你会看到的提示与反馈

- 当触发压缩时，系统会提示触发原因，例如：
  - `🔍 Token限制触发...`
  - `🔍 对话轮次限制触发...`
  - `🔍 Token和轮次双重限制触发...`
- 如果当前消息本身过长，系统可能先提示内容过长并截断中间部分。
- 压缩成功后，你会看到：
  - `✅ 自适应压缩完成，对话上下文已更新`
  - 或 `✅ 完整摘要压缩完成，对话上下文已更新`
- 如果截断后已经足够，系统会提示：`✅ 截断后无需压缩`

## 注意事项

- 自动压缩的目标是保住任务连续性，不是保留每一句原始对话细节。
- 对特别关键的信息，建议你提前固定或重新明确说明，不要完全依赖历史自动保留。
- 如果你发现代理在超长会话后理解出现偏差，应优先检查是否发生过摘要压缩。
