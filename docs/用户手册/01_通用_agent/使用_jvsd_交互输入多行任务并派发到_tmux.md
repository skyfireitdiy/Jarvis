# 使用 jvsd 交互输入多行任务并派发到 tmux

## 使用场景

当你的任务内容较长、需要分段组织，但又希望把执行过程派发到独立终端面板里运行时，可以使用 `jvsd` 的多行输入与 tmux 派发能力。

## 操作步骤

1. 直接执行 `jvsd`，不带任务参数。
2. 系统会进入多行输入，并提示：`请输入任务内容（Ctrl+J/Ctrl+] 确认，Enter 换行）`。
3. 输入完成后提交。
4. 系统会自动在任务前补充内置规则入口和“原始需求”前缀。
5. 然后把完整任务写入临时文件，并提示临时文件位置。
6. 最终通过 tmux 面板派发执行。

## 你会看到的提示与反馈

- 多行任务会先被写入临时文件，并显示：`📝 临时文件已创建: ...`。
- 派发模式下，系统会尝试创建 tmux panel 来执行任务。
- 如果 tmux 派发失败，会看到类似：
  - `❌ 错误: dispatch 模式创建 tmux panel 失败`
  - 或 `❌ dispatch 失败: ...`
- 如果你直接提交空内容，系统会提示后退出，不会继续派发。

## 注意事项

- 这项能力适合较长任务，不适合只输入一句很短的话。
- 使用前最好确认当前环境支持 tmux。
- 多行派发更适合后台持续执行或希望把任务隔离到独立面板的场景。
