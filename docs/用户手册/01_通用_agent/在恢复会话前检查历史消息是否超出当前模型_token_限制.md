# 在恢复会话前检查历史消息是否超出当前模型 token 限制

## 使用场景

当你准备恢复一个很长的历史会话时，系统会先检查这些历史消息是否已经超过当前模型能承载的输入上限，避免刚恢复就立刻超限失败。

## 操作步骤

1. 先回到原项目目录并启动 Jarvis。
2. 如果当前入口支持会话恢复，在对话输入框中输入 `'<RestoreSession>'`，再按 `Ctrl+J` 或 `Ctrl+]` 提交。
3. 选中要恢复的会话后，系统会先统计这段历史消息的 token 使用量。
4. 系统再把这个结果与当前模型组的最大输入限制比较。
5. 如果已经超限，就直接阻止恢复；如果接近上限，也会先提醒你风险。

## 你会看到的提示与反馈

- 在恢复过程中，系统会打印统计信息：`📊 会话token统计: 已使用 X, 最大限制 Y`
- 如果已经超出上限，系统会阻止恢复，并提示：`❌ 会话恢复失败：历史消息的token数量超出当前模型的限制。`
- 如果虽然没超，但已经非常接近上限，系统也会先给出警告，提醒你恢复后可用空间有限。

## 注意事项

- 这一步检查的是当前模型，不同模型组的可承载上限可能不同。
- 同一个会话在一种模型下能恢复，不代表在另一种模型下也一定能恢复。
- 如果遇到超限问题，可以先切换到支持更大上下文的模型，再重新尝试恢复。
