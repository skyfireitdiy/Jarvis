# 自动保存会话到项目下的 `.jarvis/sessions` 目录

## 使用场景

当你希望当前任务可以在之后继续接着做，而不是每次都从头开始时，可以把当前会话保存到项目目录里，方便后续恢复。

## 操作步骤

1. 在当前项目里正常对话，确保这次会话里已经有真实的用户输入内容。
2. 如果当前入口支持会话保存，在输入框中输入 `'<SaveSession>'`。
3. 按 `Ctrl+J` 或 `Ctrl+]` 提交。
4. 保存成功后，当前会话会写入项目下的 `.jarvis/sessions` 目录，供之后恢复使用。

## 你会看到的提示与反馈

- 保存成功时，会提示：`✅ 会话已成功保存。`
- 保存失败时，会提示：`❌ 保存会话失败。`
- 如果当前入口不允许使用这个命令，你会先看到权限不足提醒。
- 如果当前会话几乎没有实际用户消息，系统可能不会保存出可用的历史会话。

## 注意事项

- 会话是按当前项目目录保存的，所以恢复时最好回到同一个项目环境。
- 这个功能依赖当前入口是否允许使用保存命令，并不是所有启动方式都一定可用。
- 如果你希望长期保留某个工作过程，建议在关键节点主动保存，而不要只依赖临时上下文。
