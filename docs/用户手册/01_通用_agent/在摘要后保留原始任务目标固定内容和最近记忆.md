# 在摘要后保留原始任务目标、固定内容和最近记忆

## 使用场景

当对话已经很长并触发摘要压缩时，你可能最担心的是：原始任务跑丢了、之前固定的重要要求被忘了、最近关键记忆消失了。这个功能就是为了解决这个问题。

## 操作步骤

1. 正常进行长对话，直到系统自动触发摘要压缩。
2. 如果你之前已经固定过长期要求，或系统已经保存了最近记忆，这些内容会在压缩时被优先整理，不需要你重新手动输入一遍。
3. 压缩完成后，继续基于新的摘要上下文工作。

## 你会看到的提示与反馈

- 摘要内容会被整理成新的上下文，而不是简单清空历史后重新开始。
- 系统会优先保留以下内容：
  - 原始任务目标
  - 通过固定功能保存的重要要求
  - 最近记忆
- 这些内容会被放在摘要前部，作为后续任务继续执行时的高优先级信息。

## 注意事项

- 如果某些信息特别关键，最好提前固定，不要只依赖系统自动记住。
- 最近记忆只会保留一部分，不代表所有历史细节都会完整留下。
- 摘要后的上下文更偏向“保主线”，不适合指望它保留每一句原始对话。
