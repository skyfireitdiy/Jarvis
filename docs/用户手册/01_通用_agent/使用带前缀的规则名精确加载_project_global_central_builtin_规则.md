# 使用带前缀的规则名精确加载 project / global / central / builtin 规则

## 使用场景

当不同来源里存在同名规则，或者你明确知道要使用某个来源下的那一条规则时，可以给规则名加前缀，减少混淆。

## 操作步骤

1. 先输入 `'<ListRule>'` 并按 `Ctrl+J` 或 `Ctrl+]`，确认目标规则确实存在。
2. 记下规则名称，并确认它属于哪个来源。
3. 在后续引用该规则时，直接使用带前缀的规则名，例如：
   - `builtin:规则名`
   - `project:规则名`
   - `global:规则名`
   - `central:规则名`
   - `configN:规则名`
4. 提交后，系统会优先按你指定的来源去查找该规则，而不是再做模糊匹配。

## 你会看到的提示与反馈

- 使用带前缀的规则名后，系统会按指定来源处理，不再优先猜测你想用哪一份同名规则。
- 如果该来源下存在对应规则，后续流程会按这条规则继续。
- 如果该来源下没有这条规则，你会遇到找不到对应规则的情况，需要重新检查前缀或规则名。

## 注意事项

- 这类前缀适合在同名规则较多时使用，能明显减少歧义。
- 前缀写错、来源写错或规则名写错，都会导致无法命中目标规则。
- 如果你不确定规则是否存在，先用 `'<ListRule>'` 看列表，再写带前缀名称最稳妥。
