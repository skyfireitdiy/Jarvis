# 以非交互模式运行通用代理

## 使用场景

适合脚本化执行、无人值守处理，或者你明确不希望在运行过程中再手动输入内容的场景。

## 操作步骤

1. 启动时加上 `-n` 或 `--non-interactive`。
2. 同时必须使用 `-T` 或 `--task` 直接提供任务内容。
3. 提交命令后，系统会直接进入执行，不再等待你在终端里继续补充输入。

## 你会看到的提示与反馈

- 如果你只开了非交互模式，却没有提供任务，系统会直接报错：非交互模式必须配合 `--task` 使用，然后退出。
- 如果参数完整，系统会继续启动，并立即执行你传入的任务。
- 在这种模式下，不会出现交互式多行输入提示。

## 注意事项

- 非交互模式最重要的前提，就是任务必须在启动时一次性写完整。
- 这种方式不适合需要中途不断补充说明的任务。
- 如果你只是想减少交互，但仍可能临时修改要求，建议继续使用交互式方式。
