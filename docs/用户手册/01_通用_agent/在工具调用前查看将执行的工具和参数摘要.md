# 在工具调用前查看将执行的工具和参数摘要

## 使用场景

当你希望在代理真正调用工具之前，先检查它准备做什么、参数是否合理时，可以使用工具调用前的预览能力。

## 操作步骤

1. 先进入开启了工具确认的会话或启动方式。
2. 正常输入你的任务，并按常规方式提交。
3. 当代理准备调用工具时，不会立刻直接执行，而是先展示工具名称和参数摘要。
4. 先看清这次准备执行什么，再决定是否继续放行。

## 你会看到的提示与反馈

- 如果本轮只有一个工具调用，系统会显示：`🔧 准备执行工具: 工具名`
- 如果同时准备执行多个工具调用，系统会显示：`🔧 准备执行 N 个工具调用:`，并逐项列出编号、工具名和参数摘要
- 参数摘要会自动压缩较长内容，并对敏感字段做隐藏处理
- 之后系统还会继续进入确认流程，让你决定这次工具调用是否执行

## 注意事项

- 你看到的是摘要，不一定是全部原始参数全文。
- 如果摘要已经明显不对，通常说明代理的下一步动作需要被拦下或修正。
- 这项能力只有在确认流程开启时才会先展示；如果没开启，工具可能直接执行而不会先预览。
