# 在对话中直接执行以 `!` 开头的 Shell 脚本并决定是否回传结果

## 使用场景

当你在对话过程中需要临时执行一段命令，再决定是否把执行结果交给代理继续分析时，可以直接在输入中写 `!` 开头的命令。

## 操作步骤

1. 在输入中编写以 `!` 开头的命令或多行脚本。
2. 提交后，系统会先把将要执行的脚本内容显示出来。
3. 系统会询问：`是否要执行以上shell脚本？`
4. 如果你确认执行，系统会运行该脚本。
5. 脚本执行后，系统会再询问：`是否将执行结果反馈给Agent？`
6. 你可以选择只执行不回传，或把结果继续交给代理处理。

## 你会看到的提示与反馈

- 输入中所有以 `!` 开头的行都会被当作脚本内容处理。
- 执行前你会先看到整理后的脚本正文。
- 如果你选择回传结果，代理后续会基于“执行了什么”和“输出了什么”继续处理。
- 如果你不回传结果，本次脚本执行就只停留在本地执行层，不会继续进入代理分析。

## 注意事项

- 这是直接执行命令的能力，使用前应确认脚本内容安全且符合你的预期。
- 如果你只是想让代理阅读命令而不是实际运行，不要用 `!` 开头。
- 某些由快捷键自动生成的终端命令会跳过首次确认，但普通手动输入的 `!` 命令会经过确认流程。
