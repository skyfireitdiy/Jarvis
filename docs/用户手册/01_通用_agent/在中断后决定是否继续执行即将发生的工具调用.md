# 在中断后决定是否继续执行即将发生的工具调用

## 使用场景

当你在代理运行过程中发现它接下来可能要做某个工具操作，但你还想先干预一下时，可以通过中断为自己争取一次人工决策机会。

## 操作步骤

1. 在代理正在输出时，按中断快捷键打断当前输出。
2. 中断后，系统会提示你输入补充信息：`⚠ 检测到中断，请输入补充信息（Ctrl+J/Ctrl+]确认，直接回车跳过）`
3. 如果你有新的限制或修正意见，就输入内容后按 `Ctrl+J` 或 `Ctrl+]` 提交；如果没有补充，直接回车跳过。
4. 如果后续仍有工具调用准备执行，并且当前会话开启了工具确认，系统会继续进入工具确认流程。
5. 在确认界面里，再决定这次工具调用是否继续执行。

## 你会看到的提示与反馈

- 中断本身不会强制替你取消所有后续动作，而是先给你插入补充信息的机会。
- 中断后你会先看到：`⚠ 检测到中断，请输入补充信息（Ctrl+J/Ctrl+]确认，直接回车跳过）`
- 如果工具确认功能处于开启状态，你仍然可以在工具真正执行前再次决定是否放行。
- 这样可以避免代理在你已经发现问题后，仍然继续执行原计划中的工具操作。

## 注意事项

- 中断和工具确认是两层不同的保护：前者用于补充说明，后者用于最终放行或取消执行。
- 如果没有开启工具确认，中断后的工具是否继续执行，取决于后续流程是否仍然认为需要执行该动作。
- 对高风险操作，建议同时使用中断能力和工具确认能力。
