# 通过内置标记执行总结、清空历史、重载配置、查看工具说明和规则状态

## 使用场景

当你已经在会话中工作了一段时间，想快速触发系统操作，而不是退出重来时，可以直接在输入里使用内置标记。

## 操作步骤

1. 在当前对话输入框中写入对应的内置标记，例如：
   - `'<Summary>'` 总结当前上下文
   - `'<Clear>'` 清空历史
   - `'<ToolUsage>'` 查看工具使用说明
   - `'<ReloadConfig>'` 重新加载配置
   - `'<ListRule>'` 查看规则状态
2. 按 `Ctrl+J` 或 `Ctrl+]` 提交。
3. 系统会立刻执行对应操作，而不是把这些标记当作普通任务文本处理。

## 你会看到的提示与反馈

- 使用 `'<Summary>'` 后，系统会执行总结，并把总结结果作为后续继续工作的基础。
- 使用 `'<Clear>'` 后，当前历史会被清空。
- 使用 `'<ToolUsage>'` 后，系统会把工具使用说明加入当前会话提示中。
- 使用 `'<ReloadConfig>'` 后，系统会重新加载配置。
- 使用 `'<ListRule>'` 后，系统会显示规则表格，包含规则名称、内容预览、文件路径和状态，并显示 `✅ 已激活` 或 `🔴 未激活`。

## 注意事项

- 这些标记属于会话控制操作，不适合和普通任务描述混在一起大量使用。
- 清空历史和总结都会影响当前上下文，执行前先确认是否还需要保留当前对话链路。
- 如果你只是想让代理解释某个概念，不要误用这些控制标记。
