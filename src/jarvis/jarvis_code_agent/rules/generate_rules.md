**深度分析当前项目实际代码结构**，从项目真实实践中提炼15个核心维度，为该项目生成**AI IDE专用的代码开发指导规则**。

这些规则将作为AI IDE的智能提示和代码生成准则，指导AI在代码开发过程中遵循该项目的实际最佳实践。

任务要求：

1. 扫描项目目录结构，识别项目技术栈和开发模式特征
2. 分析现有代码风格、架构模式、设计哲学和编码惯例
3. 提炼项目特有的代码组织智慧和成功实践模式
4. 总结AI在代码开发中应遵循的核心原则和决策依据
5. 基于项目实际精神，为每个维度生成**AI可执行的指导规则**
6. 将规则写入.jarvis/rules/目录的单一文件中，供AI IDE直接调用

15个维度规则章节（合并到单一文件中）：

### 1. 编码规范
定义代码风格、命名约定、格式化标准

### 2. 工程结构规范  
定义目录布局、模块组织、文件放置规则

### 3. 错误处理规范
定义异常处理策略、错误日志记录、故障恢复机制

### 4. 数据处理规范
定义数据验证、清洗、转换和存储标准

### 5. 接口设计规范
定义API命名、参数规范、响应格式、版本控制

### 6. 性能与资源规范
定义性能指标、资源限制、优化策略

### 7. 版本控制规范
定义分支策略、提交规范、代码审查流程

### 8. 文档规范
定义代码注释、API文档、README标准

### 9. 协作流程规范
定义分支管理、PR流程、代码审查标准

### 10. 测试规范
定义测试策略、测试覆盖率、测试用例编写标准

### 11. 代码评审规范
定义审查标准、审查流程、质量门禁

### 12. 安全规范
定义安全检查、漏洞扫描、敏感数据处理规则

### 13. 构建部署规范
定义构建流程、部署策略、环境管理

### 14. 日志监控规范
定义日志级别、监控指标、告警规则

### 15. 容灾备份规范
定义备份策略、恢复流程、灾难应对预案

输出格式：每个文件使用Markdown格式，包含**AI开发指令**和**代码生成准则**，用简洁的机器学习友好的方式描述核心原则，避免冗长的人类阅读文档

生成原则：

- **AI指令优先**：规则应格式化为AI可直接理解和执行的指令，而非人类阅读文档
- **代码生成导向**：每个指导原则都必须能转化为具体的代码生成决策依据
- **必须基于项目实际代码验证**：每个维度都必须通过实际项目代码分析，确认其与项目开发模式的真实关联性
- **严禁虚构内容**：绝对禁止生成任何未经项目代码验证的虚构信息、假设或推测，所有规则必须基于真实存在的代码实践
- **项目特定性**：禁止通用建议，所有规则必须反映该项目的独特开发智慧
- **上下文感知**：规则应帮助AI理解何时何地应用特定模式，而非提供固定模板
- **简洁机器友好**：用结构化、标签化的方式表达核心概念，便于AI快速解析和应用
