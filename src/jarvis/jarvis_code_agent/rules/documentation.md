# 文档编写规则

## 文档原则

1. **清晰性**：
   - 文档应该清晰、准确
   - 使用简单的语言
   - 避免技术术语，必要时提供解释

2. **完整性**：
   - 文档应该覆盖所有重要功能
   - 包含使用示例
   - 说明限制和注意事项

3. **及时性**：
   - 代码变更时同步更新文档
   - 文档应该反映当前实现
   - 删除过时的文档

## 文档类型

1. **代码注释**：
   - 解释"为什么"，而不是"做什么"
   - 复杂逻辑需要注释
   - 公共 API 必须有文档字符串

2. **README**：
   - 项目概述和快速开始
   - 安装和配置说明
   - 使用示例

3. **API 文档**：
   - 函数签名和参数说明
   - 返回值说明
   - 使用示例

## 文档检查清单

- [ ] 所有公共 API 都有文档
- [ ] 复杂逻辑有注释说明
- [ ] README 包含快速开始指南
- [ ] 有使用示例
- [ ] 文档与代码保持同步
- [ ] 文档清晰、易读
